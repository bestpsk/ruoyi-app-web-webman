# 出库订单显示优化计划

## 问题描述
根据之前入库管理的修改模式，需要统一出库订单的显示逻辑：

### 当前问题（从截图分析）
1. **出库管理列表** - 总数量已显示"1盒（10支）"，基本符合要求
2. **查看出库单** - 存在以下问题：
   - **规格列**：固定显示"支"，未根据单位类型动态变化
   - **单位类型列**：显示"盒"，标签不够明确（应改为"主单位(整)"或"副单位(拆)"）
   - **数量列**：固定显示原始数值"10"，未根据单位类型转换显示
   - **出货单价列**：只显示数字"258"，缺少单位说明（如"258/支"）

### 目标效果（参考入库管理）
参考 [index.vue (入库)](front/src/views/wms/stockIn/index.vue) 的实现：

1. **查看出库单 - 规格列**：
   - `unitType === '1'`（主单位/整）：显示主单位规格（如"盒"）
   - `unitType === '2'`（副单位/拆）：显示副单位规格（如"支"）

2. **查看出库单 - 单位类型列**：
   - 显示完整标签："主单位(整)" 或 "副单位(拆)"

3. **查看出库单 - 数量列**：
   - `unitType === '1'`（主单位/整）：显示整套装数量（如"1"）
   - `unitType === '2'`（副单位/拆）：显示拆分后数量（如"10"）
   - 格式：`数量 + 单位`

4. **查看出库单 - 出货单价列**：
   - `unitType === '1'`（主单位/整）：显示整套价格（如"2580/盒"）
   - `unitType === '2'`（副单位/拆）：显示拆分单价（如"258/支"）
   - 格式：`价格 + "/" + 单位`

5. **打印预览**：
   - 同步更新数量和单价的显示格式

function formatItemQuantityForView(item) {
  const unitLabel = getUnitLabel(item.unit)
  const specLabel = getSpecLabel(item.spec)
  
  if (item.unitType === '1') {
    // 主单位(整)：显示实际整套数量
    const originalQty = item.originalQuantity || item.quantity || 0
    return originalQty + unitLabel

    // 副单位(拆)：显示实际拆分数量
    const originalQty = item.originalQuantity || item.quantity || 0

  }
}

### 步骤6：新增 formatItemPrice 函数

```javascript
function formatItemPrice(item) {
  const unitLabel = getUnitLabel(item.unit)
  const specLabel = getSpecLabel(item.spec)
  const packQty = item.packQty || 1
  
  if (item.unitType === '1') {
    // 主单位(整)：显示整套价格
    let price = item.salePrice || 0
    if (packQty > 1) {
      price = parseFloat((price * packQty).toFixed(2))
    }
    return price + '/' + (unitLabel || '')
  } else {
    // 副单位(拆)：显示拆分单价
    return item.salePrice + '/' + (specLabel || '')
  }
}
```
