# 库存盘点数据显示问题排查

## 问题现象

- **测试1**（CS1-20260504）：当前库存 `0盒（0支）`，预警
- **GCS-p7**：当前库存 `9盒（90支）`，正常

## 排查方向

1. 检查数据库中 biz_inventory 表的实际 quantity 值
2. 检查测试1的入库单是否已确认（status='1'）
3. 确认 confirmStockIn 逻辑是否正确执行

## 实施步骤

### 步骤1：创建诊断脚本检查数据

创建 PHP 脚本查询：
- biz_inventory 表的 quantity
- biz_stock_in 表中测试1相关的入库单及 status

### 步骤2：根据结果决定修复方案

可能原因：
- 入库单未确认 → 需要确认或手动同步
- confirmStockIn bug 未完全修复 → 检查代码
