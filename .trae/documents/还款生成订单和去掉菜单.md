# 还款生成订单 + 去掉还款管理菜单

## 需求说明
1. 还欠款时，在订单管理（biz_sales_order）中生成一个还款订单
2. 去掉业务管理下的"还款管理"子菜单（SQL文件中的菜单配置）

## 现有逻辑分析

### 订单表 `biz_sales_order` 结构
- `order_no`: 订单编号（SO+日期+序号）
- `order_status`: 订单状态（0未成交 1已成交 2已取消）
- `total_amount` / `deal_amount` / `paid_amount` / `owed_amount`: 金额字段
- `package_name`: 套餐名称
- `enterprise_audit_status` / `finance_audit_status`: 审核状态

### 还款记录表 `biz_repayment_record` 结构
- `repayment_no`: 还款编号（RP+日期+序号）
- `package_id` / `package_no` / `package_name`: 关联套餐
- `order_id` / `order_no`: 原订单ID和编号
- `repayment_amount`: 还款金额
- `status`: 状态（0待审核 1已审核 2已取消）

### 当前 `insertRepayment` 流程
1. 生成还款编号
2. 创建还款记录
3. 审核时更新套餐欠款金额

## 实现方案

### 1. 修改 `BizRepaymentService.php` 的 `insertRepayment` 方法
在创建还款记录的同时，在 `biz_sales_order` 表中生成一条还款订单：
- `order_no`: 使用 SO 编号（复用 `BizSalesOrderService::generateOrderNo()`）
- `order_status`: 设为 `'3'`（还款订单，新增状态值）
- `package_name`: 设为 `'还款-xxx'`（标识为还款订单）
- `total_amount` / `deal_amount` / `paid_amount`: 都等于还款金额
- `owed_amount`: 设为 0（还款订单无欠款）
- `enterprise_audit_status` / `finance_audit_status`: 设为 `'1'`（自动审核通过）
- 在还款记录中保存生成的订单ID和编号

### 2. 修改 `biz_repayment_record` 表
添加 `repayment_order_id` 和 `repayment_order_no` 字段，记录生成的还款订单

### 3. 修改订单管理页面
在 `front/src/views/business/order/index.vue` 中：
- 订单状态增加"还款"选项（值='3'）
- 还款订单在列表中显示时，标签用不同颜色区分

### 4. 修改SQL文件
去掉 `biz_repayment.sql` 中第2、3部分的菜单配置和权限分配

## 详细实现步骤

### 步骤1：修改 `biz_repayment.sql`
- 去掉菜单配置部分（第44-70行）
- 添加 `repayment_order_id` 和 `repayment_order_no` 字段到建表语句

### 步骤2：修改 `BizRepaymentService.php`
- 在 `insertRepayment` 方法中，同时创建还款订单
- 引入 `BizSalesOrder` 模型
- 生成订单编号，创建订单记录
- 将订单ID和编号写回还款记录

### 步骤3：修改 `BizRepaymentRecord.php` Model
- 在 `$fillable` 中添加 `repayment_order_id` 和 `repayment_order_no`

### 步骤4：修改订单管理页面
- `front/src/views/business/order/index.vue`：增加还款订单状态显示
- `front/src/views/business/sales/index.vue`：开单记录Tab中增加还款订单显示
