# 打印预览优化计划 - 规格显示与合计行

## 需求分析

### 问题1：规格显示不正确
**当前行为：**
- 规格列总是显示货品的副单位规格（如 `支`）
- 代码：`getSpecLabel(item.spec)` - spec 字段存储的是副单位规格

**用户期望：**
- 如果入库选择的是**主单位(整)**，规格应该显示主单位（如 `盒`）
- 如果入库选择的是**副单位(拆)**，规格应该显示副单位规格（如 `支`）
- 即：规格根据入库订单的 `unitType` 动态显示

### 问题2：合计行太窄导致换行
**当前行为：**
```
| 序号 | 货品名称 | 供货商 | 换算 | 数量 | 规格 | 单价 | 金额 |
| ...  | ...      | ...    | ...  | ...  | ...  | ...  | ...  |
| ------ 合计：------ | 2盒（20支） | 9380.00 |
       ↑ colspan=6     ↑ 10%宽度太窄    ↑ 14%
```
- 数量合计显示 `2盒（20支）` 内容较长
- 数量列只占 10% 宽度，导致内容换行

**用户建议方案：**
- 方案A：将数量、规格、单价三列合并给合计使用
- 方案B：合计行全部合并，左侧显示合计数量，右侧显示金额

## 数据结构说明

```javascript
// 入库明细项字段
{
  unitType: '1',        // '1'=主单位(整), '2'=副单位(拆)
  unit: '5',            // 主单位ID (如盒)
  spec: '1',            // 副单位规格ID (如支)
  packQty: 10,          // 换算率: 1盒=10支
  quantity: 1,          // 入库数量
  originalQuantity: 10, // 最小单位数量
}
```

## 实施步骤

### 步骤1：修改规格显示逻辑
修改打印预览中规格列的数据绑定（第221行）：

**当前代码：**
```html
<td align="center">{{ getSpecLabel(item.spec) }}</td>
```

**修改为：**
```html
<td align="center">{{ item.unitType === '1' ? getUnitLabel(item.unit) : getSpecLabel(item.spec) }}</td>
```

**逻辑说明：**
- `unitType === '1'` (主单位/整) → 显示主单位标签（如"盒"）
- `unitType === '2'` (副单位/拆) → 显示副单位规格标签（如"支"）

### 步骤2：优化合计行布局
采用**方案B**：合计行简化为两列显示

**当前结构（第226-232行）：**
```html
<tfoot>
  <tr>
    <td colspan="6" align="right"><strong>合 计：</strong></td>
    <td align="center"><strong>{{ formatTotalQuantity(form) }}</strong></td>
    <td align="right"><strong>{{ form.totalAmount ? Number(form.totalAmount).toFixed(2) : '0.00' }}</strong></td>
  </tr>
</tfoot>
```

**修改为：**
```html
<tfoot>
  <tr>
    <td colspan="7" align="right"><strong>合 计：{{ formatTotalQuantity(form) }}</strong></td>
    <td align="right"><strong>{{ form.totalAmount ? Number(form.totalAmount).toFixed(2) : '0.00' }}</strong></td>
  </tr>
</tfoot>
```

**效果：**
- 前7列合并显示："合 计：2盒（20支）"
- 最后1列显示金额："9380.00"
- 避免内容换行问题

### 步骤3：（可选）调整 formatTotalQuantity 函数
如果需要更简洁的合计显示格式，可优化该函数（第378-393行）。
