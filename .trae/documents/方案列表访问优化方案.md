# 方案列表访问优化方案

## 问题分析

当前方案管理页面（`d:\fuchenpro\front\src\views\business\plan\index.vue`）的设计存在以下问题：

1. **访问层级过深**：用户需要先进入企业管理页面 → 点击某个企业的"查看方案"按钮 → 才能看到方案列表
2. **方案列表是弹窗形式**：不是独立页面，无法直接通过菜单访问
3. **不便于全局管理**：用户无法快速查看所有企业的方案，必须逐个企业查看

## 解决方案

### 方案一：创建独立的方案管理页面（推荐）

创建一个新的方案管理页面，直接显示所有方案列表，支持按企业筛选。

**优点**：
- 用户可以直接从菜单访问方案列表
- 支持全局查看所有方案
- 保留原有企业管理功能不变

**实现步骤**：

1. **创建新的方案列表页面** `front/src/views/business/planList/index.vue`
   - 直接显示所有方案列表（不分企业）
   - 支持按企业名称、方案名称、审核状态筛选
   - 保留原有的方案详情、编辑、审核、出货等功能

2. **添加后端API** 支持全局方案列表查询
   - 修改 `BizPlanController.php` 添加全局列表接口
   - 支持按企业名称、方案名称等条件筛选

3. **添加菜单配置**
   - 在系统菜单中添加"方案管理"入口
   - 路由指向新的方案列表页面

### 方案二：改造现有页面结构

将现有的企业管理页面改为方案管理页面，企业列表作为侧边栏筛选条件。

**优点**：
- 复用现有代码
- 左侧企业列表，右侧方案列表，布局更清晰

**缺点**：
- 需要较大改动
- 可能影响现有功能

## 推荐实现：方案一

### 具体实现内容

#### 1. 前端：创建方案列表页面

文件：`front/src/views/business/planList/index.vue`

主要功能：
- 搜索条件：企业名称、方案名称、审核状态、日期范围
- 表格列：方案编号、企业名称、方案名称、分成比例、方案金额、配赠金额、剩余金额、审核状态、启用状态、操作
- 操作按钮：详情、编辑、提交审核、通过、驳回、出货
- 弹窗：方案详情、编辑方案、新建出货单（复用现有组件）

#### 2. 后端：添加全局方案列表API

文件：`webman/app/controller/business/BizPlanController.php`

添加方法：
```php
public function list(Request $request)
{
    // 支持按企业名称、方案名称、审核状态筛选
    // 关联查询企业信息
}
```

#### 3. 菜单配置

在系统管理 → 菜单管理中添加：
- 菜单名称：方案管理
- 路由地址：business/planList
