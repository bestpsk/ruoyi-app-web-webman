# 操作档案合并 + 新增无档案问题修复方案

## 📋 问题清单

### 问题1：一次性操作两个项目产生多条档案（未合并）
**现象**：同一天同一客户操作2个项目 → 产生2条独立档案  
**原因**：之前已修改了合并逻辑，但**Webman服务需要重启才能加载新代码**

### 问题2：新增操作没有生成档案
**现象**：最新的操作记录在"操作记录"表中存在，但"档案"表中没有对应记录  
**根因分析**：

查看 [BizOperationRecordService.php:77-82](file:///d:\fuchenpro\webman\app\service\BizOperationRecordService.php#L77-L82)：

```php
$record = BizOperationRecord::create($data);

try {
    $archiveService = new BizCustomerArchiveService();
    $archiveService->insertArchiveFromOperation($record);
} catch (\Exception $e) {
    \support\Log::error('写入操作档案失败: ' . $e->getMessage(), ['trace' => $e->getTraceAsString()]);
}
```

**可能原因**：
1. ⚠️ **服务未重启** - Webman常驻进程，代码修改后需重启
2. ⚠️ **异常被静默捕获** - try-catch 吞掉了错误，只写日志不报错
3. ⚠️ **`customer_id` 为空** - 第98行检查 `if (!$recordModel->customer_id) return null;`

## ✅ 解决方案

### 方案A：重启Webman服务（首选）

**为什么需要重启？**
- Webman 是基于 Workerman 的常驻内存框架
- PHP文件修改后，正在运行的进程不会自动重载
- 必须重启才能加载新的业务逻辑代码

**重启命令**：
```bash
# 在 webman 目录下执行
php start.php restart
# 或者
php start.php reload  # 平滑重启（推荐，不断开现有连接）
```

**验证重启成功**：
```bash
php start.php status
```

### 方案B：增强错误提示（可选优化）

如果重启后仍有问题，增强错误日志以便排查：

**修改 [BizOperationRecordService.php:77-82](file:///d:\fuchenpro\webman\app\service\BizOperationRecordService.php#L77-L82)**：

```php
$record = BizOperationRecord::create($data);

try {
    $archiveService = new BizCustomerArchiveService();
    $result = $archiveService->insertArchiveFromOperation($record);
    if (!$result) {
        \support\Log::warning('操作档案返回null', [
            'record_id' => $record->record_id,
            'customer_id' => $record->customer_id ?? 'NULL'
        ]);
    }
} catch (\Exception $e) {
    \support\Log::error('写入操作档案失败: ' . $e->getMessage(), [
        'trace' => $e->getTraceAsString(),
        'record_id' => $record->record_id ?? 'unknown',
        'customer_id' => $record->customer_id ?? 'unknown'
    ]);
}
```

## 💡 补充说明

### 关于历史数据

**已存在的重复档案不会被自动合并**，因为合并逻辑只在"新增操作时"触发。

如果需要修复历史重复档案，可以手动执行SQL或编写一次性修复脚本。

### 关于"新增操作无档案"

最常见的原因就是**服务未重启**。Webman与传统的PHP-FPM不同：
- **PHP-FPM**：每次请求都重新加载代码（无需重启）
- **Webman**：代码常驻内存（必须重启/reload）

> 这也是为什么之前的代码修改看起来"没生效"的原因。
