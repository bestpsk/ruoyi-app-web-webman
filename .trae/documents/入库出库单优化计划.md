# 入库单打印预览与出库单查看优化计划

## 需求分析

### 问题1：入库单打印预览
**子问题1.1：文字大小需要调整**
- 当前字体 11pt 可能偏小
- 需要整体合理调整

**子问题1.2：操作人/制单人显示用户名而非姓名**
- 当前显示：`admin`（用户名）
- 原因分析：
  - 后端 `BizStockInService::selectStockInById()` **没有**像出库单那样做用户名→姓名的转换
  - 出库单有转换逻辑（第75-80行），入库单缺失
  - 已保存的历史数据存的是用户名，需要兼容处理

**子问题1.3：签名区域仍有横线**
- 当前使用了 `sign-line` 和 `sign-line-long` 样式
- 用户要求完全去掉横线

---

### 问题2：查看出库单企业/员工显示数字
**现象：**
- 出库企业显示：`2`（应该是企业ID）
- 对接员工显示：`100`（应该是员工ID）

**原因分析：**

查看后端 [BizStockOutService.php](file:///d:/fuchenpro/webman/app/service/BizStockOutService.php) 第84-102行：

```php
$enterpriseName = $stockOutArray['enterprise_name'] ?? '';
if ((is_numeric($enterpriseName) || empty($enterpriseName)) && !empty($stockOutArray['enterprise_id'])) {
    $enterprise = BizEnterprise::find($stockOutArray['enterprise_id']);
    if ($enterprise) {
        $enterpriseName = $enterprise->enterprise_name;
    }
}
```

**问题所在：**
1. 数据库 `enterprise_name` 字段存的是 ID（如 `"2"`）而不是企业名
2. 后端检测到是数字，尝试查询 `BizEnterprise::find(2)` 但可能找不到
3. 用户确认数据库实际保存了正确的名称数据
4. **解决方案**：简化逻辑，直接读取数据库字段值，不做额外查询转换

### 步骤1：入库单后端添加姓名转换逻辑
**文件：** [BizStockInService.php](file:///d:/fuchenpro/webman/app/service/BizStockInService.php)

在 `selectStockInById()` 方法中（第77行 return 之前），添加：

```php
use app\model\SysUser;

// 转换操作人/制单人为真实姓名
if ($stockIn->operator_name && !preg_match('/[\x{4e00}-\x{9fa5}]/u', $stockIn->operator_name)) {
    $user = SysUser::where('user_name', $stockIn->operator_name)->first();
    if ($user && $user->real_name) {
        $stockIn->operator_name = $user->real_name;
    }
}

if ($stockIn->create_by && !preg_match('/[\x{4e00}-\x{9fa5}]/u', $stockIn->create_by)) {
    $user = SysUser::where('user_name', $stockIn->create_by)->first();
    if ($user && $user->real_name) {
        $stockIn->create_by = $user->real_name;
    }
}
```

### 步骤3：出库单后端简化企业/员工名称读取
**文件：** [BizStockOutService.php](file:///d:/fuchenpro/webman/app/service/BizStockOutService.php)

修改 `selectStockOutById()` 第82-112行，简化为直接读取：

```php
$stockOutArray = $stockOut->toArray();

// 直接读取数据库保存的名称，不做额外转换
$enterpriseName = $stockOutArray['enterprise_name'] ?? '-';
$contactEmployeeName = $stockOutArray['contact_employee_name'] ?? '-';
$responsibleName = $stockOutArray['responsible_name'] ?? '-';

// 如果值为空但有ID，尝试通过ID查找
if (empty($enterpriseName) && !empty($stockOutArray['enterprise_id'])) {
    $enterprise = BizEnterprise::find($stockOutArray['enterprise_id']);
    $enterpriseName = $enterprise ? $enterprise->enterprise_name : '-';
}

return [
    ...
    'enterpriseId' => $stockOutArray['enterprise_id'],
    'enterpriseName' => $enterpriseName,
    'contactEmployeeId' => $stockOutArray['contact_employee_id'],
    'contactEmployeeName' => $contactEmployeeName,
    'responsibleId' => $stockOutArray['responsible_id'],
    'responsibleName' => $responsibleName,
    ...
];
```

**关键变更：**
- 移除 `is_numeric()` 判断条件
- 如果数据库有值就直接用，只有空值才去查询
- 这样即使存的是正确名称也能正常显示
