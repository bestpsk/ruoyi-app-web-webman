# 照片加载失败修复方案

## 🔍 根因分析

### 数据链路追踪

```
1. 文件上传 → 后端返回: "20260509/74e407483fabcd1997c631eab465649e.png"
2. 存入数据库: ["20260509/74e407...png", "20260509/a4c36981...jpg"]
3. 前端读取 → 拼接 baseUrl(/dev-api) → "/dev-api/20260509/xxx.png"
4. Vite代理 → "http://localhost:8787/20260509/xxx.png"  ❌ 404!
```

### 实际文件位置
```
webman/public/profile/upload/20260509/74e407483fabcd1997c631eab465649e.png ✅ 存在
```

### 问题原因
| 项目 | 值 | 说明 |
|------|-----|------|
| `VITE_APP_BASE_API` | `/dev-api` | API代理前缀 |
| 数据库存储路径 | `20260509/xxx.png` | 相对路径（缺前缀） |
| 实际文件路径 | `profile/upload/20260509/xxx.png` | 完整相对路径 |
| 当前拼接结果 | `/dev-api/20260509/xxx.png` → `http://localhost:8787/20260509/xxx.png` | **❌ 路径错误！** |

**核心问题**：
- `/dev-api` 是API代理前缀，用于访问后端接口
- 静态文件应该直接通过Webman的静态文件服务访问
- 静态文件根目录是 `public/`，所以正确路径应该是 `/profile/upload/20260509/xxx.png`

---

## ✅ 修复方案

### 方案：使用正确的静态文件基础路径

不使用 `VITE_APP_BASE_API`（API代理），而是使用**静态文件服务的基础路径**：

```javascript
// 图片应该直接访问静态文件，不走API代理
const imgBaseUrl = import.meta.env.VITE_APP_BASE_API?.replace('/dev-api', '') || ''
// 或者更明确地：
const imgBaseUrl = ''  // 因为前端代理会把 /dev-api 转发到后端，而静态文件不需要代理
```

但实际上更好的方式是：**根据当前环境动态构建图片URL**

```javascript
function getImgUrl(path) {
  if (!path) return ''
  // 如果已经是完整URL，直接返回
  if (path.startsWith('http')) return path
  // 拼接完整路径：/profile/upload/ + 相对路径
  return '/profile/upload/' + path
}
```

#### 修改2：重写 parseArchivePhotos 方法

```javascript
function parseArchivePhotos(photosStr) {
  if (!photosStr) return []
  try {
    const parsed = JSON.parse(photosStr)
    return Array.isArray(parsed) ? parsed.map(url => {
      if (!url) return null
      // 如果已是完整URL，直接返回
      if (url.startsWith('http://') || url.startsWith('https://')) return url
      // 拼接静态文件路径前缀
      return '/profile/upload/' + url
    }).filter(Boolean) : []
  } catch { 
    return photosStr.split(',').map(s => {
      s = s.trim()
      if (!s) return null
      if (s.startsWith('http') || s.startsWith('/')) return s
      return '/profile/upload/' + s
    }).filter(Boolean)
  }
}
```
