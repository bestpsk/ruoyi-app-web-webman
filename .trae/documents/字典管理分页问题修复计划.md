# 字典管理分页问题修复计划

## 📋 问题描述

字典管理页面点击下一页没有作用，还是只显示第一页。

---

## 🔍 问题分析

### 问题根源

刚才修复字典筛选问题时，在 Controller 中添加了 `convert_to_snake_case` 参数转换，导致分页参数也被转换：

#### 参数转换流程
```
前端发送: pageNum=2&pageSize=10 (驼峰命名)
    ↓
Controller: convert_to_snake_case($request->all())
    ↓
转换结果: page_num=2&page_size=10 (下划线命名)
    ↓
Service层: $params['pageNum'] (期望驼峰命名)
    ↓
结果: 找不到参数，默认使用第1页 ❌
```

### 代码对比

#### Controller 层（刚修改）
```php
public function list(Request $request)
{
    $service = new SysDictDataService();
    $params = convert_to_snake_case($request->all());  // 转换所有参数
    $result = $service->selectDictDataList($params);
    return TableDataInfo::result($result->items(), $result->total());
}
```

#### Service 层（期望驼峰命名）
```php
public function selectDictDataList($params = [])
{
    // ...
    $pageNum = intval($params['pageNum'] ?? 1);   // 期望驼峰
    $pageSize = intval($params['pageSize'] ?? 10); // 期望驼峰
    return $query->orderBy('dict_sort', 'asc')->paginate($pageSize, ['*'], 'page', $pageNum);
}
```

#### convert_to_snake_case 函数
```php
function to_snake_case($key)
{
    return strtolower(preg_replace('/([a-z])([A-Z])/', '$1_$2', $key));
}
// pageNum → page_num
// pageSize → page_size
// dictType → dict_type
```

## 🎯 解决方案

### 方案选择：修改 Service 层使用下划线命名

**理由：**
- ✅ 符合数据库字段命名规范（下划线命名）
- ✅ 与数据字段命名保持一致（`dict_type`, `dict_label`）
- ✅ 一次性修复，避免后续问题
- ✅ `GenTableService.php` 已采用此方式（兼容两种命名）

### 修复范围

需要修改 **15个 Service 文件**，将分页参数从驼峰改为下划线：

#### 系统模块（11个）
1. `SysDictDataService.php` - 字典数据
2. `SysDictTypeService.php` - 字典类型
3. `SysUserService.php` - 用户管理
4. `SysRoleService.php` - 角色管理（3处）
5. `SysPostService.php` - 岗位管理
6. `SysConfigService.php` - 参数配置
7. `SysNoticeService.php` - 通知公告（2处）
8. `SysOperLogService.php` - 操作日志
9. `SysLogininforService.php` - 登录日志
10. `SysJobService.php` - 定时任务
11. `SysJobLogService.php` - 任务日志

#### 业务模块（1个）
12. `BizEnterpriseService.php` - 企业管理

#### 代码生成（1个）
13. `GenTableService.php` - 已兼容，无需修改

## 📝 实施步骤

### 步骤1：修改 SysDictDataService.php

**修改前：**
```php
$pageNum = intval($params['pageNum'] ?? 1);
$pageSize = intval($params['pageSize'] ?? 10);
```

**修改后：**
```php
$pageNum = intval($params['page_num'] ?? 1);
$pageSize = intval($params['page_size'] ?? 10);
```

### 步骤2：批量修改其他 Service 文件

按照相同方式修改所有 Service 文件的分页参数。
