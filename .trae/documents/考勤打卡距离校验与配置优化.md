### 问题2：缺少考勤配置功能

**当前状态：**
- `getActiveRule()` 只返回第一个 `status='0'` 的规则
- 所有员工共用同一个考勤规则

**期望功能：**
- 不同员工/部门可以使用不同的考勤规则

### 方案B：新增考勤配置功能

#### 数据库设计

**新增表：`biz_attendance_config`（考勤配置表）**

| 字段 | 类型 | 说明 |
|-----|------|------|
| config_id | int | 主键 |
| rule_id | int | 关联考勤规则 |
| user_id | int | 关联用户（可选） |
| dept_id | int | 关联部门（可选） |
| config_type | tinyint | 1=用户级, 2=部门级 |
| status | char(1) | 状态 |
| create_time | datetime | 创建时间 |

**优先级规则：**
1. 用户级配置 > 部门级配置 > 默认规则
2. 如果用户有个人配置，使用个人配置的规则
3. 如果用户所属部门有配置，使用部门配置的规则
4. 否则使用默认规则（getActiveRule）

#### 菜单结构

```
考勤管理
├── 考勤记录 (现有)
├── 考勤规则 (现有)
└── 考勤配置 (新增) ← 配置员工/部门使用哪个规则
```

#### 功能实现

**1. 新增后端服务**

`BizAttendanceConfigService.php`:
```php
// 获取用户应使用的考勤规则
public function getUserRule($userId) {
    // 1. 查找用户级配置
    $userConfig = BizAttendanceConfig::where('user_id', $userId)
        ->where('config_type', 1)->where('status', '0')->first();
    if ($userConfig) {
        return BizAttendanceRule::find($userConfig->rule_id);
    }
    
    // 2. 查找部门级配置
    $user = SysUser::find($userId);
    if ($user && $user->dept_id) {
        $deptConfig = BizAttendanceConfig::where('dept_id', $user->dept_id)
            ->where('config_type', 2)->where('status', '0')->first();
        if ($deptConfig) {
            return BizAttendanceRule::find($deptConfig->rule_id);
        }
    }
    
    // 3. 使用默认规则
    return (new BizAttendanceRuleService())->getActiveRule();
}
```

**2. 修改打卡逻辑**

将 `clockIn` 中的：
```php
$rule = $ruleService->getActiveRule();
```
改为：
```php
$rule = $configService->getUserRule($userId);
```

**3. 新增前端页面**

`attendance/config.vue`:
- 列表显示所有配置
- 可按用户/部门配置规则
- 支持批量配置
