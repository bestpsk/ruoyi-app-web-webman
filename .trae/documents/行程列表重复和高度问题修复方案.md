# 行程安排列表重复和高度问题修复方案

## 🚨 问题分析

### 问题1：数据重复显示（2条变12条）

#### 现象
- 正常应该显示2条行程记录
- 实际显示了12条
- 存在大量重复数据

#### 根因定位

**文件位置**：[index.vue#L175-L200](file:///d:/fuchenpro/AppV3/src/pages/business/schedule/index.vue#L175-L200)

**分组逻辑**：
```javascript
function groupScheduleList(list) {
  const groupMap = new Map()

  list.forEach(item => {
    const key = `${item.userId}_${item.enterpriseId}_${item.purpose}_${item.status}`
    // 按员工+企业+目的+状态分组
  })
}
```

**数据加载逻辑**（[index.vue#L233](file:///d:/fuchenpro/AppV3/src/pages/business/schedule/index.vue#L233)）：
```javascript
scheduleList.value = isRefresh ? grouped : [...scheduleList.value, ...grouped]
```

**可能的原因**：

1. **后端返回重复数据** ⭐⭐⭐⭐⭐（最可能）
   - 后端可能对每个日期都返回一条记录
   - 例如：一个行程有6天，后端返回6条记录（每天一条）
   - 前端分组后应该是1条，但如果分组逻辑有问题，会显示6条

2. **分页加载重复** ⭐⭐⭐⭐
   - 第241-246行的 `loadMore()` 函数
   - 每次加载更多时，直接追加数据：`[...scheduleList.value, ...grouped]`
   - 如果后端分页有bug，返回了重复数据，前端没有去重

3. **分组key不够精确** ⭐⭐⭐
   - 当前key：`${userId}_${enterpriseId}_${purpose}_${status}`
   - 如果同一员工对同一企业有多个不同目的的行程，会被正确分组
   - 但如果remark不同，会被合并到一起

### 问题2：可视高度太小（只能显示2条）

#### 现象
- 列表可视区域高度太小
- 默认只能显示2条数据
- 需要频繁滚动

#### 根因定位

**文件位置**：[index.vue#L279-L282](file:///d:/fuchenpro/AppV3/src/pages/business/schedule/index.vue#L279-L282)

```javascript
function calcScrollHeight() {
  const systemInfo = uni.getSystemInfoSync()
  scrollHeight.value = systemInfo.windowHeight - 220  // ← 220太小了！

```

**高度计算分析**：

- 减去220px：顶部搜索区域 + 筛选区域
- 剩余：530px
- 每条卡片高度约：~200px
- 可显示：530 / 200 ≈ 2.6条

**问题**：
- 220px 的预留高度不够
- 没有考虑：
  - 顶部状态栏（44px）
  - 搜索区域（实际约120px）
  - 筛选标签区域（约60px）
  - 底部浮动按钮（120rpx ≈ 60px）
  - 页面内边距（48rpx ≈ 24px）

## 🔧 解决方案

### 方案1：修复数据重复问题

#### 方法A：增强分组逻辑的准确性 ⭐⭐⭐⭐⭐（推荐）

**修改位置**：[index.vue#L175-L200](file:///d:/fuchenpro/AppV3/src/pages/business/schedule/index.vue#L175-L200)

**改进点**：
1. 添加remark到分组key
2. 添加日期去重逻辑
3. 添加调试日志

**修改后的代码**：
```javascript
function groupScheduleList(list) {
  const groupMap = new Map()

  console.log('[Schedule] 原始数据条数:', list.length)
  console.log('[Schedule] 原始数据:', list)

  list.forEach(item => {
    // 添加remark到分组key，确保不同备注的行程分开
    const key = `${item.userId}_${item.enterpriseId}_${item.purpose}_${item.status}_${item.remark || ''}`

    if (!groupMap.has(key)) {
      groupMap.set(key, {
        ...item,
        scheduleIds: [item.scheduleId],
        scheduleDates: [item.scheduleDate]
      })
    } else {
      const group = groupMap.get(key)
      // 去重：避免重复添加相同的日期
      if (!group.scheduleIds.includes(item.scheduleId)) {
        group.scheduleIds.push(item.scheduleId)
      }
      if (!group.scheduleDates.includes(item.scheduleDate)) {
        group.scheduleDates.push(item.scheduleDate)
      }
    }
  })

  const result = Array.from(groupMap.values())
    .map(group => ({
      ...group,
      scheduleDates: [...new Set(group.scheduleDates)].sort()  // 去重并排序
    }))
    .sort((a, b) => new Date(a.scheduleDates[0]) - new Date(b.scheduleDates[0]))

  console.log('[Schedule] 分组后数据条数:', result.length)
  console.log('[Schedule] 分组后数据:', result)

  return result
}
```

#### 方法B：修复分页加载逻辑 ⭐⭐⭐⭐

**修改位置**：[index.vue#L213-L239](file:///d:/fuchenpro/AppV3/src/pages/business/schedule/index.vue#L213-L239)

**改进点**：
1. 添加数据去重逻辑
2. 避免分页加载时重复追加

**修改后的代码**：
```javascript
async function getList(isRefresh = false) {
  if (loading.value) return
  loading.value = true
  if (isRefresh) { queryParams.pageNum = 1; loadStatus.value = 'loadmore' }

  try {
    const [year, month] = queryParams.yearMonth.split('-')
    const startDate = `${year}-${month}-01`
    const endDate = `${year}-${month}-${new Date(year, month, 0).getDate()}`
    const params = { ...queryParams, startDate, endDate }
    if (params.keyword) { params.userName = params.keyword; params.enterpriseName = params.keyword }
    delete params.keyword; delete params.yearMonth

    const response = await listSchedule(params)
    const data = response.data || response
    const list = data.rows || []
    const total = data.total || 0

    console.log('[Schedule] API返回条数:', list.length, '总数:', total)

    const grouped = groupScheduleList(list)

    if (isRefresh) {
      scheduleList.value = grouped
    } else {
      // 去重：避免重复添加已有的数据
      const existingIds = new Set(scheduleList.value.flatMap(item => item.scheduleIds))
      const newItems = grouped.filter(item => 
        !item.scheduleIds.some(id => existingIds.has(id))
      )
      scheduleList.value = [...scheduleList.value, ...newItems]
    }

    loadStatus.value = scheduleList.value.length >= total ? 'nomore' : 'loadmore'
  } catch (e) {
    console.error('获取行程列表失败:', e)
    loadStatus.value = 'error'
  } finally { loading.value = false; refreshing.value = false }
}
```

## ⚠️ 注意事项

### 1. 数据重复的根本原因

修复后如果仍然有重复，可能是后端问题：
- 检查后端API是否有bug
- 确认数据库是否有重复记录
- 查看后端分页逻辑是否正确

### 2. 高度计算的适配

不同设备的屏幕高度不同：
- iPhone SE：667px
- iPhone 12：844px
- iPhone 14 Pro Max：932px

建议使用动态计算：
```javascript
function calcScrollHeight() {
  const systemInfo = uni.getSystemInfoSync()
  const topAreaHeight = 180  // 顶部区域固定高度
  const bottomAreaHeight = 120  // 底部浮动按钮高度
  scrollHeight.value = systemInfo.windowHeight - topAreaHeight - bottomAreaHeight
}
```

### 3. 性能考虑

添加调试日志后，生产环境应移除：
```javascript
if (process.env.NODE_ENV === 'development') {
  console.log('[Schedule] 调试信息:', data)
}
```
