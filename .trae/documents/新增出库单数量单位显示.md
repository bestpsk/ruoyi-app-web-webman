# 新增出库单数量列显示单位修复计划

## 问题描述

新增出库单时，**数量列后面的单位没了**。

### 当前状态
- **查看模式** (`isView=true`)：数量显示 "1 盒" 或 "10 支" ✅ 有单位
- **新增/编辑模式** (`!isView`)：数量只有数字输入框 ❌ **没有单位**

## 问题分析

**当前代码** (stockOut/index.vue 第160-165行):
```vue
<el-table-column label="数量" min-width="120" align="center">
  <template #default="scope">
    <el-input-number v-if="!isView" v-model="scope.row.quantity" ... />
    <span v-else>{{ ... }} {{ 动态单位 }}</span>  <!-- 只有查看模式有单位 -->
  </template>
</el-table-column>
```

**问题**: 新增/编辑模式下，`el-input-number` 后面没有显示单位。

---

## 解决方案

### 方案：在 el-input-number 后面添加动态单位显示

修改新增/编辑模式下的数量列，在输入框后显示当前选择的单位：

```vue
<el-table-column label="数量" min-width="140" align="center" header-align="center">
  <template #default="scope">
    <div v-if="!isView" style="display: flex; align-items: center; justify-content: center; gap: 8px;">
      <el-input-number v-model="scope.row.quantity" :min="1" @change="calcAmount(scope.$index)" style="width: 100px;" />
      <span style="font-size: 13px; color: #606266; min-width: 30px;">
        {{ scope.row.unitType === '1' ? getUnitLabel(scope.row.unit) : getSpecLabel(scope.row.spec) }}
      </span>
    </div>
    <span v-else>{{ (scope.row.originalQuantity || scope.row.quantity || 0) }} {{ scope.row.unitType === '1' ? getUnitLabel(scope.row.unit) : getSpecLabel(scope.row.spec) }}</span>
  </template>
</el-table-column>
```

## 预期效果

### 新增/编辑模式
```
数量
┌─────────────────────────────┐
│  [  1  ] 盒                 │  ← 主单位(整) 显示 "盒"
├─────────────────────────────┤
│  [ 10 ] 支                 │  ← 副单位(拆) 显示 "支"
└─────────────────────────────┘
```

### 查看模式（保持不变）
```
数量
┌──────────┐
│  1 盒    │  ← 显示 数字 + 单位
├──────────┤
│  10 支   │
└──────────┘
```

---

## 单位显示逻辑

| unitType | 显示内容 | 示例 |
|----------|---------|------|
| `'1'` (主单位/整) | **主单位规格** | `盒`、`箱`、`包` |
| `'2'` (副单位/拆) | **副单位规格** | `支`、`个`、`件` |

**动态变化**: 当用户切换单位类型时，单位会自动更新显示！

---

## 测试要点

1. ✅ 新增出库单 → 数量列显示：`[输入框] + 单位`
2. ✅ 选择货品后，默认显示主单位（如"盒"）
3. ✅ 切换单位类型为"副单位(拆)"，单位自动变为"支"
4. ✅ 不同货品显示各自的单位
5. ✅ 查看出库单时，数量格式保持不变
