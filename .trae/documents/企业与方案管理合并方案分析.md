# 企业管理与方案管理合并方案分析

## 当前结构分析

### 企业管理页面 (`enterprise/index.vue`)
| 字段 | 说明 |
|------|------|
| 企业名称 | 主键 |
| 老板姓名 | 联系人 |
| 联系电话 | - |
| 企业类型 | 字典 |
| 门店数量 | 数值 |
| 年业绩 | 金额 |
| 企业级别 | 字典 |
| 服务人 | 负责员工 |
| 开始合作 | 日期 |
| 结束合作 | 日期 |
| 状态 | 启用/停用 |
| 创建时间 | - |

**操作**：新增企业、修改企业、删除企业

### 方案管理页面 (`plan/index.vue`)
| 字段 | 说明 |
|------|------|
| 企业名称 | 主键 |
| 开始合作 | 日期 |
| 结束合作 | 日期 |
| 方案数量 | 统计 |
| 状态 | 启用/停用 |

**操作**：查看方案、开方案（弹窗显示方案列表）

### 字段重叠分析
- **重叠字段**：企业名称、开始合作、结束合作、状态
- **企业管理独有**：老板姓名、联系电话、企业类型、门店数量、年业绩、企业级别、服务人、创建时间
- **方案管理独有**：方案数量

## 合并可行性评估

### ✅ 适合合并的理由

1. **数据关系紧密**：方案属于企业，是1:N关系
2. **操作关联性强**：查看企业时经常需要查看/操作方案
3. **减少页面跳转**：用户无需在两个页面间切换
4. **字段重叠度高**：核心字段基本一致

### ⚠️ 需要注意的问题

1. **页面复杂度增加**：合并后列数和操作会增加
2. **权限控制**：企业管理权限和方案管理权限需要分开控制
3. **一个企业多个方案**：如果直接合并列表，会导致数据行数膨胀

## 推荐方案：主从结构（左侧企业 + 右侧方案）

### 布局设计

```
┌─────────────────────────────────────────────────────────────┐
│  搜索栏：企业名称 | 老板姓名 | 企业类型 | 状态 | [搜索] [重置]  │
├─────────────────────────────────────────────────────────────┤
│  [新增企业] [新增方案]                                        │
├──────────────────────┬──────────────────────────────────────┤
│     企业列表          │           方案列表                    │
│  ┌────────────────┐  │  方案编号 | 方案名称 | 金额 | 状态 | 操作│
│  │ ▸ 企业A (3方案) │  │  PL001   | 方案1   | 5000 | 已审核|...│
│  │   企业B (1方案) │  │  PL002   | 方案2   | 3000 | 待审核|...│
│  │   企业C (0方案) │  │  PL003   | 方案3   | 2000 | 草稿  |...│
│  └────────────────┘  │                                      │
│                      │  [详情] [编辑] [提交审核] [出货]       │
└──────────────────────┴──────────────────────────────────────┘
```

### 交互逻辑

1. **左侧企业列表**：
   - 点击企业 → 右侧显示该企业的方案列表
   - 显示方案数量
   - 支持搜索筛选

2. **右侧方案列表**：
   - 显示选中企业的所有方案
   - 支持方案的新增、编辑、审核、出货等操作
   - 未选择企业时显示提示

3. **新增企业**：弹出企业表单
4. **新增方案**：先选择企业，再弹出方案表单

### 优点

1. **信息层次清晰**：企业为主，方案为从
2. **操作便捷**：选择企业即可看到所有方案
3. **减少弹窗层级**：方案列表直接显示，无需再弹窗
4. **保留原有功能**：企业管理和方案管理功能都保留

### 实现步骤

1. **修改 `plan/index.vue`**：
   - 添加左侧企业列表（带搜索）
   - 右侧显示方案列表
   - 合并企业管理的搜索条件

2. **保留 `enterprise/index.vue`**：
   - 作为完整的企业管理页面
   - 支持更详细的企业信息维护

3. **菜单调整**：
   - "企业管理"：完整的企业CRUD
   - "方案管理"：企业+方案的合并视图（主从结构）

## 替代方案：表格展开行

在企业列表中，点击展开按钮可以看到该企业的方案列表。

**优点**：改动最小
**缺点**：展开后占用空间大，不适合方案数量多的情况

## 结论

**推荐采用主从结构合并**，将方案管理页面改造为左侧企业列表+右侧方案列表的布局。这样既解决了方案列表藏得太深的问题，又保持了清晰的信息层次。
