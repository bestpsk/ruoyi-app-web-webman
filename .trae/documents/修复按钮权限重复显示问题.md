# 修复按钮权限重复显示问题

## 问题分析

当前实现中，按钮权限显示了两次：
1. **树形子节点**：按钮权限作为菜单的子节点显示在树形结构中
2. **右侧复选框**：按钮权限显示在菜单右侧

用户需求：去掉树形子节点中的按钮权限，只保留右侧复选框显示。

## 解决方案

创建一个过滤后的树数据，只保留菜单节点，过滤掉按钮权限节点（叶子节点）。

### 实施步骤

**文件**: `front/src/views/system/role/index.vue`

#### 步骤1：添加计算属性过滤树数据

```javascript
const menuTreeOptions = computed(() => {
  return filterButtonNodes(menuOptions.value)
})

function filterButtonNodes(nodes) {
  if (!nodes) return []
  return nodes.map(node => {
    if (node.children && node.children.length > 0) {
      const hasNestedMenu = node.children.some(child => child.children && child.children.length > 0)
      if (hasNestedMenu) {
        return {
          ...node,
          children: filterButtonNodes(node.children)
        }
      } else {
        return {
          ...node,
          children: []
        }
      }
    }
    return node
  })
}
```

#### 步骤2：修改el-tree使用过滤后的数据

```html
<el-tree
   class="tree-border"
   :data="menuTreeOptions"
   ...
>
```

#### 步骤3：保留原始menuOptions用于获取按钮权限

模板中获取按钮权限时仍使用原始数据：
```html
<span v-if="getButtonPermissions(data.id).length > 0" class="button-permissions">
   <el-checkbox
      v-for="btn in getButtonPermissions(data.id)"
      ...
```

添加辅助函数：
```javascript
function getButtonPermissions(parentId) {
  const parent = findNodeById(menuOptions.value, parentId)
  if (parent && parent.children && isButtonGroup(parent.children)) {
    return parent.children
  }
  return []
}

function findNodeById(nodes, id) {
  for (const node of nodes) {
    if (node.id === id) return node
    if (node.children) {
      const found = findNodeById(node.children, id)
      if (found) return found
    }
  }
  return null
}
```
