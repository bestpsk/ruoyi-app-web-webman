## 🔍 问题分析

### 问题1：record_id=3,4,5,6 完全没有生成档案

**时间线**：
- `23:01:36` - record_id=1,2 创建 → 生成了 archive_id=2,3 ✅（旧代码）
- `23:22:06` - record_id=3 创建 → **无档案** ❌
- `23:29:20` - record_id=5,6 创建 → **无档案** ❌

**根因推测**：
Webman服务重启后，新代码可能存在运行时错误，被 try-catch 静默捕获。

### 问题2：archive_id=2,3 未合并

这两条是用**旧代码**创建的（23:01:36），旧逻辑只按 `source_id` 去重，不按"客户+日期"合并。

## 🔧 修复方案

### 步骤1：增强错误日志（排查根因）

修改 [BizOperationRecordService.php:77-82](file:///d:\fuchenpro\webman\app\service\BizOperationRecordService.php#L77-L82)，让错误更可见：

```php
$record = BizOperationRecord::create($data);

try {
    $archiveService = new BizCustomerArchiveService();
    $result = $archiveService->insertArchiveFromOperation($record);
    if (!$result) {
        \support\Log::warning('操作档案返回null', [
            'record_id' => $record->record_id,
            'customer_id' => $record->customer_id ?? 'NULL',
            'product_name' => $record->product_name ?? ''
        ]);
    } else {
        \support\Log::info('操作档案创建成功', [
            'record_id' => $record->record_id,
            'archive_id' => $result->archive_id ?? 'unknown'
        ]);
    }
} catch (\Exception $e) {
    \support\Log::error('写入操作档案失败: ' . $e->getMessage(), [
        'trace' => $e->getTraceAsString(),
        'record_id' => $record->record_id ?? 'unknown',
        'customer_id' => $record->customer_id ?? 'unknown',
        'file' => $e->getFile(),
        'line' => $e->getLine()
    ]);
}
```

```bash
cd d:\fuchenpro\webman

php windows.php start -d
```

> `reload` 可能不会完全重新加载所有文件，`stop + start` 确保完全重启

## 💡 备选方案：前端批量提交

如果后端合并逻辑难以调试，可以考虑从**前端**改为批量提交：

**当前流程**（每次操作1个项目，循环调用N次）：
```
选择2个项目 → 提交请求1(品项1) → 提交请求2(品项2)
```

**优化流程**（一次性提交所有项目）：
```
选择2个项目 → 提交1次请求([{品项1}, {品项2}])
```

这样后端只需要创建1条操作记录和1条档案，从根本上避免重复问题。
