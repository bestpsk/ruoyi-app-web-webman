## 🔍 问题1：统计数据全部显示为0

### 问题定位

**文件位置**：[record.vue#L189-L191](file:///d:/fuchenpro/AppV3/src/pages/attendance/record.vue#L189-L191)

```javascript
if (statsRes.data) {
  stats.value = statsRes.data  // 直接赋值，可能字段名不匹配
}
```

**初始状态定义**（[record.vue#L102](file:///d:/fuchenpro/AppV3/src/pages/attendance/record.vue#L102)）：
```javascript
const stats = ref({ normal: 0, late: 0, early: 0, late_and_early: 0, absent: 0 })
```

### 可能原因分析

#### 原因A：后端返回的字段名不匹配 ⭐⭐⭐⭐⭐（最可能）

**前端期望的字段名**：
- `normal` - 正常次数
- `late` - 迟到次数
- `early` - 早退次数
- `absent` - 缺勤次数

**后端可能返回的字段名**（常见Java后端命名）：
- `normalCount` / `normal_count`
- `lateCount` / `late_count`
- `earlyCount` / `early_count`
- `absentCount` / `absent_count`

或者完全不同的结构。

#### 原因B：API响应结构嵌套过深 ⭐⭐⭐

根据 [request.js#L47](file:///d:/fuchenpro/AppV3/src/utils/request.js#L47) 的实现：
```javascript
resolve(res.data)  // request已解包一层
```

所以在 record.vue 中：
- `statsRes` = 后端返回的 `{ code: 200, data: {...}, msg: '' }`
- `statsRes.data` = 实际的统计对象

但如果后端返回的是多层嵌套：
```json
{
  "code": 200,
  "data": {
    "stats": { "normal": 10, ... }  // 多了一层
  }
}
```
则 `statsRes.data` 是 `{ stats: {...} }` 而非直接的统计对象。

#### 原因C：接口调用失败或无数据 ⭐⭐

可能的情况：
- 接口报错但被静默捕获
- 当前月份确实没有数据
- 权限问题导致返回空数据

### 🔧 解决方案

#### 步骤1：添加调试日志查看实际返回数据

修改 [record.vue#L183-L191](file:///d:/fuchenpro/AppV3/src/pages/attendance/record.vue#L183-L191)：

  if (statsRes.data) {
    // 尝试多种可能的字段映射
    const rawData = statsRes.data
    stats.value = {
      normal: rawData.normal || rawData.normalCount || 0,
      late: rawData.late || rawData.lateCount || 0,

## 🔍 问题3：打卡详情显示经纬度而非地址

### 问题现象

从截图中可以看到：
- **2026-04-29 上班打卡**：地址显示 `31.044384, 121.466400` （经纬度坐标）
- **2026-04-29 下班打卡**：地址显示 `115151541` （可能是ID或其他数字）
- **2026-04-30 补卡**：地址显示 `1212` （未知格式）

### 问题定位

**文件位置**：[record.vue#L73](file:///d:/fuchenpro/AppV3/src/pages/attendance/record.vue#L73)

```vue
<text class="clock-address">{{ clock.address || '' }}</text>
```

**原因分析**：

1. 打卡时如果地址解析失败，系统会将**经纬度坐标作为address保存**（见 [index.vue#L287](file:///d:/fuchenpro/AppV3/src/pages/attendance/index.vue#L287)）
   ```javascript
   location.value.address = `${lat.toFixed(6)}, ${lng.toFixed(6)}`
   ```

2. 或者用户手动输入了无效数据

3. 显示时未做过滤，直接显示了所有address值

### 🔧 解决方案

#### 方案A：添加地址格式检测和友好显示（推荐）⭐⭐⭐⭐⭐

复用 [index.vue#L253-L256](file:///d:/fuchenpro/AppV3/src/pages/attendance/index.vue#L253-L256) 的 `isCoordinateOnly` 函数逻辑。

**步骤1**：在 record.vue 中添加辅助函数

```javascript
// 检测是否为纯经纬度格式
function isCoordinateOnly(str) {
  if (!str) return false
  return /^\s*[-+]?\d+\.\d+,\s*[-+]?\d+\.\d+\s*$/.test(str)
}

// 检测是否为纯数字（可能是ID）
function isNumericOnly(str) {
  if (!str) return false
  return /^\d+$/.test(str.trim())
}

// 格式化地址显示
function formatAddress(address) {
  if (!address) return ''
  
  // 如果是经纬度坐标，不显示
  if (isCoordinateOnly(address)) {
    return ''
  }
  
  // 如果是纯数字，不显示
  if (isNumericOnly(address)) {
    return ''
  }
  
  // 正常地址文本，限制长度
  if (address.length > 30) {
    return address.substring(0, 30) + '...'
  }
  
  return address
}
```

**步骤2**：修改模板显示逻辑

修改 [record.vue#L69-L74](file:///d:/fuchenpro/AppV3/src/pages/attendance/record.vue#L69-L74)：

#### 方案B：后端存储优化（长期方案）⭐⭐⭐
