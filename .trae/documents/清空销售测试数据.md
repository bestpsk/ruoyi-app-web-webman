# 清空销售测试数据计划

## 📋 任务目标
清空所有顾客的开单、操作、欠款、档案、开单记录、操作记录、订单数据，以便重新测试销售开单功能。

## 🗄️ 涉及的数据表（按依赖关系排序）

### 核心业务表（需要清空）
| 序号 | 表名 | 说明 | 对应用户需求 |
|------|------|------|-------------|
| 1 | `biz_repayment_record` | 还款记录表 | 欠款/还款 |
| 2 | `biz_operation_record` | 操作记录表 | 操作记录 |
| 3 | `biz_customer_archive` | 客户档案表 | 档案 |
| 4 | `biz_package_item` | 套餐明细表 | 开单（子表） |
| 5 | `biz_customer_package` | 客户套餐表 | 开单（关联） |
| 6 | `biz_order_item` | 订单明细表 | 开单记录（子表） |
| 7 | `biz_sales_order` | 销售订单表 | 开单/订单 |
| 8 | `biz_customer` | 客户表 | 顾客基础信息 |

### 表关系说明
```
biz_customer (客户)
├── biz_sales_order (订单) ──┬── biz_order_item (订单明细)
│                             └── biz_customer_package (套餐) ── biz_package_item (套餐明细)
├── biz_operation_record (操作记录)
├── biz_repayment_record (还款记录)
└── biz_customer_archive (客户档案)
```

## 🔧 清空方案

### 方案选择：TRUNCATE（推荐）

**优点：**
- ✅ 速度快，直接删除所有数据并重置自增ID
- ✅ 不会产生大量binlog日志
- ✅ 适合测试环境完整清空

**注意事项：**
- ⚠️ **不可回滚**，执行前务必备份数据
- ⚠️ 需要按照外键依赖顺序执行（先子表后父表）

### 步骤3：按顺序清空所有表（从子表到父表）

#### 第一批：独立业务表（无强依赖）
```sql
-- 1. 清空还款记录表（欠款）
TRUNCATE TABLE `biz_repayment_record`;

-- 2. 清空操作记录表
TRUNCATE TABLE `biz_operation_record`;

-- 3. 清空客户档案表
TRUNCATE TABLE `biz_customer_archive`;
```

#### 第二批：订单相关子表
```sql
-- 4. 清空套餐明细表
TRUNCATE TABLE `biz_package_item`;

-- 5. 清空客户套餐表
TRUNCATE TABLE `biz_customer_package`;

-- 6. 清空订单明细表
TRUNCATE TABLE `biz_order_item`;
```

#### 第三批：主表
```sql
-- 7. 清空销售订单表（开单）
TRUNCATE TABLE `biz_sales_order`;

-- 8. 清空客户表
TRUNCATE TABLE `biz_customer`;
```

## 📊 完整SQL脚本（一键执行版）

```sql
-- =============================================
-- 销售测试数据清空脚本
-- 执行时间：2026年01月09日
-- 注意：此操作不可回滚，请先确认已备份！
-- =============================================

-- 1. 禁用外键检查
SET FOREIGN_KEY_CHECKS = 0;

-- 2. 清空所有业务数据表（按依赖顺序）
TRUNCATE TABLE `biz_repayment_record`;      -- 还款记录（欠款）
TRUNCATE TABLE `biz_operation_record`;      -- 操作记录
TRUNCATE TABLE `biz_customer_archive`;      -- 客户档案
TRUNCATE TABLE `biz_package_item`;          -- 套餐明细
TRUNCATE TABLE `biz_customer_package`;      -- 客户套餐
TRUNCATE TABLE `biz_order_item`;            -- 订单明细
TRUNCATE TABLE `biz_sales_order`;           -- 销售订单（开单）
TRUNCATE TABLE `biz_customer`;              -- 客户表

-- 3. 重新启用外键检查
SET FOREIGN_KEY_CHECKS = 1;

-- 4. 验证清空结果
SELECT '✅ 数据清空完成' AS result;
SELECT
  CONCAT(table_name, ': ', table_rows, ' 行') AS 清空结果
FROM information_schema.tables
WHERE table_schema = 'fuchenpro'
AND table_name IN (
  'biz_customer',
  'biz_sales_order',
  'biz_order_item',
  'biz_customer_package',
  'biz_package_item',
  'biz_operation_record',
  'biz_customer_archive',
  'biz_repayment_record'
);
```

## 💡 补充说明

### 为什么用 TRUNCATE 而不是 DELETE？
1. **性能**：TRUNCATE 比 DELETE 快得多（尤其是大数据量时）
2. **资源占用**：不产生大量事务日志
3. **ID重置**：自动将自增ID重置为1，便于重新测试
4. **原子性**：DDL操作，一次性完成

### 如果只想清空部分数据？
如果不想清空所有客户，只清空特定客户的数据，可以使用带条件的DELETE：
```sql
-- 示例：只清空某个客户的订单数据
DELETE FROM biz_order_item WHERE order_id IN (
  SELECT order_id FROM biz_sales_order WHERE customer_id = [客户ID]
);
DELETE FROM biz_sales_order WHERE customer_id = [客户ID];
-- ... 其他表类似
```
