## 二、技术方案

### 2.1 后端修改 - 补充postName字段

#### 文件：`d:\fuchenpro\webman\app\service\BizScheduleService.php`

**修改位置**：`selectEmployeeSchedule()` 方法（第140-144行）

**当前代码**：
```php
$result[] = [
    'user_id' => $user->user_id,
    'user_name' => $user->user_name,
    'schedules' => $scheduleMap
];
```

**修改为**：
```php
$result[] = [
    'user_id' => $user->user_id,
    'user_name' => $user->user_name,
    'post_name' => $user->post_name ?? '',
    'schedules' => $scheduleMap
];
```

**说明**：
- 从SysUser模型获取post_name字段
- 使用空字符串作为默认值，防止前端显示undefined
- 确保前端日历视图和排班列表都能显示职位信息

## 五、风险点与应对措施

### 5.1 潜在风险
1. **后端字段名不匹配**：如果SysUser模型的字段不是`post_name`，需要确认实际字段名
2. **浏览器兼容性**：flex布局在某些旧版浏览器可能有兼容性问题
3. **性能问题**：大量数据时，绝对定位的计算可能影响性能

### 5.2 应对措施
1. 先检查SysUser模型确认字段名
2. 使用标准的CSS flexbox属性，确保现代浏览器兼容
3. 对于大量数据场景，考虑虚拟滚动或分页加载
