# 修复按钮复选框状态问题

## 问题分析

当多次点击菜单右侧的按钮复选框时，菜单会持续出现被选中/被取消的状态。

**原因**：
1. el-checkbox 使用 `:checked="isMenuChecked(btn.id)"` 单向绑定
2. 点击复选框时，复选框内部状态先切换，然后 `@change` 触发
3. `isMenuChecked` 从 `menuRef.getCheckedKeys()` 获取状态，但这个状态可能因为父子联动等原因发生变化
4. 复选框的视觉状态和实际数据状态不一致

## 解决方案

使用 `v-model` 替代 `:checked` + `@change`，通过计算属性管理按钮权限的选中状态。

### 实施步骤

**文件**: `front/src/views/system/role/index.vue`

#### 步骤1：修改模板，使用 v-model

```html
<el-checkbox
   v-for="btn in getButtonPermissions(data.id)"
   :key="btn.id"
   v-model="buttonCheckState[btn.id]"
   @change="(val) => handleButtonCheck(btn.id, val)"
>{{ btn.label }}</el-checkbox>
```

#### 步骤2：添加响应式状态管理

```javascript
const buttonCheckState = ref({})

// 监听 menuRef 变化，同步 buttonCheckState
watch(() => menuRef.value?.getCheckedKeys(), (keys) => {
  if (keys) {
    const newState = {}
    keys.forEach(id => {
      newState[id] = true
    })
    buttonCheckState.value = newState
  }
}, { deep: true })
```

#### 步骤3：修改 handleButtonCheck 函数

```javascript
function handleButtonCheck(menuId, checked) {
  buttonCheckState.value[menuId] = checked
  menuRef.value.setChecked(menuId, checked, false)
}
```

## 预期效果

点击按钮复选框时，状态正确切换，不会影响菜单的选中状态。
