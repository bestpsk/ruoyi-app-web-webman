# 表格宽度自适应优化 + 有效期管理功能计划

## 问题分析

### 问题1：入库管理表格宽度只有屏幕一半
**原因分析**：
- 表格可能缺少 `width="100%"` 属性
- 或者父容器 `app-container` 的样式限制了宽度

### 问题2：货品管理列宽不均衡
- 品名列使用了 `min-width:150px` 会自动扩展占据剩余空间
- 操作列固定 `120px` 太窄
- 需要改为更合理的固定宽度分配

### 问题3：有效期管理功能
- 美容院产品通常有保质期（如面膜、精华液等）
- 需要能追踪临期和过期商品

---

## 解决方案

### 一、表格宽度自适应修复

#### 1.1 入库管理页面
给 el-table 添加 `style="width: 100%"` 确保撑满容器

#### 1.2 货品管理页面 - 列宽重新分配

| 列名 | 当前 | 修改后 | 说明 |
|------|------|--------|------|
| 选择 | width=50 | width=50 | 保持 |
| 货品编码 | width=130 | **width=120** | 缩小 |
| 品名 | min-width=150 | **width=140** | 改为固定宽度 |
| 供货商 | width=120 | **width=110** | 缩小 |
| 类别 | width=100 | **width=95** | 缩小 |
| 单位(整) | width=80 | width=75 | 微调 |
| 规格(拆) | width=80 | width=75 | 微调 |
| 包装数量 | width=85 | width=85 | 保持 |
| 进货价 | width=85 | width=90 | 加宽 |
| 出货价(整) | width=95 | width=100 | 加宽 |
| 出货价(拆) | width=95 | width=100 | 加宽 |
| 预警数量 | width=80 | width=75 | 缩小 |
| 状态 | width=70 | width=70 | 保持 |
| **操作** | **width=120** | **width=150** | **加宽！** |

## 二、货品有效期管理功能规划

### 2.1 数据库设计

```sql
-- 1. 货品表添加有效期相关字段
ALTER TABLE `biz_product` ADD COLUMN `shelf_life_days` int DEFAULT NULL COMMENT '保质期(天)' AFTER `sale_price_spec`;
ALTER TABLE `biz_product` ADD COLUMN `has_expiry` char(1) DEFAULT '0' COMMENT '是否有有效期(0否 1是)' AFTER `shelf_life_days`;

-- 2. 入库明细表添加生产日期和有效期字段
ALTER TABLE `biz_stock_in_item` ADD COLUMN `production_date` date DEFAULT NULL COMMENT '生产日期' AFTER `remark`;
ALTER TABLE `biz_stock_in_item` ADD COLUMN `expiry_date` date DEFAULT NULL COMMENT '有效期至' AFTER `production_date`;

-- 3. 库存表添加最早到期时间字段
ALTER TABLE `biz_inventory` ADD COLUMN `earliest_expiry` date DEFAULT NULL COMMENT '最早批次有效期至' AFTER `quantity`;
```

### 2.2 业务流程

```
┌─────────────────────────────────────────────────────┐
│                    添加货品时                        │
│  ┌──────────┐   ┌──────────────┐                   │
│  │ 设置保质期 │──▶│ 如：365天    │                   │
│  └──────────┘   └──────────────┘                   │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                    入库时                            │
│  ┌──────────┐   ┌──────────────┐   ┌─────────────┐ │
│  │ 生产日期  │ + │ 保质期天数    │ = │ 有效期至     │ │
│  │2024-01-15│   │   365天      │   │2025-01-14   │ │
│  └──────────┘   └──────────────┘   └─────────────┘ │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                  库存盘点时                          │
│  ┌─────────────────────────────────────────────┐   │
│  │ 🟢 正常：距过期 > 30天                      │   │
│  │ 🟡 临期：距过期 ≤ 30天 且 > 0              │   │
│  │ 🔴 过期：已过有效期                        │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### 2.3 前端展示

#### 货品列表新增列
| 列名 | 宽度 | 说明 |
|------|------|------|
| 保质期 | 90px | 显示天数，如"365天" |
| 有效期状态 | 100px | 标签显示：正常/临期/过期 |

#### 库存盘点页面增强
- 新增"有效期预警"筛选条件
- 列表显示最早到期时间
- 临期/过期商品高亮显示

### 2.4 字段说明

**货品表 (biz_product)**:
| 字段 | 类型 | 说明 |
|------|------|------|
| shelf_life_days | int | 保质期天数（如365表示一年） |
| has_expiry | char(1) | 是否需要管理有效期 |

**入库明细表 (biz_stock_in_item)**:
| 字段 | 类型 | 说明 |
|------|------|------|
| production_date | date | 生产日期 |
| expiry_date | date | 有效期至（可自动计算） |

**库存表 (biz_inventory)**:
| 字段 | 类型 | 说明 |
|------|------|------|
| earliest_expiry | date | 该货品所有批次中最早的到期日 |
