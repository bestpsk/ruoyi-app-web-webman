# 销售开单还款功能实施计划（v2）

## 一、需求背景

### 1.1 业务场景
在销售开单过程中，顾客可能不会一次性付清全部款项，而是先支付一部分，剩余部分形成欠款，后续再逐步还款。

### 1.2 核心概念
```
应付金额（订单总额）= 品项1金额 + 品项2金额 + ...
实付金额（顾客实际支付）= 用户输入的金额
欠款金额 = 应付金额 - 实付金额（当 实付 < 应付 时产生）
```

**关键点：**
- ✅ **应付金额**：系统根据选中的品项自动计算，不可修改
- ✅ **实付金额**：用户输入，必须 > 0 且 ≤ 应付金额
- ✅ **欠款金额**：系统自动计算（应付 - 实付），当实付 < 应付时自动产生欠款

```javascript
const actualPaidAmount = ref('') // 实付金额（用户输入）

// 是否显示欠款金额
const showOwedAmount = computed(() => {
  const total = calculateTotalAmount()
  const paid = parseFloat(actualPaidAmount.value) || 0
  return paid > 0 && paid < total
})

// 计算应付金额（基于选中的品项）
function calculateTotalAmount() {
  let total = 0
  packageList.value.forEach(pkg => {
    if (pkg.status === '2') return // 跳过已用完的套餐
    ;(pkg.items || []).forEach(item => {
      if (item.orderCount > 0) {
        const price = item.dealPrice || item.unitPrice || 0
        total += price * item.orderCount
      }
    })
  })
  return total
}

// 计算欠款金额
function calculateOwedAmount() {
  const total = calculateTotalAmount()
  const paid = parseFloat(actualPaidAmount.value) || 0
  return Math.max(0, total - paid)
}

// 实付金额输入事件处理
function onActualPaidInput(e) {
  let value = e.detail.value
  const total = calculateTotalAmount()

  // 移除非数字字符（保留小数点）
  value = value.replace(/[^\d.]/g, '')

  // 验证不能超过应付金额
  const numValue = parseFloat(value) || 0
  if (numValue > total) {
    value = String(total)
    uni.showToast({
      title: '实付金额不能超过应付金额',
      icon: 'none'
    })
  }

  actualPaidAmount.value = value
}

// 设置全额支付快捷操作
function setFullPayment() {
  actualPaidAmount.value = String(calculateTotalAmount())
}
```

```javascript
async function submitOrder() {
  // 构建订单明细
  const orderItems = []
  packageList.value.forEach(pkg => {
    if (pkg.status === '2') return
    ;(pkg.items || []).forEach(item => {
      if (item.orderCount > 0) {
        orderItems.push({
          packageItemId: item.packageItemId,
          productName: item.productName,
          quantity: item.orderCount,
          planPrice: item.unitPrice || 0,
          dealPrice: item.unitPrice || 0,
          isDeal: '1'
        })
      }
    })
  })

  if (orderItems.length === 0) {
    uni.showToast({ title: '请选择开单项目', icon: 'none' })
    return
  }

  // 验证实付金额
  const totalAmount = calculateTotalAmount()
  const paidAmount = parseFloat(actualPaidAmount.value) || 0

  if (paidAmount <= 0) {
    uni.showToast({ title: '请输入实付金额', icon: 'none' })
    return
  }

  if (paidAmount > totalAmount) {
    uni.showToast({ title: '实付金额不能超过应付金额', icon: 'none' })
    return
  }

  submitting.value = true
  try {
    const currentPackageName =
      packageList.value.length > 0 ? packageList.value[0].packageName : ''
    const owedAmount = totalAmount - paidAmount

    await addSalesOrder({
      customerId: customerId.value,
      customerName: customerInfo.value?.customerName || '',
      storeId: storeId.value,
      packageName: currentPackageName,
      items: orderItems,
      remark: orderRemark.value,
      // 新增字段
      totalAmount: totalAmount, // 应付金额
      paidAmount: paidAmount, // 实付金额
      owedAmount: owedAmount, // 欠款金额
      paymentMethod: selectedPaymentMethod.value // 支付方式
    })

    // 根据是否有欠款显示不同的提示
    if (owedAmount > 0) {
      uni.showToast({
        title: `开单成功，欠款¥${owedAmount.toFixed(2)}`,
        icon: 'success',
        duration: 2000
      })
    } else {
      uni.showToast({ title: '开单成功', icon: 'success' })
    }

    // 重置表单
    orderRemark.value = ''
    actualPaidAmount.value = ''
    selectedPaymentMethod.value = 'cash'

    // 刷新数据
    loadPackages()
  } catch (e) {
    console.error('开单失败:', e)
    uni.showToast({ title: '开单失败: ' + (e.message || '未知错误'), icon: 'none' })
  } finally {
    submitting.value = false
  }
}
```

```php
        $package->owed_amount += $owedAmount;
```

### 4.5 还款业务流程

#### 详细步骤：

```
步骤1: 用户点击"还款"按钮
       ↓

       ↓
步骤3: 用户填写信息并点击"确认还款"
       ↓
步骤4: 前端校验
       ✓ 还款金额 > 0
       ✓ 还款金额 ≤ 当前欠款金额
       ✓ 已选择支付方式
       ↓
步骤5: 调用后端API: POST /business/repayment/add
       请求数据:
       {
         customerId: 1,
         customerName: "客户1",
         packageId: 1,
         packageNo: "PK202605090001",
         packageName: "套餐卡1",
         orderId: 1,
         orderNo: "SO202605090001",
         repaymentAmount: 1000,      // 本次还款金额
         repaymentType: "1",          // 1=套餐还款
         paymentMethod: "cash",       // 支付方式
         remark: "",                  // 备注
         enterpriseId: 2,
         enterpriseName: "逆龄奢",
         storeId: 3,
         storeName: "宜川店"
       }
       ↓
步骤6: 后端处理（参见 BizRepaymentService.php 第71-100行）
       6a. 生成还款编号: RP + 日期 + 序号
       6b. 开始数据库事务
       6c. 创建还款订单（biz_sales_order）
           • order_status = '3' (还款订单)
           • paid_amount = 还款金额
       6d. 创建还款记录（biz_repayment_record）
           • status = '0' (待审核)
       6e. 提交事务
       6f. 写入客户档案（异步）
       ↓
步骤7: 返回成功响应
       ↓
步骤8: 前端处理
       • 显示"还款成功"提示
       • 关闭还款弹窗
       • 刷新欠款列表
       • 刷新套餐列表（更新欠款金额）
       ↓
步骤9: (可选) 审核流程
       如果开启审核模式：
       9a. 还款记录状态为"待审核"
       9b. 管理员审核通过后：
           • 更新还款记录状态为"已审核"
           • 更新套餐欠款金额（减少）
           • 更新套餐实付金额（增加）
       9c. 如果审核拒绝：
           • 更新还款记录状态为"已取消"
           • 取消关联的还款订单
```

## 七、风险评估与应对策略

### 7.1 技术风险

| 风险项 | 可能性 | 影响 | 应对策略 |
|-------|-------|------|---------|
| 数据不一致（订单/套餐/明细金额不匹配） | 中 | 高 | 使用数据库事务 + 完善的单元测试 |
| 并发还款导致超还 | 低 | 高 | 使用乐观锁 + 金额二次校验 |
| 前后端金额精度丢失 | 低 | 中 | 统一使用Decimal类型，避免浮点运算 |
| 历史数据兼容性问题 | 低 | 中 | 新增字段设置合理默认值 |

### 7.2 业务风险

| 风险项 | 可能性 | 影响 | 应对策略 |
|-------|-------|------|---------|
| 用户误操作导致错误欠款 | 中 | 中 | 添加确认提示 + 操作日志可追溯 |
| 还款审核流程复杂影响效率 | 低 | 中 | 可配置是否开启审核模式 |
| 大额欠款坏账风险 | 低 | 高 | 添加欠款预警机制 + 定期催收提醒 |

### 7.3 应对措施详解

**数据一致性保障：**
```php
// 使用数据库事务确保原子性
Db::beginTransaction();
try {
    // 1. 创建订单
    // 2. 创建订单明细
    // 3. 创建/更新套餐
    // 4. 创建/更新套餐明细
    Db::commit();
} catch (\Exception $e) {
    Db::rollBack(); // 任一步骤失败全部回滚
    throw $e;
}
```

**并发安全控制：**
```php
// 使用乐观锁防止并发问题
$affected = BizCustomerPackage::where('package_id', $packageId)
    ->where('owed_amount', $currentOwedAmount) // 版本号检查
    ->update(['owed_amount' => $newOwedAmount]);

if (!$affected) {
    throw new \Exception('数据已被其他操作修改，请刷新后重试');
}
```

**金额精度控制：**
```php
// PHP端统一使用 bcmath 函数
$totalAmount = bcmul($unitPrice, $quantity, 2); // 保留2位小数
$owedAmount = bcsub($totalAmount, $paidAmount, 2);
```
