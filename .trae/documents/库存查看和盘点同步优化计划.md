# 库存查看和库存盘点同步优化计划

## 当前问题分析

### 库存查看（inventory/index.vue）
1. **规格列**：显示原始字典值（如 `1`），没有转换为标签（如"支"）
2. **单位列**：显示原始字典值（如 `1`），没有转换为标签（如"箱"）
3. **当前库存**：只显示纯数字（如 `10`），没有单位，应改为 `1箱（10支）` 格式
4. **预警数量**：只显示纯数字，没有单位
5. **缺少换算列**：没有显示换算关系
6. **缺少字典**：没有引入 `biz_product_unit` 和 `biz_product_spec` 字典
7. **数据来源**：通过 `product` 关联获取货品信息，`product` 对象中有 `unit`、`spec`、`pack_qty` 字段

### 库存盘点（stockCheck/index.vue）
1. **列表页**：盘点总数量和差异数量只显示纯数字，没有单位
2. **弹窗明细**：
   - 规格列显示原始值，没有转换
   - 单位列显示原始值，没有转换
   - 系统库存、实际数量、差异数量都没有单位
   - 缺少换算列
3. **缺少字典**：没有引入 `biz_product_unit` 和 `biz_product_spec` 字典
4. **后端 loadInventoryData**：返回的 items 中有 `spec`、`unit` 字段，但没有 `pack_qty`

## 数据结构分析

### 库存查看
- API 返回 `rows` 中每条数据有 `product` 关联对象
- `product` 对象包含：`unit`（主单位字典值）、`spec`（规格字典值）、`pack_qty`（换算数量）
- `quantity` 是最小单位数量

### 库存盘点
- 列表 API 返回 `rows` 中有 `totalQuantity`、`totalDiffQuantity`
- 详情 API 返回 `items` 中每条有 `spec`、`unit`、`system_quantity`、`actual_quantity`、`diff_quantity`
- **缺少 `pack_qty`**：后端 `loadInventoryData` 方法没有返回 `pack_qty`

## 优化方案

### 一、库存查看优化

#### 1.1 前端修改 - inventory/index.vue

**引入字典**：
```javascript
const { biz_product_category, biz_product_unit, biz_product_spec } = useDict("biz_product_category", "biz_product_unit", "biz_product_spec")
```

**添加方法**：
```javascript
function getUnitLabel(value) { ... }
function getSpecLabel(value) { ... }
function formatInventoryQty(row) {
  const packQty = row.product?.pack_qty || 1
  const unitLabel = getUnitLabel(row.product?.unit)
  const specLabel = getSpecLabel(row.product?.spec)
  const qty = row.quantity || 0
  if (packQty > 1 && specLabel) {
    const mainQty = qty / packQty
    const mainQtyStr = Number.isInteger(mainQty) ? mainQty : mainQty.toFixed(1).replace(/\.0$/, '')
    return mainQtyStr + unitLabel + '（' + qty + specLabel + '）'
  } else if (unitLabel) {
    return qty + unitLabel
  } else {
    return qty
  }
}
```

**表格列调整**：

| 列名 | 旧 | 新 |
|------|-----|-----|
| 规格 | `scope.row.product?.spec`（原始值） | `getSpecLabel(scope.row.product?.spec)`（标签） |
| 单位 | `scope.row.product?.unit`（原始值） | 改为"换算"列，显示 `1箱=10支` |
| 当前库存 | `10` | `1箱（10支）` |
| 预警数量 | `5` | `5支`（带副单位） |

**新列结构**：货品编码 | 品名 | 类别 | 换算 | 当前库存 | 预警数量 | 库存状态 | 进货价 | 出货价 | 最后入库 | 最后出库

#### 1.2 后端无需修改
库存查看已通过 `with('product')` 关联获取了 product 的所有字段（包括 `pack_qty`、`unit`、`spec`），前端可以直接使用。

### 二、库存盘点优化

#### 2.1 后端修改 - BizStockCheckService.php

**loadInventoryData 方法**：添加 `pack_qty` 字段
```php
$items[] = [
    'product_id' => $product->product_id,
    'product_name' => $product->product_name,
    'spec' => $product->spec,
    'unit' => $product->unit,
    'pack_qty' => $product->pack_qty ?? 1,  // 新增
    'system_quantity' => $inventory ? $inventory->quantity : 0,
    'actual_quantity' => $inventory ? $inventory->quantity : 0,
    'diff_quantity' => 0,
];
```

**selectStockCheckById 方法**：items 中也需要添加 `pack_qty`
```php
$stockCheck->items = BizStockCheckItem::where('stock_check_id', $stockCheckId)->get();
```
需要确认 `biz_stock_check_item` 表是否有 `pack_qty` 字段。如果没有，需要从 product 表获取。

#### 2.2 前端修改 - stockCheck/index.vue

**引入字典**：
```javascript
const { biz_doc_status, biz_product_unit, biz_product_spec } = useDict("biz_doc_status", "biz_product_unit", "biz_product_spec")
```

**列表页**：
- 盘点总数量和差异数量由于涉及多种货品，保持纯数字即可（不同货品单位不同，无法统一格式化）

**弹窗明细表格调整**：

| 列名 | 旧 | 新 |
|------|-----|-----|
| 规格 | `spec`（原始值） | `getSpecLabel(spec)`（标签） |
| 单位 | `unit`（原始值） | 改为"换算"列 |
| 系统库存 | `10` | `1箱（10支）` |
| 实际数量 | `10` | `1箱（10支）` |
| 差异数量 | `+5` / `-3` | `+0.5箱（5支）` / `-0.3箱（3支）` |

**新列结构**：品名 | 规格 | 换算 | 系统库存 | 实际数量 | 差异数量 | 备注

## 实施步骤

### 步骤1：修改库存查看前端 - inventory/index.vue
1. 引入 `biz_product_unit`、`biz_product_spec` 字典
2. 添加 `getUnitLabel`、`getSpecLabel`、`formatInventoryQty` 方法
3. 修改表格列：规格→标签、单位→换算、当前库存→带单位、预警数量→带单位
4. 表格宽度调整，使用 min-width 自适应

### 步骤2：修改库存盘点后端 - BizStockCheckService.php
1. `loadInventoryData` 方法添加 `pack_qty` 字段
2. `selectStockCheckById` 方法中为 items 补充 `pack_qty`（从 product 表获取）

### 步骤3：修改库存盘点前端 - stockCheck/index.vue
1. 引入 `biz_product_unit`、`biz_product_spec` 字典
2. 添加 `getUnitLabel`、`getSpecLabel`、`formatItemQuantity` 方法
3. 修改弹窗明细表格列结构
4. 系统库存、实际数量、差异数量都带单位显示
