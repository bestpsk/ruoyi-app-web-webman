# 操作档案问题根因修复方案

## 🎯 根因确认

### Bug：`update_time` 列不存在

**数据库表结构**（`biz_customer_archive`）：
```
archive_id, customer_id, customer_name, enterprise_id, enterprise_name,
store_id, store_name, archive_date, archive_type, source_type, source_id,
plan_items, amount, satisfaction, photos, customer_feedback,
operator_user_id, operator_user_name, remark, create_by, create_time
```

**⚠️ 没有 `update_time` 列！**

但在合并代码中写了：
```php
$existing->update_time = date('Y-m-d H:i:s');  // ❌ 列不存在，导致SQL报错
$existing->save();  // 抛出异常，被try-catch静默捕获
```

### 错误链路

```
record_id=3 提交操作
  → insertRecord() 创建操作记录 ✅
  → insertArchiveFromOperation($record)
    → 查找已有档案 → 找到 archive_id=2 ✅
    → 进入合并逻辑
    → $existing->update_time = ... ❌ 列不存在
    → $existing->save() ❌ SQL报错
    → 被try-catch捕获，只写日志
  → 返回 $record（操作记录），前端以为成功了
  → 但档案没有生成/合并 ❌
```

### 修改2：BizOperationRecordService.php 第77-82行

增强错误日志：
```php
$record = BizOperationRecord::create($data);

try {
    $archiveService = new BizCustomerArchiveService();
    $result = $archiveService->insertArchiveFromOperation($record);
    if (!$result) {
        \support\Log::warning('操作档案返回null', [
            'record_id' => $record->record_id,
            'customer_id' => $record->customer_id ?? 'NULL'
        ]);
    }
} catch (\Exception $e) {
    \support\Log::error('写入操作档案失败: ' . $e->getMessage(), [
        'record_id' => $record->record_id ?? 'unknown',
        'file' => $e->getFile(),
        'line' => $e->getLine()
    ]);
}
```

### 修改3：清理历史重复档案 + 重新生成

```sql
-- 删除旧的非合并操作档案
DELETE FROM biz_customer_archive WHERE source_type = '1';
```

然后通过脚本或手动为现有操作记录重新生成合并后的档案。
