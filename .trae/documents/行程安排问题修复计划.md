# 行程安排功能问题修复计划

## 一、问题分析

根据用户截图和反馈，存在以下4个问题：

### 问题1：企业名不显示
**现象**：员工行程/企业排班的排班列表中，"企业"列显示为空
**原因分析**：
- BizSchedule模型有`enterprise_name`字段
- `listSchedule` API返回的数据中`enterpriseName`可能为空
- 可能是添加行程时没有正确保存企业名称，或数据库数据缺失

### 问题2：员工配置--职位不显示
**现象**：员工配置tab页的"职位"列全部为空
**根因**：查看 [biz_schedule_upgrade.sql](file:///d:/fuchenpro/webman/sql/biz_schedule_upgrade.sql#L48-L61) 第48-61行，初始化员工配置数据时：
```sql
INSERT INTO `biz_employee_config` (`user_id`, `user_name`, `dept_id`, `dept_name`, ...)
SELECT u.user_id, u.user_name, u.dept_id, d.dept_name, ...
```
**只同步了user_name和dept_name，没有同步post_id和post_name！**

### 问题3：列表列宽不均匀
**现象**：排班列表各列宽度差异大，有的很宽有的很窄
**原因**：el-table-column使用了混合的width和min-width设置

### 问题4：休息日选中颜色区分不明显
**现象**：配置休息日弹窗中选中的日期颜色不够明显

## 二、修复方案

### 修复1：后端 - 员工配置查询关联SysUser获取职位 ⏱️ 5分钟

**文件**：[BizEmployeeConfigService.php](file:///d:/fuchenpro/webman/app/service/BizEmployeeConfigService.php)

**修改`selectConfigList`方法**：关联SysUser表补充post_name字段

```php
public function selectConfigList($params = [])
{
    $query = BizEmployeeConfig::query();
    
    // ... 现有筛选条件 ...
    
    $result = $query->orderBy('config_id', 'desc')->paginate($pageSize, ['*'], 'page', $pageNum);
    
    // 关联SysUser补充职位信息
    foreach ($result->items() as $item) {
        if (empty($item->post_name)) {
            $user = SysUser::find($item->user_id);
            if ($user) {
                $item->post_name = $user->post_name ?? '';
                $item->post_id = $user->post_id ?? null;
            }
        }
    }
    
    return $result;
}
```

### 修复2：后端 - 补充SQL脚本同步职位数据 ⏱️ 3分钟

**新建或追加到** biz_schedule_upgrade.sql：

```sql
-- 同步员工职位信息到配置表
UPDATE `biz_employee_config` c
INNER JOIN sys_user u ON c.user_id = u.user_id
SET c.post_id = u.post_id,
    c.post_name = u.post_name
WHERE c.post_name IS NULL OR c.post_name = '';
```
