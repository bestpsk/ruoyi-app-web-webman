# 行程日期选择功能优化方案

## 🚨 问题分析

### 问题1：选择日期后显示 "undefined-undefined-undefined"

#### 现象
用户在日历选择器中选择日期范围后，表单输入框显示：`undefined-undefined-undefined`

#### 根因定位

**文件位置**：[form.vue#L199-L204](file:///d:/fuchenpro/AppV3/src/pages/business/schedule/form.vue#L199-L204)

```javascript
function onCalendarConfirm(e) {
  const [startDate, endDate] = e  // ❌ 问题所在！
  form.startDate = `${startDate.year}-${String(startDate.month).padStart(2, '0')}-${String(startDate.day).padStart(2, '0')}`
  form.endDate = `${endDate.year}-${String(endDate.month).padStart(2, '0')}-${String(endDate.day).padStart(2, '0')}`
  showCalendarPicker.value = false
}
```

**问题分析**：
1. **u-calendar 组件的 confirm 事件参数格式**
   - 当前代码假设 `e` 是一个数组 `[startDate, endDate]`
   - 但实际上 **u-calendar 的 range 模式** 返回的可能是：
     - 数组形式：`[{year, month, day}, {year, month, day}]`
     - 或者对象形式：`{ startDate: {...}, endDate: {...} }`
     - 或者直接是日期对象数组

2. **解构失败的原因**
   - 如果 `e` 不是预期的数组格式，解构会得到 `undefined`
   - 访问 `undefined.year` → `undefined`
   - 最终拼接出 `"undefined-undefined-undefined"`

3. **正确的处理方式应该先打印 e 的值确认数据结构**

### 问题2：日期选择模式优化（范围选择 → 多日期灵活选择）

#### 当前实现
- 使用 **range 模式**（范围选择）
- 用户必须选择"开始日期"和"结束日期"
- 系统自动填充中间所有日期

#### 用户期望
- **灵活多选**：可以自由选择多个离散的日期（不一定连续）
- 例如：选择5月2日、5月15日、6月1日（非连续日期）
- 更符合实际业务场景（如一周去某企业3次）

#### 技术可行性分析

**方案A：使用 multi 模式（推荐）⭐⭐⭐⭐⭐**

u-calendar 支持 `mode="multi"` 模式：
- 允许用户点击多个日期
- 返回选中的日期数组
- 更灵活，用户体验更好

**方案B：保持 range + 增加多次添加 ⭐⭐⭐**

保持当前的范围选择模式，但允许用户：
- 第一次选择一个范围
- 可以再次打开日历添加更多日期
- 最终提交时合并所有日期

**方案C：自定义日期选择器 ⭐⭐**

完全自定义日历组件，支持更复杂的选择逻辑

```javascript
function onMultiDateConfirm(e) {
```

```javascript
const dateRangeText = computed(() => {
  if (!form.selectedDates || form.selectedDates.length === 0) {
    return ''
```

```javascript
async function submitForm() {
  if (!form.userName) { uni.showToast({ title: '请选择员工', icon: 'none' }); return }
  if (!form.enterpriseId) { uni.showToast({ title: '请选择企业', icon: 'none' }); return }
  if (!form.selectedDates || form.selectedDates.length === 0) { uni.showToast({ title: '请选择至少一个日期', icon: 'none' }); return }
  if (!form.purpose) { uni.showToast({ title: '请选择下店目的', icon: 'none' }); return }

  // 检查日期冲突
  const conflictDates = form.selectedDates.filter(date => bookedDates.value.includes(date))
  if (conflictDates.length > 0) {
    uni.showModal({ title: '日期冲突', content: `以下日期已有安排：${conflictDates.join('、')}`, showCancel: false })
    return
  }

  submitting.value = true
  try {
    const scheduleList = form.selectedDates.map(scheduleDate => ({
      userId: form.userId,
      userName: form.userName,
      enterpriseId: form.enterpriseId,
      enterpriseName: form.enterpriseName,
      scheduleDate,
      purpose: form.purpose,
      status: form.status,
      remark: form.remark
    }))

    await addScheduleBatch(scheduleList)
    uni.showToast({ title: `新增成功（共${scheduleList.length}天）`, icon: 'success' })
    setTimeout(() => goBack(), 1500)
  } catch (e) {
    console.error('提交失败:', e)
    const msg = e?.msg || e?.message || '操作失败，请重试'
    uni.showToast({ title: msg, icon: 'none', duration: 2000 })
  } finally { submitting.value = false }
}
```

**改进点**：
- ✅ 直接使用 `selectedDates` 数组生成提交列表
- ✅ 不再依赖开始/结束日期循环
- ✅ 支持离散日期（如5月2日、5月15日、6月1日）
- ✅ 冲突检测更精确

## ⚠️ 注意事项

### 1. 向后兼容性

- 保留 `startDate` 和 `endDate` 字段用于兼容
- 后端API可能需要适配新的数据格式
- 编辑历史数据时需要特殊处理

### 2. 性能考虑

- 多选模式下，选择的日期数量不应过多（建议≤31天）
- 冲突检测需要对每个选中日期进行检查
- 可以添加最大选择数量限制

### 3. 用户体验

- 日历上清晰标识已选中的日期（高亮/打勾）
- 提供"清空重新选择"选项
- 已安排的日期仍然标记为不可选（disabled）

### 4. 数据验证

- 至少选择1个日期
- 不能选择过去的日期（可选）
- 不能选择超过1年的范围（可选）
