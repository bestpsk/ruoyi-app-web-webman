# 销售开单 - 操作记录同步到档案问题修复

## 🔍 问题分析

用户报告：顾客在操作以后，操作记录中存在记录了，但是档案中没有记录。

### 原因分析

在 `BizOperationRecordService::insertRecord` 方法中，异常被静默捕获：

```php
try {
    $archiveService = new BizCustomerArchiveService();
    $archiveService->insertArchiveFromOperation($record);
} catch (\Exception $e) {}  // 异常被静默吞掉，无法知道真正的问题
```

这导致如果写入档案时发生错误，我们无法知道具体原因。可能的问题包括：
1. 数据库字段缺失或不匹配
2. 某些必填字段为 null
3. JSON 编码失败

---

## 🛠️ 修复方案

### 方案：记录异常日志

修改 `BizOperationRecordService::insertRecord`，将异常记录到日志文件中，方便排查问题：

```php
try {
    $archiveService = new BizCustomerArchiveService();
    $archiveService->insertArchiveFromOperation($record);
} catch (\Exception $e) {
    \support\Log::error('写入操作档案失败: ' . $e->getMessage(), ['trace' => $e->getTraceAsString()]);
}
```

同时检查 `BizCustomerArchiveService::insertArchiveFromOperation` 方法，确保所有字段正确处理。

---

## 📝 修改文件清单

| 文件 | 修改内容 |
|------|----------|
| `BizOperationRecordService.php` | 记录异常日志 |
| `BizCustomerArchiveService.php` | 检查并确保字段正确 |

---

## ✅ 预期效果

1. 操作记录创建后能正确同步到档案表
2. 如果发生错误，会在日志文件中记录具体原因
3. 方便后续排查问题
