# 新增出库单报错修复 - 添加缺失的数据库字段

## 问题描述

**错误信息**: 
```
SQLSTATE[42S22]: Column not found: 1054 Unknown column 'out_target_type' in 'field list'
```

**触发场景**: 新增出库单时点击"确定"

**根本原因**: 数据库表 `biz_stock_out` 中缺少 `out_target_type` 字段

---

## 问题分析

### 错误SQL
```sql
INSERT INTO `biz_stock_out` (
  `stock_out_type`, `out_target_type`,  -- ❌ out_target_type 不存在
  `enterprise_id`, `enterprise_name`, ...
) VALUES (1, 1, 1, '馥田诗', ...)
```

### 原因
之前我们在代码中添加了 `out_target_type` 字段支持（对象类型：企业出库/员工领用），但**没有在数据库中创建对应的列**。

---

## 解决方案

创建 SQL 迁移脚本，向 `biz_stock_out` 表添加缺失的字段。

---

## 详细修改步骤

### 步骤1: 创建 SQL 迁移脚本

**文件**: `d:\fuchenpro\webman\sql\add_out_target_type.sql`

```sql
-- 为出库单表添加对象类型字段
-- 用于区分：企业出库(1) / 员工领用(2)

ALTER TABLE `biz_stock_out`
ADD COLUMN `out_target_type` VARCHAR(1) NOT NULL DEFAULT '1' COMMENT '出库对象类型（1-企业出库 2-员工领用）'
AFTER `stock_out_type`;
```

**字段说明**:
- **字段名**: `out_target_type`
- **类型**: `VARCHAR(1)`
- **默认值**: `'1'`（企业出库）
- **注释**: 出库对象类型（1-企业出库 2-员工领用）

---

### 步骤2: 执行 SQL 脚本
