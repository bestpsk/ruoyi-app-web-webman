## 🔧 解决方案（按顺序尝试）

### 方案1：强制刷新浏览器（推荐首选）

**操作步骤：**
1. 在出错的页面按 **`Ctrl + Shift + R`** （Windows/Linux）
   或 **`Cmd + Shift + R`** （Mac）
2. 或者按 **`Ctrl + F5`**
3. 这会绕过缓存，重新加载最新代码

**原理：** 浏览器可能缓存了旧版本的 JavaScript 文件（包含重复变量声明的错误版本）

---

### 方案2：清除浏览器缓存并重启开发服务器

#### 步骤1：清除浏览器缓存
```
Chrome/Edge:
1. 按 F12 打开开发者工具
2. 右键点击浏览器的刷新按钮

# 重新启动:
npm run dev
# 或

# 或
pnpm dev

#### 步骤3：重新访问页面
```
1. 等待开发服务器完全启动（看到 Local: http://localhost:xxxx）

如果上述方案无效，请执行以下操作获取更多信息：

1. **打开浏览器开发者工具**

3. **重新点击"开单"按钮触发错误**

4. **复制完整的错误信息**，包括：

   - 错误消息（Failed to fetch dynamically imported module）
   - 堆栈跟踪（Stack Trace）
   - 任何其他红色错误信息

5. **特别关注以下信息：**
   ```
   ❌ 如果看到类似这样的错误：
   "SyntaxError: Identifier has already been declared"
   → 说明确实还有重复声明，需要进一步检查

   ❌ 如果看到：
   "Cannot find module"
   → 可能是依赖缺失或路径问题

   ❌ 如果看到：
   "Unexpected token"
   → 可能有语法错误（括号、引号等）
   ```

## 💡 常见问题 FAQ

### Q1: 为什么会报 TypeError？
**A:** 这是浏览器缓存的旧版代码导致的。旧代码中有重复的变量声明，虽然我们已经修复了，但浏览器还在使用缓存。

### Q2: 清除缓存后还是报错怎么办？
**A:** 请尝试：
1. 完全关闭浏览器后重新打开
2. 使用无痕/隐私模式打开
3. 更换浏览器测试（Chrome/Firefox/Edge）
4. 重启电脑（最后手段）

### Q3: 还款功能在哪里？找不到"还欠款"Tab？
**A:** 
- 确保您已经进入了"销售开单"页面（点击客户的"开单"按钮后）
- 在页面**顶部**寻找Tab标签栏（不是底部）
- Tab顺序：[开单] [开单记录] [操作记录] [还欠款]
- 点击第4个"还欠款"即可

### Q4: 为什么没有欠款数据？
**A:** 
- 该客户可能没有产生过欠款订单
- 需要先通过"部分付款"开单产生欠款
- 或者手动测试：创建一个部分付款的订单

### Q5: 开单和还款使用相同的支付方式变量会有冲突吗？
**A:** 不会！这是合理的设计：
- `selectedPaymentMethod` 是共享变量
- 开单时设置 → 用于记录本次开单的支付方式
- 还款时设置 → 用于记录本次还款的支付方式
- 两者不会同时使用，所以不会冲突
