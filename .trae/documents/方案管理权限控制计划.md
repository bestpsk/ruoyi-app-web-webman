# 方案管理权限控制和操作记录计划

## 当前状态分析

### 已有字段
- `create_by` - 创建人
- `create_time` - 创建时间
- `update_by` - 更新人
- `update_time` - 更新时间
- `audit_by` - 审核人
- `audit_time` - 审核时间
- `audit_remark` - 审核备注

### 需要新增的字段
1. `submit_by` - 提交审核人
2. `submit_time` - 提交审核时间
3. `status_change_by` - 状态变更人（启用/停用）
4. `status_change_time` - 状态变更时间

---

## 重要操作权限控制

| 操作 | 权限标识 | 说明 |
|------|---------|------|
| 新增方案 | `business:plan:add` | 创建新方案 |
| 编辑方案 | `business:plan:edit` | 编辑草稿/已驳回的方案 |
| 删除方案 | `business:plan:remove` | 删除草稿/已驳回的方案 |
| 提交审核 | `business:plan:submitAudit` | 提交方案审核 |
| 审核通过 | `business:plan:audit` | 审核通过方案 |
| 审核驳回 | `business:plan:audit` | 驳回方案 |
| 启用/停用 | `business:plan:changeStatus` | 切换方案启用状态 |
| 新建出货单 | `business:shipment:add` | 从方案创建出货单 |

---

## 实施步骤

### 步骤1：修改数据库表结构
添加新字段到 `biz_plan` 表：
```sql
ALTER TABLE biz_plan 
ADD COLUMN submit_by VARCHAR(64) DEFAULT NULL COMMENT '提交审核人' AFTER audit_remark,
ADD COLUMN submit_time DATETIME DEFAULT NULL COMMENT '提交审核时间' AFTER submit_by,
ADD COLUMN status_change_by VARCHAR(64) DEFAULT NULL COMMENT '状态变更人' AFTER submit_time,
ADD COLUMN status_change_time DATETIME DEFAULT NULL COMMENT '状态变更时间' AFTER status_change_by;
```

### 步骤2：修改后端模型
**文件**: `d:\fuchenpro\webman\app\model\BizPlan.php`

在 `$fillable` 中添加新字段。

### 步骤3：修改后端服务
**文件**: `d:\fuchenpro\webman\app\service\BizPlanService.php`

- `submitAudit` 方法：记录提交人和提交时间
- `changeStatus` 方法：记录状态变更人和时间

### 步骤4：修改前端页面
**文件**: `d:\fuchenpro\front\src\views\business\plan\index.vue`

1. 为操作按钮添加权限指令 `v-hasPermi`
2. 在方案详情中显示操作人信息

### 步骤5：确保菜单权限已配置
检查数据库中方案管理的按钮权限是否完整。
