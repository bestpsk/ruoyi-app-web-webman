# 查看出库单领用员工/目标企业显示数字修复

  NULL, NULL,             -- responsible_id=NULL, responsible_name=NULL

## 问题根因

查看 [stockOut/index.vue](file:///d:/fuchenpro/front/src/views/wms/stockOut/index.vue) 第209-211行：

```html
<!-- 员工领用(outTargetType=2)时显示 -->
<div class="info-item" v-if="form.outTargetType === '1'">
  <span>出库企业：</span>{{ form.enterpriseName || '-' }}
</div>
<div class="info-item" v-else>
  <span>领用员工：</span>{{ form.responsibleName || '-' }}   <!-- ← 用 responsibleName -->
</div>
<div class="info-item">
  <span>对接员工：</span>{{ form.contactEmployeeName || '-' }}
</div>
```

**关键发现：**
- 企业出库(`outTargetType=1`)：使用 `form.enterpriseName` → 显示 **2**（数字）
- 员工领用(`outTargetType=2`)：使用 `form.responsibleName` → 显示 **100**（数字）

但数据库中：
- `enterprise_name` = `'逆龄奢'`
- `responsible_name` = `NULL`

**结论：后端返回的数据字段值不对，或者前端 response.data 包装问题**

---

## 解决方案

### 步骤1：检查API响应包装

用户规则要求：**前端需要 response.data**

查看第437行：
```javascript
getStockOut(row.stockOutId).then(response => {
  form.value = response;  // ← 可能应该是 response.data
})
```

如果 API 返回 `{ code: 200, data: {...}, msg: '' }`，则 `response` 是整个对象，`response.data` 才是实际数据。

### 步骤2：修复后端直接返回数据库值

确保 [BizStockOutService.php](file:///d:/fuchenpro/webman/app/service/BizStockOutService.php) 的 `selectStockOutById()` 直接返回：

```php
return [
    ...
    'enterpriseName' => $stockOutArray['enterprise_name'] ?? '-',
    'contactEmployeeName' => $stockOutArray['contact_employee_name'] ?? '-',
    'responsibleName' => $stockOutArray['responsible_name'] ?? '-',
    ...
];
```

### 步骤3：（如需要）重启后端服务

代码修改后需要重启 webman 服务才能生效。
