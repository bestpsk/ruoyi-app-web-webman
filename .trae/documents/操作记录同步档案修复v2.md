# 操作记录同步档案问题 - 彻底修复计划

## 🔍 问题现状

从用户截图分析：
- **操作记录Tab**：客户"李玲地方"有1条操作记录（11111, 2026-05-08, 持卡）
- **档案Tab**：只显示"开单"类型记录（4条），**没有"操作"类型**
- **开单记录Tab**：有订单记录

## 🎯 根本原因

后端服务可能**未重启**，导致之前修改的代码未生效：
1. `BizOperationRecordService::insertRecord` 中的 `insertArchiveFromOperation` 调用
2. 路由 `/business/archive/list` 可能未注册

---

## 🛠️ 修复步骤

### 第1步：确认后端服务已重启
- 确认 webman 服务已使用最新代码重启
- 如果没有重启，需要执行 `php windows.php restart`

### 第2步：验证路由是否生效
检查 `/business/archive/list` 路由是否能正常访问

### 第3步：手动同步缺失的操作记录到档案
编写脚本，将所有未被同步的操作记录写入档案表：
```sql
-- 查找所有在 biz_operation_record 中但不在 biz_customer_archive (source_type=1) 的记录
SELECT r.* FROM biz_operation_record r 
LEFT JOIN biz_customer_archive a ON a.source_type='1' AND a.source_id = r.record_id 
WHERE a.archive_id IS NULL AND r.customer_id IS NOT NULL
```

### 第4步：测试新增操作时自动同步
- 新增一条操作记录
- 验证档案表中是否自动出现该记录

### 第5步：前端确认
- 刷新页面
- 点击有操作记录的客户
- 切换到"档案"Tab，确认显示操作类型记录
