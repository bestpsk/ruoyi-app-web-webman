# 修复入库明细表格列宽挤压问题

## 问题分析

### 当前问题（图2显示）
- **现象**：新增入库单时，所有列被严重挤压在一起
- **原因**：使用了纯百分比 `width="16%"` 等固定百分比
- **结果**：在弹窗空间有限时，百分比计算出的像素值太小，导致内容无法正常显示

### 错误的配置示例
```vue
<!-- ❌ 当前写法：纯百分比 -->
<el-table-column label="货品" width="16%">
<el-table-column label="规格" width="6%">
<el-table-column label="数量" width="7%">
```

**问题所在：**
- 弹窗宽度假设为 1200px
- 16% × 1200 = 192px （货品列）
- 6% × 1200 = 72px （规格列）
- 7% × 1200 = 84px （数量列）
- 这些值在数据为空或窗口较小时会更小

---

## 解决方案

### ✅ 正确做法：使用 min-width + 自动扩展

**核心思路：**
1. **使用 `min-width` 替代 `width`** - 设置最小宽度保证可读性
2. **不设固定宽度** - 让 Element Plus 表格自动分配剩余空间
3. **保留操作列为固定宽度** - 操作按钮需要固定大小

### 新的列宽配置

| 列名 | 配置 | 最小宽度 | 说明 |
|------|------|----------|------|
| **货品** | `min-width="160"` | 160px | 较长文本+下拉框 |
| **供货商** | `min-width="100"` | 100px | 供货商名称 |
| **规格** | `min-width="60"` | 60px | 短文本（支/瓶等） |
| **单位类型** | `min-width="90"` | 90px | 下拉选择框 |
| **换算** | `min-width="80"` | 80px | 显示换算关系 |
| **数量** | `min-width="80"` | 80px | 数字输入框 |
| **进货单价** | `min-width="100"` | 100px | 金额输入框 |
| **金额** | `min-width="80"` | 80px | 只读显示 |
| **备注** | `min-width="80"` | 80px | 文本输入框 |
| **生产日期** | `min-width="110"` | 110px | 日期选择器 |
| **有效期至** | `min-width="110"` | 110px | 日期选择器 |
| **操作** | `width="60"` | 固定60px | 删除按钮 |

**总计最小宽度：** 160+100+60+90+80+80+100+80+80+110+110+60 = **1110px**

**工作原理：**
1. 当容器宽度 > 1110px 时，各列按比例自动扩展填满100%
2. 当容器宽度 ≤ 1110px 时，出现横向滚动条（保证可读性）
3. 操作列始终保持60px不变

## 技术说明

### Element Plus 表格列宽机制

#### ❌ `width="16%"` 的问题：
- **固定宽度**：严格按照百分比计算，不会自适应
- **压缩风险**：当容器变小时，列宽成比例缩小
- **不可读风险**：小屏幕下可能只有几十像素

#### ✅ `min-width="160"` 的优势：
- **最小保证**：至少占用160px，确保内容可读
- **自动扩展**：容器更大时，自动分配多余空间
- **弹性布局**：各列按 min-width 比例扩展

#### 📐 工作原理演示：

**场景1：容器宽度 = 1400px**
```
总 min-width = 1110px
剩余空间 = 1400 - 1110 = 290px

分配比例（基于 min-width）：
- 货品: 160 + (160/1110)*290 = 160 + 41.8 = 201.8px
- 规格: 60 + (60/1110)*290 = 60 + 15.7 = 75.7px
- ...以此类推
✅ 所有列都有足够空间显示完整
```

**场景2：容器宽度 = 1000px**
```
总 min-width = 1110px > 容器宽度 1000px
→ 出现横向滚动条
✅ 保证每列最小宽度不被压缩
```
