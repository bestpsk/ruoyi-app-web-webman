# 操作记录合并方案

## 📋 需求分析

**当前状态**：一次操作2个项目 → 生成2条操作记录
```
record_id=7: 品项1 ×1 (2026-05-09 23:42:39)
record_id=8: 品项2 ×1 (2026-05-09 23:42:39)
```

**期望状态**：一次操作2个项目 → 生成1条合并的操作记录
```
record_id=7: 品项1×1, 品项2×1 (2026-05-09 23:42:39)
```

---

## 🔍 当前表结构限制

### biz_operation_record 表字段

| 字段 | 类型 | 说明 | 问题 |
|------|------|------|------|
| product_name | varchar(100) | 品项名称 | 只能存1个名称 |
| operation_quantity | int | 操作次数 | 只能存1个数量 |
| package_item_id | bigint | 套餐明细ID | 只能关联1个套餐项 |
| consume_amount | decimal(12,2) | 消耗金额 | 可以累加 |

**核心问题**：当前表结构设计为"一条记录=一个品项"，无法直接存储多个品项。

---

## ✅ 解决方案

### 方案A：修改表结构（推荐）

#### 步骤1：添加 items 字段（JSON格式）

```sql
ALTER TABLE biz_operation_record 
ADD COLUMN items JSON DEFAULT NULL COMMENT '操作项目JSON:[{name,quantity,package_item_id}]' 
AFTER product_name;
```

#### 步骤2：修改后端逻辑

**文件**：[BizOperationRecordService.php](file:///d:\fuchenpro\webman\app\service\BizOperationRecordService.php)

**核心改动**：
1. 检查是否已有同客户+同日期+同操作人的记录
2. 如果有 → 更新该记录的 items 字段（JSON追加）
3. 如果没有 → 创建新记录

```php
public function insertRecord($data)
{
    $data['create_time'] = date('Y-m-d H:i:s');
    if (empty($data['operation_date'])) $data['operation_date'] = date('Y-m-d');
    if (empty($data['operation_type'])) $data['operation_type'] = '0';

    // 构建新项目数据
    $newItem = [
        'name' => $data['product_name'] ?? '',
        'quantity' => intval($data['operation_quantity'] ?? 1),
        'package_item_id' => $data['package_item_id'] ?? null
    ];

    // 查找已有记录（同客户+同日期+同操作人）
    $existing = BizOperationRecord::where('customer_id', $data['customer_id'])
        ->where('operation_date', $data['operation_date'])
        ->where('operator_user_id', $data['operator_user_id'] ?? 0)
        ->where('operation_type', $data['operation_type'])
        ->first();

    if ($existing) {
        // 合并到已有记录
        $existingItems = json_decode($existing->items, true) ?: [];
        $existingItems[] = $newItem;

        $existing->items = json_encode($existingItems, JSON_UNESCAPED_UNICODE);
        $existing->operation_quantity += $newItem['quantity'];
        $existing->consume_amount += $this->calculateConsumeAmount($newItem);
        $existing->save();

        // 更新套餐次数
        $this->updatePackageItem($newItem['package_item_id'], $newItem['quantity']);

        return $existing;
    }

    // 创建新记录
    $data['items'] = json_encode([$newItem], JSON_UNESCAPED_UNICODE);
    $record = BizOperationRecord::create($data);

    // 更新套餐次数
    $this->updatePackageItem($newItem['package_item_id'], $newItem['quantity']);

    // 生成档案
    $this->createArchive($record);

    return $record;
}
```

#### 步骤3：修改前端显示

**文件**：操作记录列表页面（Web端和AppV3端）

将 `product_name` 改为解析 `items` 字段：
```vue
<template>
  <view v-if="record.items">
    <text v-for="(item, idx) in JSON.parse(record.items)" :key="idx">
      {{ item.name }}×{{ item.quantity }}
      <text v-if="idx < JSON.parse(record.items).length - 1">, </text>
    </text>
  </view>
</template>
```

### 方案B：不修改表结构（简化方案）

如果不想修改表结构，可以：

1. **product_name 改为拼接字符串**：`"品项1×1, 品项2×1"`
2. **operation_quantity 改为总次数**：`2`
3. **package_item_id 设为 null**（无法关联多个）

**缺点**：
- 无法精确追踪每个品项的套餐扣减
- 无法区分不同品项的单价

---

## 📊 方案对比

| 维度 | 方案A（修改表结构） | 方案B（拼接字符串） |
|------|-------------------|-------------------|
| 数据完整性 | ✅ 保留完整信息 | ⚠️ 丢失部分信息 |
| 套餐追踪 | ✅ 精确扣减 | ❌ 无法追踪 |
| 实现复杂度 | 中等 | 简单 |
| 兼容性 | 需要迁移数据 | 兼容现有数据 |
