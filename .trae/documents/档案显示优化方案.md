# 档案显示优化方案

## 📋 问题清单

### 问题1：档案内容靠左，右侧空白
**现象**：档案卡片所有内容都集中在左侧，右边大片空白区域

**当前布局**：单列垂直排列，无宽度限制

### 问题2：照片加载失败（"加载失败"）
**现象**：档案中照片显示为灰色占位图，提示"加载失败"

**根因分析**：

数据库中的照片数据：
```json
["20260509/74e407483fabcd1997c631eab465649e.png","20260509/a4c36981c3f7fe74d9580c9133ab66a7.jpg"]
```

**这是相对路径**，但 `el-image` 的 `:src="url"` 直接使用原始路径，没有拼接服务器基础URL。

对比 ImageUpload 组件的处理逻辑（[index.vue:119-120](file:///d:\fuchenpro\front\src\components\ImageUpload\index.vue#L119-L120)）：
```javascript
if (item.indexOf(baseUrl) === -1 && !isExternal(item)) {
  item = { name: baseUrl + item, url: baseUrl + item }  // ✅ 拼接了baseUrl
}
```

**档案中的 `el-image` 缺少这个拼接逻辑！**

---

## ✅ 修复方案

### 修复1：优化档案卡片布局（左右分布）

将档案卡片改为**左右两栏布局**：
- 左侧：日期、类型、操作人、方案、金额、满意度
- 右侧：照片（如果有）

如果没有照片，则内容居左显示。

### 修复2：修复照片加载失败

在 `el-image` 的 `:src` 中拼接 `baseUrl`。

## 📝 具体修改

### 文件：[d:\fuchenpro\front\src\views\business\sales\index.vue](file:///d:\fuchenpro\front\src\views\business\sales\index.vue)

#### 修改1：添加 baseUrl 导入

在 `<script setup>` 部分添加：
```javascript
const baseUrl = import.meta.env.VITE_APP_BASE_API
```

#### 修改2：修改 parseArchivePhotos 方法，返回完整URL

```javascript
function parseArchivePhotos(photosStr) {
  if (!photosStr) return []
  try {
    const parsed = JSON.parse(photosStr)
    return Array.isArray(parsed) ? parsed.map(url => {
      if (url.indexOf(baseUrl) === -1 && !isExternal(url)) {
        return baseUrl + url
      }
      return url
    }) : []
  } catch { return photosStr.split(',').filter(s => s.trim()) }
}
```

#### 修改3：优化档案卡片布局

将 `archive-card-body` 改为左右两栏布局：

**修改前**（垂直单列）：
```vue
<div class="archive-card-body">
  <div>方案：...</div>
  <div>金额：... 满意度：...</div>
  <div>照片：[图片]</div>
  <div>顾客反馈：...</div>
  <div>备注：...</div>
</div>
```

**修改后**（左右两栏）：
```vue
<div class="archive-card-body">
  <div class="archive-main">
    <div>方案：...</div>
    <div>金额：... 满意度：...</div>
    <div>顾客反馈：...</div>
    <div>备注：...</div>
  </div>
  <div v-if="parseArchivePhotos(item.photos).length > 0" class="archive-photos-right">
    <el-image ... />
  </div>
</div>
```
