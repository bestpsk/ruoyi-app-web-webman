# 考勤配置修改报错修复计划

## 问题分析

**错误信息**：`Column not found: 1054 Unknown column 'rule' in 'field list'`

**原因**：
1. 后端 `info` 方法返回的数据包含了关联关系 `rule` 和 `dept`（通过 `with(['rule', 'dept'])` 加载）
2. 前端获取数据后修改时，把整个对象（包括 `rule` 和 `dept` 关联对象）都传回后端
3. 后端 `updateConfig` 方法直接使用 `$data` 数组进行更新，没有过滤掉非数据库字段
4. 导致 SQL 语句尝试更新 `rule` 和 `dept` 字段，但这些字段在数据库表中不存在

**SQL 示例**：
```sql
UPDATE `biz_attendance_config` 
SET ..., `rule` = ?, `dept` = ? 
WHERE `config_id` = 2
```

---

## 解决方案

在 `updateConfig` 方法中，只保留 `$fillable` 中定义的字段，过滤掉其他字段（如 `rule`, `dept`）。

---

## 实施步骤

### 步骤1：修改后端服务
**文件**: `d:\fuchenpro\webman\app\service\BizAttendanceConfigService.php`

修改 `updateConfig` 方法，只更新 `$fillable` 中定义的字段：

```php
public function updateConfig($data)
{
    if (!empty($data['user_ids']) && is_array($data['user_ids'])) {
        $data['user_ids'] = implode(',', $data['user_ids']);
    }
    $data['update_time'] = date('Y-m-d H:i:s');
    
    // 只保留 fillable 字段
    $fillable = [
        'config_name', 'rule_id', 'config_type', 'user_ids', 'dept_id',
        'status', 'remark', 'create_by', 'create_time', 'update_by', 'update_time'
    ];
    $updateData = array_intersect_key($data, array_flip($fillable));
    
    return BizAttendanceConfig::where('config_id', $data['config_id'])->update($updateData);
}
```
