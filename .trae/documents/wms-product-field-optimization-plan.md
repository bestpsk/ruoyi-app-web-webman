# 货品管理字段优化方案对比

## 需求回顾

1. 类别放在供货商后面
2. 规格和单位换个位置
3. 在规格前加一项：数量（一盒为单位的货品中可能含有10支的规格）
4. 规格和单位都要加入字典
5. 单位：箱、件、套、罐、盒、袋、包
6. 规格：支、瓶、件、套、瓶、片、个
7. 出货可以按照整套也可以按照规格出

---

## 方案对比

### 方案一：包装换算方案（推荐）

**核心思路**：货品设置主单位和副单位，通过换算比例自动换算

**字段设计**：
| 字段 | 类型 | 说明 |
|------|------|------|
| unit | char(1) | 主单位（字典：箱/件/套/罐/盒/袋/包） |
| spec | char(1) | 副单位/规格（字典：支/瓶/件/套/片/个） |
| pack_qty | int | 换算比例（如：10，表示1主单位=10副单位） |
| purchase_price | decimal | 进货价（按主单位） |
| sale_price | decimal | 出货价（按主单位，整件出货价） |
| sale_price_spec | decimal | 出货价（按副单位，拆分出货价） |

**库存管理**：
- 库存统一按**副单位（最小单位）**存储
- 入库5盒（换算比例10）→ 库存+50
- 出库时可选单位，系统自动换算

**优点**：
- 库存精确、自动换算、不会出现"半盒"问题
- 入库简单（按整件），出货灵活（可整可拆）
- 实现适中，适合美容院业务规模

**缺点**：
- 需要修改入库/出库逻辑
- 只支持两级单位换算

---

### 方案二：SKU多规格方案

**核心思路**：一个货品可以有多个规格（SKU），每个规格独立库存和价格

**表设计**：
```
biz_product（货品主表）
├── product_id, product_name, category, supplier_id...

biz_product_sku（规格表）
├── sku_id, product_id, sku_name, unit, price, stock...
```

**示例**：
- 货品：某某面膜
  - SKU1：盒装（10片/盒），价格100元，库存20盒
  - SKU2：单片装，价格12元，库存50片

**优点**：
- 最灵活，支持任意规格组合
- 可以设置不同包装不同价格

**缺点**：
- 实现复杂，需要新增表和页面
- 库存管理需要关联多个SKU
- 对于小型美容院可能过于复杂

---

### 方案三：批次管理方案

**核心思路**：入库时记录批次信息，每批次可以有不同包装规格和价格

**表设计**：
```
biz_stock_in_batch（入库批次表）
├── batch_id, stock_in_id, product_id, unit, pack_qty, price...
```

**优点**：
- 可追踪批次，支持价格变动
- 适合有保质期管理需求的场景

**缺点**：
- 管理成本高
- 操作相对复杂
- 对于美容院场景可能过度设计

---

### 方案四：简化描述方案

**核心思路**：规格仅作为描述信息，不做单位换算

**字段设计**：
| 字段 | 类型 | 说明 |
|------|------|------|
| unit | char(1) | 单位（字典） |
| spec | varchar(50) | 规格描述（如"10支/盒"） |
| pack_qty | int | 包装数量（仅用于显示） |

**库存管理**：
- 库存按单位管理，不做换算
- 拆零出库时手动调整数量

**优点**：实现最简单，改动最小

**缺点**：无法自动换算，容易出错

---

## 推荐方案：方案一（包装换算方案）

### 理由：
1. **符合美容院实际业务**：产品通常有包装单位（盒/箱）和最小单位（支/片）
2. **库存精确**：统一按最小单位管理，不会出现"半盒"问题
3. **操作灵活**：出库时可选择按盒出或按支出，系统自动换算
4. **实现适中**：比SKU方案简单，比简化方案更实用
5. **价格灵活**：支持整件出货价和拆分出货价

### 具体实现：

#### 1. 货品表字段（调整后顺序）
```
product_id       -- 货品ID
product_name     -- 品名
product_code     -- 货品编码
supplier_id      -- 供货商ID
category         -- 类别（字典）
unit             -- 主单位（字典：箱/件/套/罐/盒/袋/包）
spec             -- 副单位/规格（字典：支/瓶/件/套/片/个）
pack_qty         -- 换算比例（如10，表示1主单位=10副单位）
purchase_price   -- 进货价（按主单位）
sale_price       -- 出货价（按主单位）
sale_price_spec  -- 出货价（按副单位）
warn_qty         -- 预警数量（按副单位）
status           -- 状态
remark           -- 备注
```

#### 2. 库存表字段
```
inventory_id     -- 库存ID
product_id       -- 货品ID
quantity         -- 库存数量（统一按副单位/最小单位）
warn_qty         -- 预警数量
```

#### 3. 入库逻辑
- 入库时选择单位（主单位或副单位）
- 入库数量自动换算为副单位存入库存
- 示例：入库5盒（换算比例10）→ 库存+50

#### 4. 出库逻辑
- 出库时选择单位（主单位或副单位）
- 出库数量自动换算为副单位扣减库存
- 示例：出库3支 → 库存-3；出库1盒 → 库存-10

#### 5. 前端展示
- 列表显示：库存50（5盒）
- 出库时提供单位选择，自动计算金额

## 实施步骤（方案一）

### 第1步：创建升级SQL脚本
- 新增单位字典 `biz_product_unit`
- 新增规格字典 `biz_product_spec`
- 修改 `biz_product` 表：新增 `pack_qty`、`sale_price_spec`
- 修改 `biz_stock_in_item` 表：新增 `unit_type` 字段
- 修改 `biz_stock_out_item` 表：新增 `unit_type` 字段

### 第2步：修改后端
- 更新 BizProduct Model
- 更新 BizStockInService（入库换算逻辑）
- 更新 BizStockOutService（出库换算逻辑）
- 更新 BizInventoryService（库存显示逻辑）

### 第3步：修改前端
- 更新货品管理页面（字段顺序、字典选择、包装数量）
- 更新入库管理页面（单位选择、数量换算）
- 更新出库管理页面（单位选择、数量换算、价格计算）

### 第4步：执行SQL升级

### 第5步：测试验证
- 测试入库按主单位
- 测试入库按副单位
- 测试出库按主单位
- 测试出库按副单位
- 测试库存显示

## 增强建议（可选）

在方案一基础上，未来可以考虑：

1. **价格策略增强**：
   - 支持不同客户等级的价格
   - 支持批量折扣

2. **库存预警**：
   - 设置最小库存量
   - 按主单位和副单位双重预警

3. **出入库记录**：
   - 记录每次出入库的单位类型
   - 方便追溯和统计
