# 新增盘点单弹窗优化计划

## 需求分析

### 当前状态
- 弹窗宽度：900px
- 列顺序：品名 → 规格 → 换算 → 系统库存 → 实际数量 → 差异数量 → 备注
- 实际数量以混合格式输入（如 0.9盒）

### 目标状态
1. **弹窗加宽**：900px → 1100px
2. **新增列**：品名后面添加「单位类型」列（主单位(整)/副单位(拆)）
3. **调整列顺序**：品名 → 单位类型 → 换算 → 系统库存 → 实际数量 → **规格** → 差异数量 → 备注
4. **规格动态显示**：根据单位类型显示对应规格
   - 主单位(整) → 显示副单位规格标签（如"支"）
   - 副单位(拆) → 显示副单位规格标签（如"支"）
5. **实际数量/备注加宽**
6. **自动换算**：用户输入的实际数量自动换算为副单位保存到数据库

### 步骤6：系统库存和实际数量的显示格式调整

根据 `unitType` 动态调整显示格式：

```javascript
function formatItemQuantity(item, qty) {
  const packQty = item.packQty || 1
  const unitLabel = getUnitLabel(item.unit)
  const specLabel = getSpecLabel(item.spec)
  const unitType = item.unitType || '2'
  
  qty = qty || 0
  
  if (unitType === '1') {
    // 主单位模式：显示为 "X盒（Y支）"
    const mainQty = qty / packQty
    const mainStr = Number.isInteger(mainQty) ? mainQty : mainQty.toFixed(2)
    return mainStr + unitLabel + '（' + Math.round(qty) + specLabel + '）'
  } else {
    // 副单位模式：直接显示 "Y支"
    return qty + specLabel
  }
}
```

### 步骤7：保存时自动换算为副单位

在 `submitForm` 函数中，提交前将所有实际数量转换为副单位：

```javascript
function submitForm() {
  proxy.$refs["stockCheckRef"].validate(valid => {
    if (valid) {
      // 转换实际数量为副单位保存
      const itemsToSave = form.value.items.map(item => ({
        ...item,
        actualQuantity: convertToMinUnit(item)
      }))
      
      const submitData = { ...form.value, items: itemsToSave }
      // ... 提交逻辑
    }
  })
}

function convertToMinUnit(item) {
  const qty = item.actualQuantity || 0
  const unitType = item.unitType || '2'
  const packQty = item.packQty || 1
  
  if (unitType === '1' && packQty > 1) {
    return Math.round(qty * packQty)
  }
  return qty
}
```
