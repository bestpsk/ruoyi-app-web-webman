# 考勤配置数据显示和修改问题修复计划

## 问题分析

### 问题1：数据列表不显示配置名称、考勤规则、关联用户、关联部门
**原因**：后端返回的数据字段是 snake_case（如 `config_name`, `config_type`），但前端模板中使用的是 camelCase（如 `configName`, `configType`）。字段名不匹配导致数据无法显示。

**示例**：
- 后端返回：`{ config_name: "测试", config_type: 1, rule: { rule_name: "标准班" } }`
- 前端期望：`{ configName: "测试", configType: 1, rule: { ruleName: "标准班" } }`

### 问题2：修改数据保存后报错 `Undefined array key "config_id"`
**原因**：后端 `edit` 方法没有使用 `convert_to_snake_case` 转换请求数据，前端传的 `configId` 没有被转换成 `config_id`，导致 `$data['config_id']` 找不到键。

---

## 解决方案

### 方案1：后端返回 camelCase 格式数据
修改后端控制器，在返回数据时使用 `convert_to_camel_case` 转换字段名。

### 方案2：后端接收数据时转换
修改后端 `add` 和 `edit` 方法，使用 `convert_to_snake_case` 转换请求数据。

---
