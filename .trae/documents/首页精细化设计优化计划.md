# 首页精细化设计优化计划

## 一、问题诊断

从浏览器检查元素发现以下具体问题：

### 问题1：菜单图标尺寸和颜色不对
- 实际渲染：`font-size: 28px; color: rgb(60, 150, 243)` — 使用了**旧颜色 #3C96F3**
- 图标尺寸 28px 在 96rpx 的 icon-wrapper 中偏大，视觉不协调
- 应为：size=20（约20px），color=#3D6DF7（新主色）

### 问题2：数据统计区设计单调
- 当前用 Tab 切换今日/本月，只能看一组数据
- 用户要求：**同时展示两组数据**，今日数据醒目（蓝色大字），本月数据在其下方以浅灰色小字显示
- 参考图1的设计感：数据层次分明，信息密度高但不拥挤

### 问题3：订单列表格式需微调
- 当前：`3/5次`
- 要求：`余3次/5次`

---

## 二、优化方案

### 模块1：常用菜单区（修复图标问题）

**改动点：**
| 属性 | 改前 | 改后 |
|------|------|------|
| up-icon size | 24 | **20** |
| up-icon color | #3D6DF7（但实际渲染旧色） | **#3D6DF7** 确保生效 |
| icon-wrapper 尺寸 | 96rpx | **88rpx** |
| icon-wrapper 与文字间距 | gap: 16rpx | **gap: 12rpx** |
| menu-label 字号 | 26rpx | **24rpx** |

**效果：** 图标更精致小巧，与容器比例协调，整体更紧凑有呼吸感。

### 模块2：数据概览区（重大重构）

**新布局设计：**

```
┌─────────────────────────────────────────────┐
│  数据概览                              🔄   │
│  ────────────────────────────────────────  │
│                                             │
│  咨询客数    成交客数    成交金额    成交项次 │
│                                             │
│   128 ↗      36 ↗     ¥12.8k ↗    89 ↘   │  ← 今日数据（蓝色粗体大字）
│ 1,256 ↗     386 ↗    ¥128.5k ↗   892 ↗   │  ← 本月数据（灰色常规字）
│                                             │
└─────────────────────────────────────────────┘
```

**样式规范：**
| 元素 | 今日数据 | 本月数据 |
|------|----------|----------|
| 字号 | 40rpx | 24rpx |
| 字重 | 700 | 400 |
| 颜色 | #3D6DF7（蓝色） | #86909C（浅灰） |
| 趋势箭头 | 有（绿色↑/红色↓） | 无 |
| 行间距 | — | 与上方数据间距 8rpx |

**移除：** Tab 切换（今日/本月标签）、tab-indicator 指示器
**保留：** 刷新按钮、4列网格布局、分割线

### 模块3：订单明细列表（格式微调）

**改动：**
- 显示格式：`余3次/5次`（加"余"字，更符合美容行业"剩余次数"语义）
- 数据字段保持不变：remainCount / totalCount

### 模块4：整体细节打磨

| 优化项 | 说明 |
|--------|------|
| 卡片间距 | 统一 16rpx（原 20rpx），更紧凑 |
| 卡片内边距 | 统一 24rpx |
| 圆角 | 统一 20rpx（卡片）、16rpx（内部元素） |
| 阴影 | 更轻柔 `0 2rpx 12rpx rgba(61,109,247,0.06)` |

## 三、实施步骤

### 步骤1：修复菜单图标
- icon size 24→20
- icon-wrapper 96rpx→88rpx  
- 调整间距 gap 16rpx→12rpx
- menu-label 26rpx→24rpx

### 步骤2：重构数据概览区
- 移除 Tab 切换组件
- 每个 stat-cell 内显示两行数据：
  - 第一行：label + 今日数值（蓝色大字 + 趋势箭头）
  - 第二行：本月数值（灰色小字，无箭头）
- 删除 timeType 相关逻辑

### 步骤3：修正订单列表格式
- `{{ item.remainCount }}/{{ item.totalCount }}次` → `余{{ item.remainCount }}次/{{ item.totalCount }}`

### 步骤4：统一间距和圆角微调
- 卡片 margin-bottom: 20rpx→16rpx
- 阴影减轻
- 整体视觉一致性检查
