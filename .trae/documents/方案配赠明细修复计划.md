# 方案配赠明细供货商和单位切换修复计划

## 问题分析

### 问题1：供货商不显示
**原因**：后端 `selectProductList` 方法没有加载关联的 `supplier` 数据，导致返回的货品列表中没有 `supplierName` 字段。

**对比**：`searchProduct` 方法使用了 `->with('supplier')` 并手动添加了 `supplier_name` 字段。

### 问题2：主副单位切换后规格、单价、总价不变
**原因**：方案管理中的 `onUnitTypeChange` 函数没有实现正确的价格换算逻辑。

**入库管理参考逻辑**：
- 选择货品时保存原始主单位价格到 `_mainPrice`
- 主单位(整)：价格 = `_mainPrice`，规格 = 主单位标签（如"盒"）
- 副单位(拆)：价格 = `_mainPrice / packQty`，规格 = 副单位标签（如"支"）
- 总价 = 单价 × 数量

## 解决方案

### 步骤1：修改后端货品列表接口
**文件**: `d:\fuchenpro\webman\app\service\BizProductService.php`

在 `selectProductList` 方法中添加 `with('supplier')` 并手动添加 `supplier_name` 字段：

```php
public function selectProductList($params = [])
{
    $query = BizProduct::with('supplier');
    // ... 其他条件
    
    $result = $query->orderBy('product_id', 'desc')->paginate($pageSize, ['*'], 'page', $pageNum);
    
    foreach ($result as $product) {
        $product->supplier_name = $product->supplier ? $product->supplier->supplier_name : null;
    }
    
    return $result;
}
```

### 步骤2：修改前端选择货品时的数据填充
**文件**: `d:\fuchenpro\front\src\views\business\plan\index.vue`

修改 `onProductSelect` 函数，保存原始主单位价格并初始化显示字段：

```javascript
function onProductSelect(index, productId) {
  const product = productOptions.value.find(p => p.productId === productId)
  if (product) {
    const item = planForm.value.items[index]
    item.productId = product.productId
    item.productName = product.productName
    item.supplierId = product.supplierId
    item.supplierName = product.supplierName || ''
    item.packQty = product.packQty || 1
    item.unitType = '1'
    // 保存原始主单位价格
    item._mainPrice = product.salePrice || 0
    item.salePrice = product.salePrice || 0
    // 设置单位标签
    const unitMap = { '1': '箱', '2': '件', '3': '套', '4': '罐', '5': '盒', '6': '袋', '7': '包' }
    const specMap = { '1': '支', '2': '瓶', '3': '件', '4': '套', '5': '片', '6': '个' }
    item.unitLabel = unitMap[product.unit] || ''
    item.specLabel = specMap[product.spec] || ''
    onUnitTypeChange(index)
  }
}
```

### 步骤3：修改单位类型切换函数
**文件**: `d:\fuchenpro\front\src\views\business\plan\index.vue`

重写 `onUnitTypeChange` 函数，实现正确的价格换算：

```javascript
function onUnitTypeChange(index) {
  const item = planForm.value.items[index]
  const packQty = item.packQty || 1

  if (item.unitType === '1') {
    // 主单位(整): 使用原始主单位价格
    if (item._mainPrice) {
      item.salePrice = item._mainPrice
    }
    item.spec = item.unitLabel || ''
  } else {
    // 副单位(拆): 价格 = 主单位价格 / 换算比例
    if (!item._mainPrice && item.salePrice) {
      item._mainPrice = item.salePrice
    }
    if (item._mainPrice && packQty > 0) {
      item.salePrice = Math.round((item._mainPrice / packQty) * 100) / 100
    }
    item.spec = item.specLabel || ''
  }

  onItemQuantityChange(index)
}
```

### 步骤4：修改新增明细时的默认值
**文件**: `d:\fuchenpro\front\src\views\business\plan\index.vue`

在 `addPlanItem` 函数中添加 `_mainPrice` 字段：

```javascript
function addPlanItem() {
  planForm.value.items.push({
    productId: undefined, productName: '', supplierId: undefined, supplierName: '',
    unitType: '1', packQty: 1, quantity: 1, spec: '', salePrice: 0, amount: 0,
    unitLabel: '', specLabel: '',
    _mainPrice: null  // 新增：保存原始主单位价格
  })
}
```

## 预期效果

1. **供货商显示**：选择货品后，供货商列会自动显示该货品的供货商名称
2. **单位切换联动**：
