# 销售开单页面筛选优化计划

## 问题分析

### 问题1: 输入框尺寸问题
- **位置**: [front/src/views/business/sales/index.vue](file:///d:/fuchenpro/front/src/views/business/sales/index.vue#L23-L29)
- **现状**: 成交筛选和满意度筛选的`el-select`组件设置了`size="small"`
- **需求**: 去掉small设置,改为默认大小显示

### 问题2: 满意度筛选精度问题
- **位置**: 
  - 前端: [front/src/views/business/sales/index.vue](file:///d:/fuchenpro/front/src/views/business/sales/index.vue#L27-L29)
  - 后端: [webman/app/service/BizCustomerService.php](file:///d:/fuchenpro/webman/app/service/BizCustomerService.php#L86-L89)
- **现状**: 
  - 前端只提供1-5星的整数选项
  - 后端使用`floor()`向下取整后匹配,例如4.3和4.7都会匹配到4星
- **需求**: 满意度可能是小数值(如4.3, 4.7, 3.2),需要更精确的筛选方式

## 实施方案

### 方案1: 输入框尺寸调整
**修改文件**: `front/src/views/business/sales/index.vue`

**修改内容**:
- 第23行:去掉成交筛选的`size="small"`属性
- 第27行:去掉满意度筛选的`size="small"`属性

### 方案2: 满意度筛选优化

#### 方案2-A: 范围筛选(推荐)
**优点**: 可以精确筛选某个范围的满意度,更灵活
**缺点**: 需要两个输入框,UI稍复杂

**实现方式**:
1. 前端改为使用两个数字输入框(最小满意度、最大满意度)
2. 后端修改筛选逻辑,支持范围查询

#### 方案2-B: 星级范围筛选(简化方案)
**优点**: UI简单,用户体验好
**缺点**: 精度稍低,但符合星级评分习惯

**实现方式**:
1. 前端保持星级选择,但改为范围选择(如"4星及以上"、"3-4星"等)
2. 后端修改为范围匹配逻辑

#### 方案2-C: 精确小数筛选
**优点**: 最精确
**缺点**: 用户需要输入具体数值,操作复杂

**实现方式**:
1. 前端改为数字输入框,允许输入小数
2. 后端支持精确匹配或范围匹配

## 推荐方案

综合考虑用户体验和功能需求,推荐采用**方案2-B(星级范围筛选)**:

// 修改前
if ($satisfaction !== null && $satisfaction !== '') {
    $satFilter = intval($satisfaction);
    if ($customer->avg_satisfaction === null || floor($customer->avg_satisfaction) !== $satFilter) continue;
}
