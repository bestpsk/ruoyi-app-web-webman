# 分析菜单无法保存的原因

## 问题分析

### 当前代码逻辑

在 `handleUpdate` 函数中，分离菜单ID和按钮ID：

```javascript
res.checkedKeys.forEach(id => {
  const node = findNodeById(menuOptions.value, id)
  if (node) {
    if (!node.children || node.children.length === 0) {
      buttonIds.push(id)  // 叶子节点 → 当作按钮
    } else {
      menuIds.push(id)   // 有子节点 → 当作菜单
    }
  }
})
```

### 问题根源

**错误的判断逻辑**：当前代码把所有"没有子节点的节点"都当成按钮权限。

但实际上：
- **真正的按钮**：如"用户查询"、"用户新增"等（父节点是"用户管理"）
- **没有子节点的菜单**：如"数据监控"、"服务监控"、"表单构建"、"系统接口"等

这些没有子节点的菜单页面被错误地归类为按钮权限，导致：
1. 它们不会显示在 el-tree 中（因为被过滤掉了）
2. 它们的选中状态由 `buttonCheckState` 管理
3. 但如果 `isButtonGroup` 判断不准确，可能导致丢失

## 解决方案

修改判断逻辑：只有当节点的**兄弟节点都是叶子节点**时，才认为是按钮权限。

### 实施步骤

**文件**: `front/src/views/system/role/index.vue`

#### 步骤1：添加辅助函数判断是否为按钮权限

```javascript
function isButtonPermission(nodeId, parentNode) {
  if (!parentNode || !parentNode.children) return false
  return parentNode.children.length > 1 &&
         parentNode.children.every(child =>
           !child.children || child.children.length === 0
         )
}
```

#### 步骤2：修改 handleUpdate 中的分离逻辑

```javascript
res.checkedKeys.forEach(id => {
  const node = findNodeById(menuOptions.value, id)
  if (node) {
    const parentNode = findParentNode(menuOptions.value, id)
    if (parentNode && isButtonGroup(parentNode.children)) {
      buttonIds.push(id)
    } else {
      menuIds.push(id)
    }
  }
})
```

#### 步骤3：添加查找父节点函数

```javascript
function findParentNode(nodes, targetId, parent = null) {
  for (const node of nodes) {
    if (node.id === targetId) return parent
    if (node.children) {
      const found = findParentNode(node.children, targetId, node)
      if (found) return found
    }
  }
  return null
}
```

## 预期效果

- "数据监控"、"服务监控"、"表单构建"等无子节点的菜单正确识别为菜单
- "用户查询"、"用户新增"等按钮正确识别为按钮权限
- 所有选中的菜单和按钮都能正确保存
