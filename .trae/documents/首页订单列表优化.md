### 步骤4：修复导航逻辑（第93-108行）
**当前状态：**
```javascript
function handleOrderClick(item) {
  if (!item.sourceId) return
  switch (item.sourceType) {
    case '0':
      uni.navigateTo({ url: `/pages/business/order/detail?id=${item.sourceId}` })
      break
    case '1':
      uni.navigateTo({ url: `/pages/business/sales/operation?recordId=${item.sourceId}` })
      break
    case '2':
      uni.showToast({ title: '还款记录', icon: 'none' })
      break
    default:
      uni.showToast({ title: '暂无详情', icon: 'none' })
  }
}
```

**问题分析：**
- 当 `sourceType === '1'`（操作类型）时，会跳转到添加操作页面 `/pages/business/sales/operation`
- 用户期望：所有订单类型都应该跳转到订单详情页

**修改为：**
```javascript
function handleOrderClick(item) {
  if (!item.sourceId) return
  
  // 统一跳转到订单详情页（无论什么类型）
  uni.navigateTo({ 
    url: `/pages/business/order/detail?id=${item.sourceId}` 
  })
}
```

**预期效果：**
- 点击"开单"订单 → 显示订单详情 ✓
- 点击"操作"订单 → 显示订单详情 ✓（修复前是添加操作页面）
- 点击其他类型订单 → 显示订单详情 ✓
