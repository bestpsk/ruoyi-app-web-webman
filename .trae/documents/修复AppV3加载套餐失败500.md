# 修复：AppV3 销售开单"加载套餐失败: 500"

## 问题分析

### 现象
AppV3 销售开单页面（operation.vue）加载套餐时报错：`加载套餐失败: 500`，提示"套餐不存在"。

### 根因
**前端 API 路径与后端路由不匹配！**

| 项目 | 路径 |
|------|------|
| **AppV3 前端调用** | `GET /business/package/customer` ❌ |
| **后端路由定义** | `GET /business/package/byCustomer` ✅ |
| **PC 端前端调用** | `GET /business/package/byCustomer` ✅ |

AppV3 的 `customerPackage.js` 中 API URL 写的是 `/business/package/customer`，而后端只有 `/business/package/byCustomer` 路由。

### 错误链路
1. AppV3 调用 `GET /business/package/customer?customerId=xxx`
2. 后端路由匹配：不匹配 `/business/package/byCustomer`，但匹配 `/business/package/{packageId}`，此时 `packageId = 'customer'`
3. 进入 `getInfo()` 方法：`intval('customer') = 0`
4. 查询 `selectPackageById(0)` 返回 `null`
5. 返回 `AjaxResult::error('套餐不存在')`，HTTP code 500
6. 前端 catch 到 500 错误，打印 `加载套餐失败: 500`

## 修复方案

修改 AppV3 的 API 文件，将 URL 从 `/business/package/customer` 改为 `/business/package/byCustomer`，与后端路由和 PC 端保持一致。

### 修改文件

**文件**: `f:\fuchen\AppV3\src\api\business\customerPackage.js`（第 24 行）

```javascript
// 修改前
return request({ url: '/business/package/customer', method: 'get', params: { customerId, status } })

// 修改后
return request({ url: '/business/package/byCustomer', method: 'get', params: { customerId, status } })
```

仅需修改 1 个文件 1 行代码。
