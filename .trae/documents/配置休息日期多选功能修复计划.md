# 配置休息日期多选功能修复计划

## 一、问题分析

### 1.1 用户反馈
员工配置--配置休息日期--选中日期后没有反应（日期要可多选）

### 1.2 问题根因
查看代码发现：

**前端代码**（index.vue 第353行）：
```vue
<div :class="['rest-date-cell', { 'is-rest': isRestDate(data.date) }]" @click="toggleRestDate(data.date)">
```

**问题**：`el-calendar`组件的`data.date`返回的是**Date对象**，而后端返回的`tempRestDates`是**字符串数组**（如`['2026-04-30']`）

**isRestDate函数**（第731行）：
```javascript
function isRestDate(dateStr) { return tempRestDates.value.includes(dateStr) }
```

**问题**：直接用Date对象和字符串比较，永远返回false

### 1.3 数据格式对比
| 来源 | 格式 | 示例 |
|------|------|------|
| `data.date` (el-calendar) | Date对象 | `new Date('2026-04-30')` |
| `tempRestDates` (后端返回) | 字符串数组 | `['2026-04-30', '2026-04-29']` |

---

## 二、修复方案

### 修复：前端 - 转换Date对象为字符串格式

**文件**：`d:\fuchenpro\front\src\views\business\schedule\index.vue`

**修改isRestDate函数**（第731行）：
```javascript
function isRestDate(date) {
  const dateStr = date instanceof Date ? date.toISOString().slice(0, 10) : date
  return tempRestDates.value.includes(dateStr)
}
```

**修改toggleRestDate函数**（第733-736行）：
```javascript
function toggleRestDate(date) {
  const dateStr = date instanceof Date ? date.toISOString().slice(0, 10) : date
  const idx = tempRestDates.value.indexOf(dateStr)
  idx > -1 ? tempRestDates.value.splice(idx, 1) : tempRestDates.value.push(dateStr)
}
```
