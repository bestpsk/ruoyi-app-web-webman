# 查看出库单显示名称问题深度修复计划

## 问题描述
查看出库单时：
- 出库企业显示 `2`（ID）而不是企业名称
- 对接员工显示 `100`（ID）而不是员工姓名

**状态**：已修改后端代码但**仍然显示数字**

---

## 问题排查分析

### 已完成的修改 ✅
1. 后端 `selectStockOutById()` 方法已添加名称查询逻辑
2. 判断条件已改为 `is_numeric() || empty()`
3. 数据库字段 `out_target_type` 已添加

### 可能的原因 🔍

#### 原因1：后端服务未重启（最可能）
- PHP Webman 框架可能使用文件缓存
- 修改的 PHP 文件未重新加载
- **解决**：必须重启 Worker 进程

#### 原因2：浏览器缓存
- 前端 JavaScript/组件被缓存
- API 响应被浏览器缓存
- **解决**：强制刷新（Ctrl+Shift+R）

#### 原因3：查看的是旧数据
- 截图显示日期是 `2026-05-04`
- 这可能是修改前创建的数据
- **解决**：新增一条出库单后立即查看

#### 原因4：新增出库单保存逻辑问题
- 新增时 `enterprise_name` 字段存储的可能就是 ID
- 需要检查 `insertStockOut()` 方法

---

## 详细修复步骤

### 步骤1：检查新增出库单时的数据存储

**文件**: `d:\fuchenpro\webman\app\service\BizStockOutService.php`

**位置**: `insertStockOut()` 方法（约第122行）

**当前代码分析**:
```php
public function insertStockOut($data)
{
    $items = $data['items'] ?? [];
    unset($data['items']);
    // ... 其他处理 ...
    $stockOut = BizStockOut::create($data);  // 直接存储 $data
}
```

**问题**: `$data` 中 `enterprise_name` 和 `contact_employee_name` 的值是什么？

**需要验证**:
- 如果前端传来的 `enterpriseName` 是 ID（如 "2"），则数据库存的就是 ID
- 需要确保前端传来的是实际名称

---

### 步骤2：强制查询名称（不依赖判断条件）

**修改方案**: 无论当前值是什么，只要有 ID 就重新查询名称

```php
// 强制模式：始终根据 ID 查询真实名称
$enterpriseName = '-';
if (!empty($stockOutArray['enterprise_id'])) {
    $enterprise = BizEnterprise::find($stockOutArray['enterprise_id']);
    if ($enterprise) {
        $enterpriseName = $enterprise->enterprise_name;
    }
}

$contactEmployeeName = '-';
if (!empty($stockOutArray['contact_employee_id'])) {
    $user = SysUser::find($stockOutArray['contact_employee_id']);
    if ($user) {
        $contactEmployeeName = $user->real_name ?: $user->user_name;
    }
}

$responsibleName = '-';
if (!empty($stockOutArray['responsible_id'])) {
    $user = SysUser::find($stockOutArray['responsible_id']);
    if ($user) {
        $responsibleName = $user->real_name ?: $user->user_name;
    }
}
```

**优势**:
- 不依赖现有值的判断
- 只要有 ID 就一定查询
- 确保返回真实名称

## 实施计划

### 文件: `d:\fuchenpro\webman\app\service\BizStockOutService.php`

**修改位置**: 第82-112行

将现有的条件判断改为**强制查询模式**：

```php
$stockOutArray = $stockOut->toArray();

// 强制根据 ID 查询真实名称（确保不显示 ID）

$contactEmployeeName = '-';
if (!empty($stockOutArray['contact_employee_id'])) {

        $contactEmployeeName = $user->real_name ?: $user->user_name;
    } else {
        $contactEmployeeName = $stockOutArray['contact_employee_name'] ?: ('ID:' . $stockOutArray['contact_employee_id']);
    }
} elseif (!empty($stockOutArray['contact_employee_name'])) {
    $contactEmployeeName = $stockOutArray['contact_employee_name'];

$responsibleName = '-';

    }

    $responsibleName = $stockOutArray['responsible_name'];
```

## 备选方案（如果上述仍无效）

### 方案A：在前端处理
如果后端确实无法正确返回名称，可以在前端 `handleView()` 中添加额外的 API 调用来获取名称。

### 方案B：修复数据
直接更新数据库中的旧数据：
```sql
UPDATE biz_stock_out so
JOIN bizEnterprise e ON so.enterprise_id = e.enterprise_id
SET so.enterprise_name = e.enterprise_name
WHERE is_numeric(so.enterprise_name);
```
