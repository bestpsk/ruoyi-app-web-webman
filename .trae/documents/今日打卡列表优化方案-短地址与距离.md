```javascript
function calculateDistance(lat1, lng1, lat2, lng2) {
  const R = 6371000
  const dLat = (lat2 - lat1) * Math.PI / 180
  const dLng = (lng2 - lng1) * Math.PI / 180
  const a = Math.sin(dLat / 2) * Math.sin(dLat / 2) +
            Math.cos(lat1 * Math.PI / 180) * Math.cos(lat2 * Math.PI / 180) *
            Math.sin(dLng / 2) * Math.sin(dLng / 2)
  const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a))
  return Math.round(R * c)
}

function formatDistance(meters) {
  if (!meters && meters !== 0) return ''
  if (meters < 1000) {
    return `${meters}米`
  }
  return `${(meters / 1000).toFixed(1)}公里`
}
```

### 步骤2：修改地址解析成功后的处理

在 [index.vue 的 handleLocationSuccess 函数](file:///d:/fuchenpro/AppV3/src/pages/attendance/index.vue) 中：

```javascript
function handleLocationSuccess(lat, lng) {
  location.value.latitude = lat
  location.value.longitude = lng
  location.value.address = `${lat.toFixed(6)}, ${lng.toFixed(6)}`
  locationLoading.value = false
  locationError.value = false
  
  // 计算距离考勤点的距离
  distanceToWorkplace.value = calculateDistance(
    lat, lng,
    WORKPLACE_LOCATION.latitude,
    WORKPLACE_LOCATION.longitude

  reverseGeocode(lat, lng)
}
```

并在 reverseGeocode 成功回调中更新短地址：

```javascript
async function reverseGeocode(lat, lng) {
  geocodingLoading.value = true
  try {
    const key = 'd184e115457658cbcf3f92ed8e3a1772'
    const url = `https://restapi.amap.com/v3/geocode/regeo?location=${lng},${lat}&key=${key}&extensions=all&output=JSON`
    
    const res = await new Promise((resolve, reject) => {
      uni.request({ url, method: 'GET', timeout: 8000, success: resolve, fail: reject })
    })
    
    if (res.data?.regeocode) {
      const regeocode = res.data.regeocode
      location.value.address = regeocode.formatted_address || location.value.address
      
      if (regeocode.pois && regeocode.pois.length > 0) {
        location.value.poi = regeocode.pois[0].name
      }
      
      if (regeocode.addressComponent) {
        const addr = regeocode.addressComponent
        location.value.district = `${addr.province || ''}${addr.city || ''}${addr.district || ''}`.trim()
      }
      
      // ✅ 新增：生成短地址用于提交
      shortAddress.value = formatShortAddress(location.value.address, location.value.poi)
    }
  } catch (e) {
    console.warn('逆地理编码失败', e)
  } finally {
    geocodingLoading.value = false
  }
}
```

## ⚠️ 注意事项

### 1. 考勤点坐标来源

当前方案中硬编码了考勤点坐标，实际应用中应考虑：

**方案A：从后端获取**（推荐）
```javascript
// 在 onMounted 或 loadData 时调用
async function loadWorkplaceConfig() {
  try {
    const res = await getWorkplaceConfig()
    WORKPLACE_LOCATION.latitude = res.data.latitude
    WORKPLACE_LOCATION.longitude = res.data.longitude
  } catch (e) {
    console.warn('加载考勤点配置失败')
  }
}
```

**方案B：从用户设置中读取**
```javascript
// 从 store 或 localStorage 读取
const workplaceConfig = userStore.workplaceConfig || {}
```

**方案C：暂时使用固定值**
```javascript
// 先用当前定位的第一个点作为参考（临时方案）
// 用户首次打卡时的位置作为考勤点
```

### 2. 兼容性处理

如果无法获取考勤点坐标或距离计算失败：
- 不显示距离信息
- 只显示刷新按钮
- 不影响其他功能

### 3. 数据迁移

对于历史数据：
- 已保存的长地址数据保持不变
- 前端显示时使用 `formatShortAddress()` 处理
- 新打卡的数据会保存短地址
