- `Key数量`显示18：因为 `dbSize` 是单独字段，不在info嵌套内 ✅

## 三、解决方案

### 方案选择

**推荐方案：后端扁平化处理**

修改 [CacheController.php](file:///d:\fuchenpro\webman\app\controller\monitor\CacheController.php#L12-L38) 的 `getInfo()` 方法，将嵌套的Redis INFO数据扁平化为前端期望的格式。

**优点：**
1. 前端代码无需改动（保持与RuoYi原版一致）
2. 后端统一数据处理逻辑，更易维护
3. 符合前后端分离原则（后端提供符合前端需求的数据格式）

## 四、实施步骤

### 步骤1：创建INFO数据扁平化方法

在 CacheController 中添加私有方法 `flattenInfo()`，将嵌套的section结构转换为扁平键值对。

**输入（当前格式）：**
```php
$info = [
    'server' => [
        'redisVersion' => '3.0.504',
        'tcpPort' => 6379,
    ],
    'memory' => [
        'usedMemoryHuman' => '5M',
    ]
]
```

**输出（目标格式）：**
```php
$flatInfo = [
    'redis_version' => '3.0.504',       // 转换为下划线格式
    'tcp_port' => '6379',
    'used_memory_human' => '5M',
]
```

### 步骤2：实现字段名转换规则

Redis INFO返回的字段使用驼峰命名（如 `redisVersion`, `tcpPort`），但前端期望下划线格式（如 `redis_version`, `tcp_port`）。

转换规则：
1. 将所有section下的字段提取到顶层
2. 驼峰命名转下划线命名（camelCase → snake_case）
3. 处理特殊字段映射

### 步骤3：修改getInfo()方法

```php
public function getInfo(Request $request)
{
    $redis = Redis::connection();
    $rawInfo = $redis->info();                    // 获取原始嵌套数据
    $info = $this->flattenInfo($rawInfo);          // 扁平化处理
    $dbSize = $redis->dbsize();
    
    // 命令统计处理（保持不变）
    $commandStats = [];
    if (isset($rawInfo['cmdstat_get']) || isset($rawInfo['stats']['cmdstat_get'])) {
        // 处理命令统计...
    }
    
    return AjaxResult::success('', [
        'info' => $info,                           // 返回扁平化的数据
        'dbSize' => $dbSize,
        'commandStats' => $commandStats,
    ]);
}
```

### 步骤4：实现flattenInfo()方法

核心逻辑：
```php
private function flattenInfo(array $nestedInfo): array
{
    $flatInfo = [];
    
    foreach ($nestedInfo as $section => $data) {
        if (is_array($data)) {
            // 如果是section（数组），递归处理其子项
            foreach ($data as $key => $value) {
                // 驼峰转下划线
                $snakeKey = $this->camelToSnake($key);
                $flatInfo[$snakeKey] = $value;
            }
        } else {
            // 如果已经是顶层字段，直接保留
            $snakeKey = $this->camelToSnake($section);
            $flatInfo[$snakeKey] = $data;
        }
    }
    
    return $flatInfo;
}

private function camelToSnake(string $str): string
{
    return strtolower(preg_replace('/([a-z])([A-Z])/', '$1_$2', $str));
}
```

## 六、技术说明

### Redis INFO命令的输出格式

Redis INFO命令返回的信息按照不同的**section**分组，每个section包含相关的配置和统计信息。主要的sections包括：

1. **Server**: 服务器基本信息（版本、端口、运行时间等）
2. **Clients**: 客户端连接信息
3. **Memory**: 内存使用情况
4. **Persistence**: 持久化配置（RDB/AOF）
5. **Stats**: 统计信息（命令执行次数、网络流量等）
6. **Replication**: 主从复制信息
7. **CPU**: CPU使用情况
8. **Cluster**: 集群信息
10. **Keyspace**: 数据库键空间信息

### PHP Redis扩展的行为

不同版本的PHP Redis扩展对INFO命令的处理方式可能不同：
- 某些版本会自动解析并返回关联数组（按section分组）
- 某些版本可能返回原始文本格式
- 字段名可能是驼峰式（camelCase）或保持原样

因此，**后端进行统一的扁平化和格式转换是最稳妥的方案**。

### 兼容性考虑

此方案兼容以下场景：
- 不同版本的Redis服务器
- 不同版本的PHP Redis扩展
- 单机模式和集群模式
- Windows/Linux操作系统
