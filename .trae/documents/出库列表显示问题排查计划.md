# 出库管理列表显示问题排查与修复计划

## 问题分析

从截图看，页面仍然显示旧的样式：
- 标题仍是"总数量(最小单位)"
- 只显示纯数字：10、1、20
- 列宽没有变化

### 代码状态检查
当前代码（index.vue 第47-57行）**已经是修改后的版本**：
- 标题已改为"总数量"
- 已添加自定义模板显示单位信息
- 列宽已调整

### 可能原因

1. **前端未热更新** - Vue 开发服务器可能需要重启
2. **浏览器缓存** - 需要强制刷新（Ctrl+Shift+R）
3. **后端服务未重启** - PHP 代码修改后需要重启 webman 服务

## 解决方案

### 步骤1：确认后端服务已更新
检查 `BizStockOutService.php` 的 `listStockOuts` 方法是否包含新增的单位信息查询代码。

### 步骤2：重启开发服务器
```bash
# 重启前端开发服务器
cd d:\fuchenpro\front
# 停止当前的 npm run dev，然后重新启动
npm run dev
```

### 步骤3：强制刷新浏览器
按 `Ctrl + Shift + R` 或 `Ctrl + F5` 强制刷新页面，清除缓存。

### 步骤4：验证数据返回
打开浏览器开发者工具（F12），查看网络请求中出库列表接口的响应数据是否包含 `display_unit_type`、`display_original_qty` 等字段。

## 预期效果

修改生效后应该显示：

| 出库单号 | ... | 总数量 | ... |
|---------|-----|--------|-----|
| CK20260504008 | ... | **1盒(10支)** | ... |
| CK20260504004 | ... | **20支** | ... |

## 如果仍然不显示

可能是因为历史数据的 `original_quantity` 字段为空（在添加字段之前创建的数据）。

需要在后端做兼容处理：
```php
$stockOut->display_original_qty = $firstItem->original_quantity ?? $firstItem->quantity;
```
