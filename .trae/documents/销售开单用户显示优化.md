# 销售开单页面用户显示优化计划

## 📋 任务目标
将销售开单页面中所有涉及用户显示的地方统一使用 **真实姓名(RealName)**，而不是用户名(username)。

## 🔍 现状分析

### 当前代码情况
当前代码已经使用了优先级显示逻辑：`u.realName || u.nickName || u.userName`

### 需要修改的位置

#### 1️⃣ 用户选择下拉框（4处）
| 位置 | 行号 | 用途 | 当前代码 |
|------|------|------|----------|
| 持卡操作表单 | L234 | 操作人选择 | `:label="u.realName \|\| u.nickName \|\| u.userName"` |
| 体验操作表单 | L296 | 操作人选择 | `:label="u.realName \|\| u.nickName \|\| u.userName"` |

## 🛠️ 实施步骤

### 步骤 1：修改下拉框显示标签
将所有 `el-option` 的 `:label` 属性统一改为只使用 `realName`：
```vue
<!-- 修改前 -->
<el-option :label="u.realName || u.nickName || u.userName" />

<!-- 修改后 -->
<el-option :label="u.realName || '未设置姓名'" />
```

- [index.vue#L464](file:///d:/fuchenpro/front/src/views/business/sales/index.vue#L464)

### 步骤 2：确认后端数据字段
检查后端 API 返回的用户数据是否包含 `realName` 字段：
- 操作记录接口 `/business/operation/list` 返回的 `operatorUserName`
- 还款记录接口 `/business/repayment/list` 返回的 `creatorUserName`

如果后端存储的是 userName，需要同步修改后端逻辑，确保返回 realName。

### 步骤 3：修改 JavaScript 赋值逻辑
统一所有获取用户姓名的代码：

```javascript
// 修改前
operationForm.value.operatorUserName = userStore.realName || userStore.name

// 修改后
operationForm.value.operatorUserName = userStore.realName || ''
```

## 📝 注意事项
1. 需要确保 `realName` 字段在后端用户数据中存在且不为空
2. 如果某些用户的 `realName` 为空，可以显示占位符如"未设置姓名"
3. 后端保存操作人和创建人时也应统一使用 `real_name` 字段

## ✅ 预期效果
- 所有用户选择下拉框只显示真实姓名
- 所有表格中的操作人/创建人列显示真实姓名
- 提交数据时统一传递真实姓名字段
