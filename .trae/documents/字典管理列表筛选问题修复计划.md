# 字典管理列表筛选问题修复计划

## 📋 问题描述

在字典管理页面，点击"列表"按钮后，字典数据列表没有根据字典名称和类型进行筛选，而是显示了所有数据。

---

## 🔍 问题分析

### 问题定位

经过代码分析，发现问题出在**前后端参数命名风格不一致**：

#### 前端代码（data.vue:210-216）
```javascript
queryParams: {
  pageNum: 1,
  pageSize: 10,
  dictType: undefined,    // 驼峰命名
  dictLabel: undefined,
  status: undefined
}
```

#### 后端代码（SysDictDataService.php:13-15）
```php
if (!empty($params['dict_type'])) {  // 期望下划线命名
    $query->where('dict_type', $params['dict_type']);
}
```

### 问题原因

1. **前端**：使用驼峰命名 `dictType` 发送请求参数
2. **后端**：期望接收下划线命名 `dict_type` 参数
3. **参数转换缺失**：
   - 前端的 `tansParams` 函数只进行 URL 编码，不进行命名风格转换
   - 后端的 `list` 方法没有调用 `convert_to_snake_case` 函数进行参数转换
   - 后端的 `add` 和 `edit` 方法有参数转换，但 `list` 方法遗漏了

### 数据流程

```
用户点击"列表"按钮
  ↓
打开页面: /system/dict-data/index/{dictId}
  ↓
调用 getTypes(dictId) 获取字典类型
  ↓
设置 queryParams.dictType = response.data.dictType
  ↓
调用 listData(queryParams) 发送请求
  ↓
前端发送: ?dictType=xxx (驼峰)
  ↓
后端接收: $params['dictType'] (驼峰)
  ↓
后端查询: if (!empty($params['dict_type'])) (下划线) ❌ 条件不成立
  ↓
结果: 没有进行筛选，返回所有数据
```

---

## 🎯 解决方案

### 方案选择

**推荐方案：后端添加参数转换**

理由：
- ✅ 前端使用驼峰命名符合 JavaScript 规范
- ✅ 后端已有 `convert_to_snake_case` 函数，说明项目有此需求
- ✅ 保持代码风格统一，其他 Controller 的 `list` 方法也应该添加
- ✅ 修改范围小，只需修改后端一个方法

### 修复步骤

#### 步骤1：修改后端 SysDictDataController.php

**文件位置**：`webman/app/controller/system/SysDictDataController.php`

**修改内容**：在 `list` 方法中添加参数转换

```php
public function list(Request $request)
{
    $service = new SysDictDataService();
    $params = convert_to_snake_case($request->all());  // 添加参数转换
    $result = $service->selectDictDataList($params);
    return TableDataInfo::result($result->items(), $result->total());
}
```

#### 步骤2：检查其他 Controller 的 list 方法

需要检查以下 Controller 是否存在相同问题：

如果发现相同问题，一并修复。

## 🔧 技术细节

### convert_to_snake_case 函数说明

该函数位于 `webman/app/functions.php`，用于将驼峰命名转换为下划线命名：

```php
function convert_to_snake_case($data) {
    // 将 camelCase 转换为 snake_case
    // 例如: dictType -> dict_type
}
```

### 影响范围

- **直接影响**：字典数据列表查询功能
- **间接影响**：所有使用驼峰命名参数的列表查询接口
- **风险评估**：低风险，只是添加参数转换，不影响现有逻辑
