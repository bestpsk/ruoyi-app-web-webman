# 还欠款功能实现计划

## 一、需求分析

在"操作记录"tab右侧添加一个"还欠款"tab页面，实现以下功能：
1. 显示当前客户的欠款订单/套餐列表
2. 选择欠款订单进行还款操作
3. 在订单管理中生成还款订单记录

## 二、现有系统分析

### 2.1 相关数据结构
- **订单表 `biz_sales_order`**：包含 `owed_amount`（欠款金额）字段
- **客户套餐表 `biz_customer_package`**：包含 `owed_amount`（欠款金额）字段
- **套餐项目表 `biz_package_item`**：包含 `owed_amount`（欠款金额）字段

### 2.2 Tab页面位置
- 文件：`d:\fuchenpro\AppV3\src\pages\business\sales\order.vue`
- 现有Tab：开单(0)、开单记录(1)、操作记录(2)
- 新增Tab：还欠款(3)

## 三、实现方案

### 3.1 数据库设计

#### 3.1.1 新建还款记录表 `biz_repayment_record`
```sql
CREATE TABLE `biz_repayment_record` (
  `repayment_id` bigint NOT NULL AUTO_INCREMENT COMMENT '还款ID',
  `repayment_no` varchar(50) DEFAULT NULL COMMENT '还款编号',
  `customer_id` bigint DEFAULT NULL COMMENT '客户ID',
  `customer_name` varchar(100) DEFAULT NULL COMMENT '客户名称',
  `package_id` bigint DEFAULT NULL COMMENT '套餐ID（还款来源）',
  `package_no` varchar(50) DEFAULT NULL COMMENT '套餐编号',
  `order_id` bigint DEFAULT NULL COMMENT '原订单ID',
  `order_no` varchar(50) DEFAULT NULL COMMENT '原订单编号',
  `repayment_amount` decimal(12,2) DEFAULT 0.00 COMMENT '还款金额',
  `repayment_type` char(1) DEFAULT '1' COMMENT '还款类型：1-套餐还款 2-订单还款',
  `payment_method` varchar(50) DEFAULT NULL COMMENT '支付方式',
  `status` char(1) DEFAULT '0' COMMENT '状态：0-待审核 1-已审核 2-已取消',
  `remark` varchar(500) DEFAULT NULL COMMENT '备注',
  `enterprise_id` bigint DEFAULT NULL COMMENT '企业ID',
  `store_id` bigint DEFAULT NULL COMMENT '门店ID',
  `create_by` varchar(64) DEFAULT NULL COMMENT '创建者',
  `create_time` datetime DEFAULT NULL COMMENT '创建时间',
  `update_by` varchar(64) DEFAULT NULL COMMENT '更新者',
  `update_time` datetime DEFAULT NULL COMMENT '更新时间',
  `audit_by` varchar(64) DEFAULT NULL COMMENT '审核人',
  `audit_time` datetime DEFAULT NULL COMMENT '审核时间',
  PRIMARY KEY (`repayment_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='还款记录表';
```

## 六、业务流程

```
1. 用户进入客户详情页 → 点击"还欠款"Tab
2. 系统加载该客户的欠款套餐列表（owed_amount > 0）
3. 用户选择要还款的套餐，输入还款金额
4. 提交还款请求
5. 后端创建还款记录，更新套餐欠款金额
6. 前端刷新欠款列表，显示还款成功
```

## 七、注意事项

1. **数据一致性**：还款后需同步更新套餐表和套餐项目表的欠款金额
2. **权限控制**：还款操作需要相应的权限验证
3. **金额校验**：还款金额不能超过欠款金额
4. **审核流程**：还款记录可能需要审核后才生效（根据业务需求）
