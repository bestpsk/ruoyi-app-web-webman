# 入库管理表单数据展示与列宽优化计划

## 问题分析

### 1. 数据显示数字问题
- **问题描述**：入库明细表格中的"规格"和"单位"列显示的是数字ID而不是中文标签
- **原因分析**：
  - 后端 `BizStockInService.php:50-51` 返回的 `spec` 和 `unit` 字段是数据库存储的原始值（字典ID）
  - 前端 `index.vue:219-229` 的 `getUnitLabel()` 和 `getSpecLabel()` 函数依赖字典数据转换
  - 如果字典数据未加载或匹配失败，会显示原始数字

### 2. 列宽不平衡问题
- **问题描述**：货品列太宽（min-width="180"），其他列偏窄
- **当前列宽配置**：
  - 货品：180px（太宽）
  - 规格：70px
  - 单位类型：100px
  - 换算：95px
  - 数量：90px
  - 进货单价：110px
  - 金额：85px
  - 备注：100px
  - 生产日期：130px
  - 有效期至：115px
  - 操作：55px

## 解决方案

### 步骤1：修复字典数据显示问题

#### 1.1 验证字典数据加载
- 确认前端是否正确引入了字典：`biz_product_unit` 和 `biz_product_spec`
- 当前代码已引入（第192行）：`useDict("biz_stock_in_type", "biz_doc_status", "biz_product_unit", "biz_product_spec")`

#### 1.2 增强字典查找函数
- 修改 `getUnitLabel()` 和 `getSpecLabel()` 函数，增加容错处理
- 当字典未找到时，尝试直接显示值或显示"未知"

#### 1.3 后端可选优化（如果前端方案不行）
- 在后端查询时，通过JOIN字典表或手动查询字典表，将ID转换为名称
- 或者在Service层添加字典转换逻辑

### 步骤2：优化表格列宽布局

#### 2.1 列宽重新分配原则
- **货品列**：缩小到150px，使用min-width自适应
- **规格/单位列**：保持70-80px，这些是短文本
- **数值列**（数量、单价、金额）：适当加宽以容纳数字
- **日期列**：保持120-130px
- **操作列**：保持固定宽度

#### 2.2 新的列宽配置方案
```
货品：150px（min-width）- 缩小30px
规格：75px - 加5px
单位类型：90px - 减10px
换算：100px - 加5px
数量：95px - 加5px
进货单价：115px - 加5px
金额：95px - 加10px
备注：110px - 加10px
生产日期：130px - 保持不变
有效期至：120px - 加5px
操作：55px - 保持不变
```

### 步骤3：测试验证

#### 3.1 功能测试
- 打开新增入库单，添加货品明细
- 验证规格和单位是否正确显示中文标签
- 编辑已有入库单，确认数据回显正常

#### 3.2 UI测试
- 在不同屏幕分辨率下测试表格显示效果
- 确认所有列的内容都能完整显示，无截断或过度空白
