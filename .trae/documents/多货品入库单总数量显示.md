# 多货品入库单 — 总数量显示方案

## 当前问题

当入库单包含**多个不同规格的货品**时，总数量列显示 `2盒（20支）` 或 `10盒（100支）`，但这只反映了**第一个货品**的信息，不能代表整张入库单的总数量。

## 主流进销存系统的做法

### 方案对比

| 方案 | 显示方式 | 优点 | 缺点 | 常见系统 |
|------|----------|------|------|----------|
| **A. 纯数字汇总** | `100` | 简洁 | 无单位，不直观 | 用友、金蝶 |
| **B. 最小单位汇总** | `100支` | 统一单位 | 大数字不易读 | 管家婆 |
| **C. 货品种类数** | `2种货品` | 一目了然 | 不显示具体数量 | 自定义 |
| **D. 第一个货品+等N项** | `GCS-p7等2项` | 信息丰富 | 占空间 | 旺店通 |
| **E. 展开行查看明细** | 点击展开 | 完整信息 | 需要操作 | SAP |

### 推荐方案：方案C + 展开行

**总数量列改为显示"货品种类数 + 最小单位汇总"**

```
入库单号    总数量              总金额     操作
RK001      2种 / 120支         51800.00   查看
RK002      1种 / 10支          2580.00    查看

点击展开后：
RK001
├── GCS-p7:   1盒(10支) × 2580/盒 = 25800
└── 测试1:    10支 × 2600/支 = 26000
```

## 实施步骤

### 步骤1：修改 formatQuantity 方法

```javascript
function formatQuantity(row) {
  const items = row.items || []
  
  if (items.length === 0) {
    return row.totalQuantity || 0
  }
  
  if (items.length === 1) {
    // 单个货品：按原有逻辑显示
    const item = items[0]
    const packQty = row.display_pack_qty || item.packQty || 1
    const unitLabel = getUnitLabel(row.display_unit || item.unit)
    const specLabel = getSpecLabel(row.display_spec || item.spec)
    const totalQty = row.totalQuantity || 0
    if (packQty > 1 && specLabel) {
      const mainQty = totalQty / packQty
      const mainQtyStr = Number.isInteger(mainQty) ? mainQty : mainQty.toFixed(1).replace(/\.0$/, '')
      return mainQtyStr + unitLabel + '（' + totalQty + specLabel + '）'
    } else if (unitLabel) {
      return totalQty + unitLabel
    } else {
      return totalQty
    }
  }
  
  // 多个货品：显示种类数 + 最小单位汇总
  return items.length + '种 / ' + (row.totalQuantity || 0) + getSpecLabel(row.display_spec || items[0].spec)
}
```
