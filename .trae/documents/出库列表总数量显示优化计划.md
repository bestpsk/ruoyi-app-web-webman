# 出库管理列表总数量显示优化计划

## 需求分析

### 当前问题
- 列表只显示数字：`10`、`1`、`20`
- 用户无法区分是主单位还是副单位
- 容易误解数量含义

### 目标效果
根据出库时选择的单位类型，直观显示：
| 场景 | unitType | original_quantity | quantity | 显示 |
|------|----------|-------------------|----------|------|
| 出库1盒(10支) | 1 | 1 | 10 | **1盒(10支)** |
| 出库20支 | 2 | 20 | 20 | **20支** |

## 技术方案

### 数据来源
明细表 `biz_stock_out_item` 已有字段：
- `unit_type` - 用户选择的单位类型 (1=主单位, 2=副单位)
- `original_quantity` - 用户输入的原始数量
- `quantity` - 换算后的最小单位数量
- `pack_qty` - 换算比例
- `spec` - 规格(副单位)
- `unit` - 单位(主单位)

### 实现步骤

### 步骤1：修改后端列表接口，返回单位信息

文件：`d:\fuchenpro\webman\app\service\BizStockOutService.php`

在 `listStockOuts` 方法中，关联查询第一条明细获取单位信息：

```php
public function listStockOuts($params = [])
{
    // ... 现有查询逻辑 ...
    
    $list = $query->paginate($params['pageSize'] ?? 10, ['*'], 'page', $params['pageNum'] ?? 1);
    
    foreach ($list->items() as $stockOut) {
        $firstItem = BizStockOutItem::where('stock_out_id', $stockOut->stock_out_id)->first();
        if ($firstItem) {
            $stockOut->display_unit_type = $firstItem->unit_type ?? '1';
            $stockOut->display_original_qty = $firstItem->original_quantity;
            $stockOut->display_pack_qty = $firstItem->pack_qty ?? 1;
            $stockOut->display_unit = $firstItem->unit;
            $stockOut->display_spec = $firstItem->spec;
        }
    }
    
    return $list;
}
```

### 步骤2：修改前端列表显示

文件：`d:\fuchenpro\front\src\views\wms\stockOut\index.vue`

将"总数量(最小单位)"列改为自定义模板：

```html
<el-table-column label="总数量" width="130" align="center">
  <template #default="scope">
    <span v-if="scope.row.display_unit_type === '1' && scope.row.display_pack_qty > 1">
      {{ scope.row.display_original_qty }}{{ getUnitLabel(scope.row.display_unit) }}({{ scope.row.totalQuantity }}{{ getSpecLabel(scope.row.display_spec) }})
    </span>
    <span v-else>
      {{ scope.row.totalQuantity }}{{ getSpecLabel(scope.row.display_spec) }}
    </span>
  </template>
</el-table-column>
```
