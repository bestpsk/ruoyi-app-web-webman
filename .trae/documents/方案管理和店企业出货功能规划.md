# 方案管理和店企业出货功能规划（详细版）

## 业务流程

```
开方案(草稿) → 修改方案(草稿状态可改) → 审核方案(锁定不可改) → 新建出货单(客服) → 审核出货单(专人) → 配货发货(物流) → 确认收货(客服)
```

### 状态流转

**方案状态**：
```
草稿(0) → 待审核(1) → 已审核(2) → 已完成(3)
                ↓
            已驳回(4)
```
- 草稿：可修改、可删除
- 待审核：提交审核后，不可修改
- 已审核：审核通过，锁定不可修改，可创建出货单
- 已完成：所有配赠金额已出完
- 已驳回：可修改后重新提交

**出货单状态**：
```
待审核(0) → 已审核(1) → 已发货(2) → 已收货(3)
                ↓
            已驳回(4)
```
- 待审核：默认状态，可修改/删除
- 已审核：审核通过，物流可操作
- 已发货：物流已发出
- 已收货：企业确认收货，扣减库存，更新方案剩余金额

## 数据库表设计

### 1. biz_plan（方案表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| plan_id | bigint AUTO_INCREMENT | 方案ID (主键) |
| plan_no | varchar(30) UNIQUE | 方案编号 (自动生成，如PL20260506001) |
| enterprise_id | bigint NOT NULL | 企业ID |
| plan_name | varchar(100) NOT NULL | 方案名称 (默认：XX企业X%方案) |
| commission_rate | decimal(5,2) DEFAULT 0 | 分成比例(%) |
| plan_amount | decimal(12,2) DEFAULT 0 | 方案金额(店家付款) |
| gift_amount | decimal(12,2) DEFAULT 0 | 配赠金额 |
| shipped_amount | decimal(12,2) DEFAULT 0 | 已出金额(已收货的出货单总金额) |
| remaining_amount | decimal(12,2) DEFAULT 0 | 剩余金额(配赠金额-已出金额) |
| effective_date | date DEFAULT NULL | 生效日期 |
| expiry_date | date DEFAULT NULL | 失效日期 |
| audit_status | char(1) DEFAULT '0' | 审核状态(0草稿 1待审核 2已审核 3已完成 4已驳回) |
| audit_by | varchar(64) DEFAULT NULL | 审核人 |
| audit_time | datetime DEFAULT NULL | 审核时间 |
| audit_remark | varchar(500) DEFAULT NULL | 审核备注(驳回原因等) |
| status | char(1) DEFAULT '0' | 启用状态(0启用 1停用) |
| remark | text DEFAULT NULL | 备注 |
| create_by | varchar(64) DEFAULT NULL | 创建者 |
| create_time | datetime DEFAULT NULL | 创建时间 |
| update_by | varchar(64) DEFAULT NULL | 更新者 |
| update_time | datetime DEFAULT NULL | 更新时间 |

**关键约束**：
- `audit_status` 为 2(已审核) 或 3(已完成) 时，方案不可修改
- `remaining_amount` = `gift_amount` - `shipped_amount`，动态计算
- 方案到期自动标记为已完成（可由定时任务处理）

### 2. biz_plan_item（方案配赠明细表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| item_id | bigint AUTO_INCREMENT | 明细ID (主键) |
| plan_id | bigint NOT NULL | 方案ID |
| product_id | bigint DEFAULT NULL | 货品ID (可为空，因为有些方案只有总金额没有明细) |
| product_name | varchar(100) DEFAULT NULL | 货品名称 |
| supplier_id | bigint DEFAULT NULL | 供货商ID |
| supplier_name | varchar(100) DEFAULT NULL | 供货商名称 |
| unit_type | char(1) DEFAULT '1' | 单位类型(1主单位整 2副单位拆) |
| pack_qty | int DEFAULT 1 | 换算比例(1主单位=N副单位) |
| quantity | int DEFAULT 0 | 数量(最小单位) |
| spec | varchar(20) DEFAULT NULL | 规格 |
| sale_price | decimal(10,2) DEFAULT 0 | 单价(取自货品出货价格) |
| amount | decimal(12,2) DEFAULT 0 | 总金额(单价×数量) |
| shipped_quantity | int DEFAULT 0 | 已出数量(已收货的出货单中该货品数量之和) |
| remaining_quantity | int DEFAULT 0 | 剩余数量(数量-已出数量) |
| remark | varchar(500) DEFAULT NULL | 备注 |

**说明**：
- `product_id` 可为空，有些方案只有总金额没有具体货品明细
- `shipped_quantity` 和 `remaining_quantity` 用于控制出货数量不超过方案数量
- 出货时校验：出货数量 ≤ remaining_quantity（如果有明细）

### 3. biz_shipment（出货单表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| shipment_id | bigint AUTO_INCREMENT | 出货单ID (主键) |
| shipment_no | varchar(30) UNIQUE | 出货单号 (自动生成，如SH20260506001) |
| plan_id | bigint NOT NULL | 关联方案ID |
| enterprise_id | bigint NOT NULL | 企业ID |
| enterprise_name | varchar(100) DEFAULT NULL | 企业名称 |
| contact_person | varchar(50) DEFAULT NULL | 收货人 |
| contact_phone | varchar(20) DEFAULT NULL | 收货电话 |
| shipping_address | varchar(255) DEFAULT NULL | 收货地址 |
| total_quantity | int DEFAULT 0 | 总数量 |
| total_amount | decimal(12,2) DEFAULT 0 | 总金额(折扣后) |
| logistics_company | varchar(50) DEFAULT NULL | 物流公司 |
| logistics_no | varchar(50) DEFAULT NULL | 物流单号 |
| shipment_status | char(1) DEFAULT '0' | 出货状态(0待审核 1已审核 2已发货 3已收货 4已驳回) |
| shipment_date | date DEFAULT NULL | 发货日期 |
| receipt_date | date DEFAULT NULL | 收货日期 |
| audit_by | varchar(64) DEFAULT NULL | 审核人 |
| audit_time | datetime DEFAULT NULL | 审核时间 |
| remark | text DEFAULT NULL | 备注 |
| create_by | varchar(64) DEFAULT NULL | 创建者 |
| create_time | datetime DEFAULT NULL | 创建时间 |
| update_by | varchar(64) DEFAULT NULL | 更新者 |
| update_time | datetime DEFAULT NULL | 更新时间 |

**关键约束**：
- 出货单总金额 ≤ 方案剩余金额
- 出货单中每个货品数量 ≤ 方案明细中该货品剩余数量（如果有明细）
- `shipment_status` 为 0(待审核) 时可修改/删除
- `shipment_status` 为 1(已审核) 后不可修改
- `shipment_status` 变为 3(已收货) 时，扣减库存并更新方案已出金额

### 4. biz_shipment_item（出货明细表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| item_id | bigint AUTO_INCREMENT | 明细ID (主键) |
| shipment_id | bigint NOT NULL | 出货单ID |
| plan_item_id | bigint DEFAULT NULL | 关联方案明细ID (可为空) |
| product_id | bigint DEFAULT NULL | 货品ID |
| product_name | varchar(100) DEFAULT NULL | 货品名称 |
| supplier_id | bigint DEFAULT NULL | 供货商ID |
| supplier_name | varchar(100) DEFAULT NULL | 供货商名称 |
| unit_type | char(1) DEFAULT '1' | 单位类型(1主单位 2副单位) |
| pack_qty | int DEFAULT 1 | 换算比例 |
| quantity | int DEFAULT 0 | 数量(最小单位) |
| spec | varchar(20) DEFAULT NULL | 规格 |
| sale_price | decimal(10,2) DEFAULT 0 | 单价(取自货品出货价格) |
| discount_price | decimal(10,2) DEFAULT 0 | 折扣单价(默认=单价，可手动调整) |
| amount | decimal(12,2) DEFAULT 0 | 总金额(折扣单价×数量) |
| remark | varchar(500) DEFAULT NULL | 备注 |

**折扣单价说明**：
- 有些方案只有总金额没有每个货品明细，出货时需要按折扣价计算
- `discount_price` 默认等于 `sale_price`
- 可手动调整折扣单价，用于分摊方案总金额
- `amount` = `discount_price` × `quantity`

## 额外考虑的细节

### 1. 方案编号自动生成规则
格式：`PL` + 年月日 + 3位序号，如 PL20260506001

### 2. 出货单号自动生成规则
格式：`SH` + 年月日 + 3位序号，如 SH20260506001

### 3. 方案到期处理
- 方案失效日期到期后，如果有未出完的金额，方案自动标记为已完成
- 可通过定时任务每天检查

### 4. 出货单取消/驳回
- 驳回后可编辑重新提交
- 已审核的出货单不可取消（需联系管理员）

### 5. 库存扣减时机
- 出货单确认收货时扣减库存（不是发货时）
- 这样如果企业拒收，库存不会受影响

### 6. 方案剩余金额更新时机
- 出货单确认收货时，更新方案的 `shipped_amount` 和 `remaining_amount`
- 同时更新方案明细的 `shipped_quantity` 和 `remaining_quantity`

### 7. 企业停用后的处理
- 企业停用后，已有的方案和出货单不受影响
- 但不能为停用的企业开新方案

### 8. 权限控制
- 方案管理：查看、新增、编辑、删除、提交审核、审核
- 出货单：查看、新增、编辑、删除、审核、发货、确认收货
- 店企业出货：查看、审核、填写物流、确认发货、确认收货
