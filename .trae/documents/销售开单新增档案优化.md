# 销售开单 - 新增档案优化计划

## 📋 需求分析

用户对新增档案功能提出以下优化需求：

1. **操作人默认显示登录用户的姓名realName** - 已实现，需确认
2. **增加档案类型字段** - 加入字典：铺垫、开方案、销售、售后、回访
3. **不添加项目也能生成档案** - 移除必须添加品项的验证

---

## 🛠️ 实施步骤

### 第1步：数据库修改
在 `biz_customer_archive` 表添加 `archive_type` 字段：
```sql
ALTER TABLE `biz_customer_archive` ADD COLUMN `archive_type` varchar(50) DEFAULT NULL COMMENT '档案类型(铺垫/开方案/销售/售后/回访)' AFTER `archive_date`;
```

### 第2步：添加字典数据
插入字典类型和字典数据：
```sql
-- 字典类型
INSERT INTO `sys_dict_type` (`dict_name`, `dict_type`, `status`, `create_by`, `create_time`, `remark`) 
VALUES ('档案类型', 'biz_archive_type', '0', 'admin', NOW(), '客户档案类型');

-- 字典数据
INSERT INTO `sys_dict_data` (`dict_type`, `dict_label`, `dict_value`, `dict_sort`, `status`, `create_by`, `create_time`) VALUES
('biz_archive_type', '铺垫', 'preparation', 1, '0', 'admin', NOW()),
('biz_archive_type', '开方案', 'plan', 2, '0', 'admin', NOW()),
('biz_archive_type', '销售', 'sales', 3, '0', 'admin', NOW()),
('biz_archive_type', '售后', 'after_sales', 4, '0', 'admin', NOW()),
('biz_archive_type', '回访', 'follow_up', 5, '0', 'admin', NOW());
```

### 第3步：后端 Model 修改
修改 `BizCustomerArchive.php`，在 `$fillable` 数组中添加 `archive_type`。

### 第4步：前端修改

#### 4.1 引入字典
在 `useDict` 中添加 `biz_archive_type`。

#### 4.2 修改 `handleAddArchive` 函数
添加 `archiveType: 'sales'` 默认值（默认选择"销售"）。

#### 4.3 修改 `submitArchiveForm` 函数
移除验证：`if (validItems.length === 0) return proxy.$modal.msgWarning('请至少添加一个品项')`
改为：如果没有品项，`planItems` 设为空数组 `[]`。

#### 4.4 修改新增档案弹窗模板
在档案日期和操作人之间添加档案类型下拉框：
```vue
<el-form-item label="档案类型">
  <el-select v-model="archiveForm.archiveType" placeholder="请选择档案类型" style="width: 100%">
    <el-option v-for="dict in biz_archive_type" :key="dict.value" :label="dict.label" :value="dict.value" />
  </el-select>
</el-form-item>
```

#### 4.5 修改档案列表展示
在档案卡片头部添加档案类型标签显示。

## ✅ 预期效果

1. 新增档案时，操作人默认显示当前登录用户的真实姓名
2. 新增档案时，可以选择档案类型（铺垫、开方案、销售、售后、回访），默认"销售"
3. 不添加品项也能成功创建档案
4. 档案列表中显示档案类型标签
