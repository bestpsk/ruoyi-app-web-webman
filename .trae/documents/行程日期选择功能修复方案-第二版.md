# 行程日期选择功能修复方案（第二版）

## 🚨 问题根因确认

经过深入分析 uview-plus (v3.7.36) 的源码，发现 **两个关键错误**：

### 错误1：mode 属性值错误

**当前代码**：
```vue
<u-calendar mode="multi" />
```

**正确值**：
```vue
<u-calendar mode="multiple" />
```

**证据来源**：[props.js#L21-L25](file:///d:/fuchenpro/AppV3/node_modules/uview-plus/components/u-calendar/props.js#L21-L25)
```javascript
// 日期类型选择，single-选择单个日期，multiple-可以选择多个日期，range-选择日期范围
mode: {
  type: String,
  default: () => defProps.calendar.mode
}
```

> ⚠️ **u-calendar 的多选模式属性值是 `'multiple'` 而非 `'multi'`！**

---

### 错误2：confirm 回调参数格式理解错误

**当前代码假设**：
```javascript
// 假设 e 是对象数组
const [startDate, endDate] = e
form.startDate = `${startDate.year}-${startDate.month.padStart(2,'0')}-${startDate.day.padStart(2,'0')}`
```

**实际数据结构**（来自源码 [month.vue#L400-L408](file:///d:/fuchenpro/AppV3/node_modules/uview-plus/components/u-calendar/month.vue#L400-L408)）：

```javascript
} else if (this.mode === 'multiple') {
  if (selected.some(item => this.dateSame(item, date))) {
    // 反选移除
    const itemIndex = selected.findIndex(item => item === date)
    selected.splice(itemIndex, 1)
  } else {
    // 添加到数组
    if (selected.length < this.maxCount) selected.push(date)  // ← date 是字符串！
  }
}
```

**confirm事件返回值**（[u-calendar.vue#L268](file:///d:/fuchenpro/AppV3/node_modules/uview-plus/components/u-calendar/u-calendar.vue#L268)）：
```javascript
confirm() {
  if (!this.buttonDisabled) {
    this.$emit('confirm', this.selected)  // ← 返回的是 selected 数组
  }
}
```

**实际返回格式**：
```javascript
e = ["2026-05-02", "2026-05-15", "2026-06-01"]  // 字符串数组！
// 而非 [{year: 2026, month: 5, day: 2}, ...]
```

> ⚠️ **multiple 模式下 confirm 返回的是日期字符串数组，不是对象数组！**

---

```vue
<!-- 错误 -->
```

### 修改2：重写 onMultiDateConfirm 函数

**文件位置**：[form.vue#L208-L223](file:///d:/fuchenpro/AppV3/src/pages/business/schedule/form.vue#L208-L223)

**错误代码**：
```javascript
function onMultiDateConfirm(e) {
  console.log('[Calendar] 多选结果:', e)
  
  if (Array.isArray(e)) {
    form.selectedDates = e.map(date => 
      `${date.year}-${String(date.month).padStart(2, '0')}-${String(date.day).padStart(2, '0')}`
      // ❌ 错误：date是字符串，没有 .year, .month, .day 属性
    ).sort()
    
    if (form.selectedDates.length > 0) {
      form.startDate = form.selectedDates[0]
      form.endDate = form.selectedDates[form.selectedDates.length - 1]
    }
  }
  
  showCalendarPicker.value = false
}
```

**正确代码**：
```javascript
function onMultiDateConfirm(e) {
  console.log('[Calendar] 多选结果:', e)
  console.log('[Calendar] 类型:', typeof e, Array.isArray(e))
  
  if (Array.isArray(e) && e.length > 0) {
    // multiple 模式下 e 已经是 ["YYYY-MM-DD", ...] 格式的字符串数组
    // 无需再转换，直接使用
    form.selectedDates = [...e].sort()
    
    // 兼容旧字段
    form.startDate = form.selectedDates[0]
    form.endDate = form.selectedDates[form.selectedDates.length - 1]
    
    console.log('[Calendar] 选中的日期:', form.selectedDates)
  } else {
    console.warn('[Calendar] 未选择任何日期或格式异常')
  }
  
  showCalendarPicker.value = false
}
```

**关键改进**：
- ✅ 不再尝试访问 `.year`, `.month`, `.day`
- ✅ 直接使用返回的字符串数组
- ✅ 添加详细的调试日志
- ✅ 添加空数组和异常情况的处理

## ⚠️ 其他可能的问题排查

如果修改后仍然有问题，请检查以下几点：

### 1. 组件版本兼容性
- 确认 uview-plus 版本 ≥ 3.0（支持 multiple 模式）
- 当前项目版本：**3.7.36** ✅ 符合要求

### 2. maxCount 配置（可选）
multiple 模式下可以设置最大可选数量：
```vue
<u-calendar 
  mode="multiple"
  :maxCount="31"  <!-- 最多选择31天 -->
/>
```

### 3. defaultDate 初始值（可选）
编辑模式下可以设置默认选中日期：
```vue
<u-calendar 
  mode="multiple"
  :defaultDate="['2026-05-02', '2026-05-15']"
/>
```
