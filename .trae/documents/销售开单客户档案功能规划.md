# 销售开单 - 客户档案功能规划

## 📋 需求分析

用户希望为每个客户建立一个**统一的档案记录**，将开单、操作、还款等所有活动汇总在一个地方，方便管理。同时支持**手动新增档案记录**。

### 核心诉求
1. **档案Tab**：汇总展示客户所有活动记录（开单、操作、还款），按时间线排列
2. **手动新增档案**：可以手动添加一条档案记录，包含时间、类型、方案、金额、满意度、照片、顾客反馈、备注
3. **统一管理**：一个地方看到客户所有历史，不用来回切换Tab

## 🏗️ 设计方案

### 方案选择：**新建 `biz_customer_archive` 表**

**为什么不用现有表聚合查询？**
- 现有三张表（订单、操作、还款）字段差异大，聚合查询复杂且性能差
- 用户需要"手动新增档案"功能，现有表无法承载自定义档案
- 档案表可以同时存储"自动同步的记录"和"手动新增的记录"

**设计思路**：
- 档案表作为**统一视图**，记录客户所有活动
- 开单、操作、还款时**自动写入档案表**（source_type + source_id 关联原始记录）
- 同时支持**手动新增**自定义档案记录（source_type = 'manual'）
- 档案Tab直接查档案表，一次查询展示所有记录

## 📊 数据库设计

### 新表：`biz_customer_archive`（客户档案表）

| 字段 | 类型 | 说明 |
|------|------|------|
| archive_id | bigint PK | 档案ID |
| customer_id | bigint | 客户ID |
| customer_name | varchar(50) | 客户姓名 |
| enterprise_id | bigint | 企业ID |
| store_id | bigint | 门店ID |
| archive_date | date | 档案日期 |
| source_type | char(1) | 来源类型：0-开单 1-操作 2-还款 3-手动新增 |
| source_id | bigint | 来源记录ID（关联原始表，手动新增为null） |
| plan_items | text | 方案项目（JSON格式：[{name:"项目名",quantity:3}]） |
| amount | decimal(12,2) | 金额 |
| satisfaction | tinyint | 满意度(1-5星) |
| photos | text | 照片（JSON数组，多张图片URL） |
| customer_feedback | varchar(500) | 顾客反馈 |
| operator_user_id | bigint | 操作人ID |
| operator_user_name | varchar(50) | 操作人姓名 |
| remark | text | 备注 |
| create_by | varchar(64) | 创建者 |
| create_time | datetime | 创建时间 |

### 索引
- `idx_customer_id` (customer_id)
- `idx_archive_date` (archive_date)
- `idx_source_type` (source_type)
- `idx_enterprise_id` (enterprise_id)

## 🔧 实施步骤

### 第1步：创建数据库表
- 编写SQL创建 `biz_customer_archive` 表
- 执行SQL

### 第2步：后端 - Model
- 创建 `BizCustomerArchive.php` 模型

### 第3步：后端 - Service
- 创建 `BizCustomerArchiveService.php`，包含：
  - `selectArchiveList($params)` - 按客户查询档案列表（支持类型筛选、日期范围）
  - `insertArchive($data)` - 手动新增档案
  - `deleteArchive($archiveIds)` - 删除档案
  - `insertArchiveFromOrder($order)` - 开单时自动写入档案
  - `insertArchiveFromOperation($record)` - 操作时自动写入档案
  - `insertArchiveFromRepayment($repayment)` - 还款时自动写入档案

### 第4步：后端 - Controller
- 创建 `BizCustomerArchiveController.php`，包含：
  - `list` - 档案列表
  - `add` - 新增档案
  - `remove` - 删除档案

### 第5步：后端 - 路由
- 在 `route.php` 添加档案相关路由

### 第6步：后端 - 自动同步
- 修改 `BizSalesOrderService.insertOrder` → 开单成功后调用 `insertArchiveFromOrder`
- 修改 `BizOperationRecordService.insertRecord` → 操作成功后调用 `insertArchiveFromOperation`
- 修改 `BizRepaymentService.insertRepayment` → 还款成功后调用 `insertArchiveFromRepayment`

### 第7步：前端 - API
- 创建 `customerArchive.js`，包含：
  - `listArchive(params)` - 档案列表
  - `addArchive(data)` - 新增档案
  - `deleteArchive(archiveIds)` - 删除档案

### 第8步：前端 - 档案Tab
- 在 `sales/index.vue` 添加档案Tab
- 实现档案列表展示（时间线样式）
- 实现筛选功能（类型、日期范围）
- 实现新增档案弹窗
- 点击客户时加载档案数据

### 第9步：历史数据迁移（可选）

## 📝 关键细节

### plan_items JSON格式
```json
[
  {"name": "面部护理", "quantity": 3},
  {"name": "身体SPA", "quantity": 2}
]
```

### source_type 映射
| 值 | 含义 | source_id 关联 |
|----|------|---------------|
| 0 | 开单 | biz_sales_order.order_id |
| 1 | 操作 | biz_operation_record.record_id |
| 2 | 还款 | biz_repayment_record.repayment_id |
| 3 | 手动新增 | null |

### 自动同步逻辑
- 开单时：从订单的 items 提取 plan_items，金额取 deal_amount，满意度/照片/反馈从订单备注提取
- 操作时：从操作记录提取品项名和次数作为 plan_items，金额取 consume_amount/trial_price，满意度/照片/反馈直接取
- 还款时：从还款记录提取套餐名作为 plan_items，金额取 repayment_amount

### 档案Tab展示样式
- 使用**卡片时间线**样式，每条档案一个卡片
- 类型用不同颜色标签区分：开单(蓝)、操作(绿)、还款(橙)、手动(紫)
- 照片支持点击预览大图
- 满意度用星级展示
