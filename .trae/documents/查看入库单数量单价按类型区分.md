# 查看入库单 — 数量/规格/进货价按单位类型区分显示

## 需求

### 主单位(整) unitType='1'
| 列 | 显示 | 示例 |
|----|------|------|
| 数量 | 实际整套数量 + 主单位标签 | `1盒` |
| 规格 | 主单位标签 | `盒` |
| 进货单价 | 原始单价 + / + 主单位标签 | `6800/盒` |

### 副单位(拆) unitType='2'
| 列 | 显示 | 示例 |
|----|------|------|
| 数量 | 实际拆分数量 + 副单位标签 | `10支` |
| 规格 | 副单位/规格标签 | `支` |
| 进货单价 | 原始单价 + / + 副单位标签 | `680/支` |

## 当前问题

- **数量列**：统一显示 `1盒（10支）` 格式，没有根据 unitType 区分
- **规格列**：已正确显示 ✅
- **进货单价列**：统一显示 `6800/盒` 格式，副单位时应为 `680/支`

## 数据来源

数据库存储的字段：
- `original_quantity` — 用户实际输入的数量
- `quantity` — 转换后的最小单位数量
- `purchasePrice` — 用户实际输入的单价
- `unitType` — '1'=主单位(整), '2'=副单位(拆)

## 修复方案

### formatItemQuantity 改造（仅查看模式）

```javascript
function formatItemQuantity(item) {
  const unitLabel = getUnitLabel(item.unit)
  const specLabel = getSpecLabel(item.spec)
  
  if (item.unitType === '1') {
    // 主单位(整)：显示原始数量 + 主单位标签
    return (item.originalQuantity || item.quantity || 0) + unitLabel
  } else {
    // 副单位(拆)：显示原始数量 + 副单位标签
    return (item.originalQuantity || item.quantity || 0) + specLabel
  }
}
```

### formatItemPrice 改造

```javascript
function formatItemPrice(item) {
  if (item.unitType === '1') {
    // 主单位(整)：原价 / 主单位
    return item.purchasePrice + '/' + (getUnitLabel(item.unit) || '')
  } else {
    // 副单位(拆)：原价 / 副单位
    return item.purchasePrice + '/' + (getSpecLabel(item.spec) || '')
  }
}
```
