# 出库单优化计划

## 需求分析

### 需求1：新增出库单 - 添加对接员工字段
企业出库时，在选择企业后面加一个"对接员工"字段（可选），用来区分对接这个企业的员工。

### 需求2：出库管理列表 - 显示单位类型
列表中需要显示单位类型/换算关系，让用户能分清是主单位还是副单位。

## 当前结构

### 新增出库单表单
```
| 出库类型 | 对象类型 | [企业出库: 出库企业] 或 [员工领用: 领用员工 | 目标企业] |
| 出库日期 | 备注                                                                  |
```

### 出库管理列表
```
| 出库单号 | 出库类型 | 出库企业 | 负责员工 | 总数量 | 总金额 | 出库日期 | 状态 |
```

## 目标结构

### 新增出库单表单
```
| 出库类型 | 对象类型 | [企业出库: 出库企业 | 对接员工] 或 [员工领用: 领用员工 | 目标企业] |
| 出库日期 | 备注                                                                        |
```

### 出库管理列表
```
| 出库单号 | 出库类型 | 出库企业/领用员工 | 对接员工 | 总数量(单位) | 总金额 | 出库日期 | 状态 |
```

## 实施步骤

### 步骤1：新增出库单 - 添加对接员工字段

#### 1.1 修改表单布局
文件：`d:\fuchenpro\front\src\views\wms\stockOut\index.vue`

在"出库企业"后面添加"对接员工"选择框：
```html
<el-col :span="12" v-if="form.outTargetType === '1'">
  <el-form-item label="出库企业" prop="enterpriseId">
    <el-select ... >...</el-select>
  </el-form-item>
</el-col>
<el-col :span="6" v-if="form.outTargetType === '1'">
  <el-form-item label="对接员工">
    <el-select v-model="form.contactEmployeeId" placeholder="选择对接员工(可选)" filterable remote :remote-method="searchEmployeeList" :loading="employeeLoading" style="width: 100%" @change="onContactEmployeeSelect">
      <el-option v-for="item in employeeOptions" :key="item.userId" :label="item.userName + (item.deptName ? '(' + item.deptName + ')' : '')" :value="item.userId" />
    </el-select>
  </el-form-item>
</el-col>
```

#### 1.2 添加表单字段
- `contactEmployeeId` - 对接员工ID
- `contactEmployeeName` - 对接员工姓名

#### 1.3 修改后端模型
文件：`d:\fuchenpro\webman\app\model\BizStockOut.php`
- 添加 `contact_employee_id`, `contact_employee_name` 字段

#### 1.4 数据库迁移
```sql
ALTER TABLE `biz_stock_out` ADD COLUMN `contact_employee_id` int DEFAULT NULL COMMENT '对接员工ID' AFTER `enterprise_name`;
ALTER TABLE `biz_stock_out` ADD COLUMN `contact_employee_name` varchar(50) DEFAULT NULL COMMENT '对接员工姓名' AFTER `contact_employee_id`;
```

### 步骤2：出库管理列表 - 显示单位类型

#### 2.1 修改列表显示
将"总数量"列改为显示单位信息：

方案A：在总数量后面显示单位
```
| 总数量 | → | 总数量(最小单位) |
```

方案B：添加新列显示明细单位信息
```
| 总数量 | 明细单位 |
```

建议采用方案A，在总数量列显示单位提示：
```html
<el-table-column label="总数量(最小单位)" prop="totalQuantity" width="120" align="center" />
```

或者显示更详细的信息，需要后端返回明细的单位类型信息。
