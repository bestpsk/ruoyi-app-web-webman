# 盘点单货品添加方式分析

## 当前实现

**方式：全部显示**
- 点击"新增盘点单"时调用 `loadInventoryData()`
- 一次性加载**所有有库存的货品**到表格
- 用户直接在各行输入实际数量

---

## 两种方案对比

| 对比项 | 方案A：全部显示（当前） | 方案B：按需添加 |
|-------|---------------------|---------------|
| **操作步骤** | 打开即看到所有货品，直接填数 | 需要逐个搜索/选择货品 |
| **适用场景** | 货品少（<20种）、全盘 | 货品多、抽盘 |
| **界面复杂度** | 表格可能很长 | 界面简洁 |
| **查找效率** | 货品多时难找 | 可搜索筛选 |
| **遗漏风险** | 低（全部列出） | 高（可能漏选） |
| **用户体验** | 一目了然 | 操作繁琐 |

---

## 行业惯例参考

### 进销存系统常见做法：

| 系统 | 盘点方式 | 说明 |
|------|---------|------|
| **金蝶/用友** | 按需添加 | 选择仓库→生成盘点单→可增删行 |
| **管家婆** | 全部显示 | 显示该仓库所有商品 |
| **速达** | 混合模式 | 默认全部，可过滤 |

### 实际业务场景：

1. **全盘（定期盘点）**：月底/年底盘点所有库存 → **适合全部显示**
2. **抽盘（日常盘点）**：随机抽查部分货品 → **适合按需添加**
3. **异常盘点**：只盘有差异的货品 → **适合按需添加**

---

## 建议：保持全部显示 + 增加搜索/筛选功能

### 理由：
1. **盘点本质是核对所有库存**，全部显示更符合业务逻辑
2. **避免遗漏** - 如果按需添加，容易漏掉某些货品
3. **当前只有2个货品**，全部显示完全没问题
4. 即使货品增多，也可以通过优化来解决

### 优化方案（如果货品多）：

```
┌─────────────────────────────────────────────┐
│ 盘点明细                    [🔍搜索货品...] │
├──────┬──────┬──────┬──────┬──────┬───────┤
│ 品名  │单位类型│换算  │系统库│实际数│规格...│
│ GCS-p7│副单位拆│1盒=10支│10支 │  [10] │ 支   │
│ 测试1 │副单位拆│1盒=10支│9支  │  [10] │ 支   │
│ ...   │      │      │      │       │      │
└──────┴──────┴──────┴──────┴──────┴───────┘
```

**可增加的功能：**
1. **搜索框** - 按品名/编码快速定位
2. **分类筛选** - 按分类/供应商筛选
3. **排序** - 按名称/库存量排序
4. **分页或虚拟滚动** - 货品特别多时
5. **只显示有库存的货品** - 过滤掉零库存

---

## 结论

**建议保持当前的"全部显示"方案**，原因：

1. ✅ 符合盘点业务逻辑（盘点 = 核对所有）
2. ✅ 避免遗漏风险
3. ✅ 操作简单直观
4. ✅ 当前货品量级完全够用

如果未来货品超过50种，再考虑增加搜索/筛选功能即可。
