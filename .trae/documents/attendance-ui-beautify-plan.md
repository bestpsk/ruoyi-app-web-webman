# 打卡页面优化计划：定位兼容 + 地址显示 + UI美化

## 问题诊断

### 1. 只显示坐标不显示地名 — 发现BUG！
**根因**：逆地理编码API路径写错了！
- 当前代码：`https://restapi.amap.com/v3/georegeo` ❌（多了个e）
- 正确路径：`https://restapi.amap.com/v3/geocode/regeo` ✅

这就是为什么坐标能获取但地址名一直不显示的原因。

### 2. QQ/Trae浏览器不能定位
- 已有降级方案但需确保超时后立即停止loading
- 部分浏览器对Geolocation支持有限，需要更好的兜底体验

### 3. 页面美化需求
- 当前页面功能完整但视觉上可以更精致
- 对齐首页的设计语言（渐变、阴影、动效）

---

## 实施步骤

### 步骤1：修复逆地理编码URL拼写错误
- `georegeo` → `geocode/regeo`

### 步骤2：定位逻辑微调
- 缩短超时到5秒（更快响应）
- 增加地址加载中的过渡态（坐标→"正在获取地址..."→最终地址）
- 确保所有分支都正确重置loading状态

### 步骤3：UI全面美化

**顶部区域**：
- 添加渐变背景头部（蓝紫渐变），白色文字
- 日期+问候语放在渐变背景中，更有层次感

**打卡按钮**：
- 增大尺寸到340rpx
- 添加脉冲动画吸引点击（未打卡时按钮有呼吸效果）
- 外圈添加旋转光环装饰

**定位卡片**：
- 成功状态增加绿色左边框指示条
- 显示精简地址（如"上海市浦东新区..."）+ 展开查看完整地址
- 坐标以小字灰色显示在下方

**今日记录卡片**：
- 上班/下班改为时间轴样式（左侧竖线+圆点+时间）
- 状态标签更醒目

**拍照区域**：
- 放大到240rpx方形
- 添加虚线边框引导感
- 拍照后显示"重新拍照"文字提示

**整体**：
- 卡片间距更舒适
- 增加微妙的入场动画（staggered fade-in）
