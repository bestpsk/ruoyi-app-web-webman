## 修复方案

### 问题1：入库单操作人/制单人显示用户名

**根本原因：**
- 登录用户信息中 `real_name` 字段可能为空
- 控制器虽然用了 `$realName ?: $userName` 回退，但如果 `real_name` 是空字符串而非 null，会优先使用空字符串

**解决方案：直接从缓存/Session获取真实姓名存储**

**文件1：[BizStockInController.php](file:///d:/fuchenpro/webman/app/controller/wms/BizStockInController.php)**

确保保存时使用真实姓名（第33-36行）：
```php
$ loginUser = $request->loginUser->user;
$data['operator_id'] = $request->loginUser->userId ?? 0;
$data['operator_name'] = trim($loginUser->real_name ?: $loginUser->user_name ?: '');
$data['create_by'] = trim($loginUser->real_name ?: $loginUser->user_name ?: '');
```

**文件2：[BizStockInService.php](file:///d:/fuchenpro/webman/app/service/BizStockInService.php)**

删除之前添加的复杂转换逻辑，保持简洁。数据库存了什么就显示什么。

---

### 问题2：出库单查看显示数字而非名称

**原因分析：**

数据库**已经正确保存**了名称（`enterprise_name='逆龄奢'`, `contact_employee_name='测试'`），但查看时显示数字。

**可能原因：**
- 后端代码做了不必要的转换判断，导致正确值被覆盖
- 需要检查 `selectStockOutById()` 的返回逻辑

**解决方案：直接读取数据库值，不做额外转换**

**文件：[BizStockOutService.php](file:///d:/fuchenpro/webman/app/service/BizStockOutService.php)**

简化 `selectStockOutById()` 第82-100行，直接使用数据库值：

```php
$stockOutArray = $stockOut->toArray();

// 直接使用数据库已保存的值，不做额外查询转换
return [
    'stockOutId' => $stockOutArray['stock_out_id'],
    'stockOutNo' => $stockOutArray['stock_out_no'],
    // ... 其他字段 ...
    'enterpriseId' => $stockOutArray['enterprise_id'],
    'enterpriseName' => $stockOutArray['enterprise_name'] ?? '-',      // 直接读
    'contactEmployeeId' => $stockOutArray['contact_employee_id'],
    'contactEmployeeName' => $stockOutArray['contact_employee_name'] ?? '-', // 直接读
    'responsibleId' => $stockOutArray['responsible_id'],
    'responsibleName' => $stockOutArray['responsible_name'] ?? '-',          // 直接读
    // ...
];
```

**删除所有 is_numeric / empty 判断和 ID 查询逻辑**

### 问题3：出库单创建人保存 username

**文件：[BizStockOutController.php](file:///d:/fuchenpro/webman/app/controller/wms/BizStockOutController.php)**

确认第33-35行使用真实姓名：
```php
$ loginUser = $request->loginUser->user;

$data['create_by'] = $realName ?: $userName;
```
