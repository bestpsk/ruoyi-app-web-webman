# 销售开单页面UI重新设计 Spec

## Why
当前销售开单页面的操作Tab布局混乱：持卡明细、持卡操作表格和表单区域垂直堆叠缺乏层次感；表单项（操作时间、操作人、满意度、顾客反馈、照片上传、备注）纵向排列过长导致页面冗余；整体视觉分区不清晰，用户操作体验差。需要通过合理的布局规划和视觉分层提升页面可用性。

## What Changes
- **重构操作Tab布局**：使用 `el-card` 和 `el-divider` 对功能区域进行逻辑分组
- **优化表单布局**：将操作表单改为更紧凑的2列/3列网格布局，减少纵向空间占用
- **统一视觉风格**：规范标题样式、间距和分隔方式
- **调整开单Tab**：微调开单区域的间距和对齐
- **优化记录Tab**：统一记录列表的展示风格

## Impact
- Affected code: `front/src/views/business/sales/index.vue`
- Affected specs: 无现有依赖

## ADDED Requirements

### Requirement: 操作Tab区域化分组设计
系统应将操作Tab内容按功能划分为清晰的视觉区域：

#### Scenario: 区域划分
- **WHEN** 用户进入操作Tab
- **THEN** 页面应显示以下独立区域：
  1. **工具栏区**：持卡明细标题 + 体验操作按钮（顶部固定）
  2. **品项选择区**：所有套餐品项平铺表格（可勾选）
  3. **操作编辑区**：选中品项后显示的操作表单（卡片包裹）
     - 品项操作表格（套餐、品项、次数、金额）
     - 操作信息表单（时间、操作人、满意度）
     - 反馈与附件（反馈文本、照片、备注）
  4. **提交操作区**：提交按钮右对齐

### Requirement: 表单紧凑布局
系统应采用紧凑的表单布局减少纵向空间：

#### Scenario: 表单网格
- **WHEN** 显示操作表单
- **THEN** 表单应采用以下布局：
  - 第一行：操作时间（span=8）| 操作人（span=8）| 满意度（span=8）
  - 第二行：顾客反馈（span=24，全宽textarea）
  - 第三行：操作前照片（span=12）| 操作后照片（span=12）
  - 第四行：备注（span=24，全宽textarea）
  - 第五行：提交按钮（右对齐）

### Requirement: 视觉分隔与层次
系统应使用统一的视觉元素区分不同功能区：

#### Scenario: 分隔元素
- **WHEN** 显示操作Tab内容
- **THEN** 页面应包含：
  - 各功能区使用 `el-card` 包裹，带阴影和圆角
  - 卡片间保持 16px 间距
  - 卡片标题使用统一的字体大小（14px）和字重（500）
  - 重要操作按钮使用主题色突出显示

## MODIFIED Requirements

### Requirement: 开单Tab布局优化
对开单Tab的布局进行微调：
- 品项表格保持不变
- 顾客反馈区域使用卡片包裹
- 合计信息栏使用浅色背景突出显示

### Requirement: 记录Tab布局统一
对记录Tab的布局进行统一：
- 订单记录和操作记录使用相同的表格样式
- 标题使用统一的格式

## REMOVED Requirements
无
