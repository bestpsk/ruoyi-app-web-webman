# 行程安排功能深度修复计划（借鉴预约管理）

## 一、核心问题分析

### 1.1 日历视图根本性问题

**当前方案失败原因：**
- el-table 的动态 rowspan/colspan 在 Vue 中**无法正常工作**
- CSS 绝对定位在 el-table 内部受限于 td 的 overflow 和 position context
- 导致行程色块不显示、列宽不均匀、拖动选择失效

**预约管理成功经验：**
- 使用 **纯 div + flex 布局**，不用 el-table
- 时间槽容器用 `display: flex` + `position: relative`
- 行程块用 `position: absolute` + `left/width 百分比`
- 完全控制每个像素，无限制

### 1.2 其他问题清单

| 问题 | 原因 | 严重程度 |
|------|------|----------|
| 日历视图单元格不显示 | el-table + 绝对定位冲突 | 🔴 致命 |
| 列宽未自动占满 | el-table 固定宽度限制 | 🔴 高 |
| 企业排班同上 | 同左 | 🔴 高 |
| 员工配置职位不显示 | 数据字段映射问题 | 🟡 中 |
| 排班列表企业列为空 | 分组逻辑问题 | 🟡 中 |
| 休息日配置 | 已支持但需验证 | 🟢 低 |

## 二、重构方案：完全采用 Div 布局（参考预约管理）

### 2.1 整体架构

```
schedule-container (div)
├── filter-section (搜索条件)
├── calendar-grid (日历网格 - 核心)
│   ├── header-row (表头：姓名 + 日期)
│   └── data-rows (数据行)
│       ├── row (每行一个员工/企业)
│       │   ├── name-column (固定左侧列)
│       │   └── days-container (日期容器 - relative)
│       │       ├── day-cell (每天格子 - 可点击)
│       │       └── schedule-block (行程块 - absolute定位)
│       └── ...
└── list-section (排班列表)
```

### 2.2 核心组件结构

```vue
<div class="calendar-grid">
  <!-- 表头 -->
  <div class="header-row">
    <div class="name-header">员工姓名</div>
    <div class="days-header">
      <div v-for="day in daysInMonth" :key="day" class="day-header">{{ day }}日</div>
    </div>
  </div>

  <!-- 数据行 -->
  <div v-for="(row, rowIndex) in scheduleData" :key="rowIndex" class="data-row">
    <!-- 左侧名称列 -->
    <div class="name-cell">
      <div class="name-text">{{ row.userName }}</div>
      <el-tag v-if="row.postName" size="small" type="info">{{ row.postName }}</el-tag>
    </div>

    <!-- 右侧日期容器 -->
    <div 
      class="days-container"
      @mousedown="handleRowMouseDown($event, row)"
      @mouseup="handleRowMouseUp"
      @mouseleave="handleMouseLeave"
    >
      <!-- 空白格子 -->
      <div
        v-for="day in daysInMonth"
        :key="day"
        :class="['day-cell', { selected: isSelectedDay(rowIndex, day), 'rest-day': isRestDay(row.userId, day) }]"
        :data-day="day"
        @mouseenter="handleCellEnter(rowIndex, day)"
      />

      <!-- 行程色块（绝对定位） -->
      <template v-for="merged in getMergedSchedules(row)" :key="'m-' + merged.startDay">
        <div
          class="schedule-block"
          :class="'purpose-' + merged.purpose + ' status-' + merged.status"
          :style="getScheduleBlockStyle(merged)"
          @click.stop="handleScheduleClick(merged)"
        >
          <el-tooltip :content="getTooltip(merged)">
            {{ activeTab === 'employee' ? merged.enterpriseName : merged.userName }}
          </el-tooltip>
        </div>
      </template>
    </div>
  </div>
</div>
```

### 2.4 计算行程块位置和宽度的函数

```javascript
function getScheduleBlockStyle(merged) {
  const totalDays = daysInMonth.value
  const startPercent = ((merged.startDay - 1) / totalDays) * 100
  const widthPercent = (merged.span / totalDays) * 100
  
  return {
    left: `${startPercent}%`,
    width: `${widthPercent}%`
  }
}
```

### 3.2 员工配置职位不显示 - 修复数据字段

**问题分析：**
后端 BizEmployeeConfigService::selectConfigList 返回的数据中 postName 字段可能为空。

**检查点：**
1. SQL 同步脚本是否正确获取了 post_name
2. 前端是否正确绑定 `postName` 字段

**修复方案：**
```sql
-- 确保 SQL 脚本正确关联岗位表
INSERT INTO biz_employee_config (...)
SELECT u.user_id, u.user_name, ..., p.post_name, ...  -- 确保有这个字段
FROM sys_user u LEFT JOIN sys_post p ON u.post_id = p.post_id
```

## 六、技术要点总结

1. **放弃 el-table 做日历** - 它不适合做这种复杂布局
2. **使用 div + flex** - 完全可控
3. **position: relative + absolute** - 行程块精确定位
4. **百分比计算位置** - 自适应容器宽度
5. **事件委托** - 在容器级别监听鼠标事件，性能更好
6. **参考预约管理** - 它的实现已验证可行
