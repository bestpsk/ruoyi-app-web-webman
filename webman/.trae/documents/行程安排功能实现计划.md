# 行程安排功能实现计划

## 一、需求概述

在业务管理菜单下新增行程安排功能，包含两个Tab页：
- **Tab页1：员工行程** - 左边显示员工，上面显示日期，可拖动选择多个天数添加行程
- **Tab页2：企业排班** - 左边显示企业，上面显示日期，可拖动选择多个天数添加行程

两个Tab页使用同一个数据表，数据相通。

## 二、数据库设计

### 2.1 创建行程安排表 `biz_schedule`

```sql
CREATE TABLE `biz_schedule` (
  `schedule_id` bigint NOT NULL AUTO_INCREMENT COMMENT '行程ID',
  `user_id` bigint NOT NULL COMMENT '员工ID',
  `user_name` varchar(50) DEFAULT NULL COMMENT '员工姓名',
  `enterprise_id` bigint NOT NULL COMMENT '企业ID',
  `enterprise_name` varchar(100) DEFAULT NULL COMMENT '企业名称',
  `schedule_date` date NOT NULL COMMENT '行程日期',
  `purpose` char(1) NOT NULL COMMENT '下店目的(1爆卡 2启动销售 3售后服务 4洽谈业务)',
  `remark` text DEFAULT NULL COMMENT '备注',
  `status` char(1) NOT NULL DEFAULT '0' COMMENT '状态(0正常 1取消)',
  `create_by` varchar(64) DEFAULT '' COMMENT '创建者',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_by` varchar(64) DEFAULT '' COMMENT '更新者',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`schedule_id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_enterprise_id` (`enterprise_id`),
  KEY `idx_schedule_date` (`schedule_date`),
  KEY `idx_user_date` (`user_id`, `schedule_date`),
  KEY `idx_enterprise_date` (`enterprise_id`, `schedule_date`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='行程安排表';
```

### 4.3 核心功能实现

#### 4.3.1 日历表格组件

**实现思路：**
1. 使用 `el-table` 实现表格
2. 第一列固定显示员工/企业名称
3. 后续列动态生成，根据当月天数（28-31天）
4. 每个单元格显示行程信息，使用 `el-tooltip` 显示详细信息

**关键代码：**
```vue
<el-table :data="scheduleData" border style="width: 100%">
  <el-table-column :label="nameLabel" prop="name" fixed width="150" />
  <el-table-column
    v-for="day in daysInMonth"
    :key="day"
    :label="day + '日'"
    width="60"
  >
    <template #default="scope">
      <div
        v-if="scope.row.schedules[day]"
        class="schedule-cell"
        @mouseenter="handleCellHover"
        @mousedown="handleCellMouseDown"
        @mouseup="handleCellMouseUp"
      >
        <el-tooltip :content="getScheduleTooltip(scope.row.schedules[day])">
          <div class="schedule-content">
            {{ scope.row.schedules[day].enterpriseName }}
          </div>
        </el-tooltip>
      </div>
    </template>
  </el-table-column>
</el-table>
```

#### 4.3.2 拖动选择功能

**实现思路：**
1. 监听鼠标按下事件，记录起始单元格
2. 监听鼠标移动事件，实时更新选择范围
3. 监听鼠标松开事件，弹出添加行程弹窗
4. 使用 CSS 高亮显示选中的单元格

**关键代码：**
```javascript
const handleCellMouseDown = (row, day) => {
  isSelecting.value = true
  startCell.value = { row, day }
  selectedCells.value = [{ row, day }]
}

const handleCellMouseMove = (row, day) => {
  if (!isSelecting.value) return
  // 计算选择范围
  updateSelectedCells(startCell.value, { row, day })
}

const handleCellMouseUp = () => {
  if (!isSelecting.value) return
  isSelecting.value = false
  // 弹出添加行程弹窗
  showAddDialog()
}
```

## 六、技术要点

### 6.1 日历视图实现
- 使用动态列生成，根据当月天数创建对应列数
- 使用固定列显示员工/企业名称
- 使用单元格合并显示连续多天的行程

### 6.2 拖动选择实现
- 监听鼠标事件实现拖动选择
- 使用 CSS 高亮显示选中区域
- 支持跨行、跨列选择

### 6.3 数据同步
- 两个Tab页使用同一个数据表
- 切换Tab页时刷新数据
- 添加/修改/删除行程后同步更新两个视图

### 6.4 性能优化
- 使用虚拟滚动优化大表格渲染
- 使用分页加载行程列表
- 使用缓存减少重复请求

## 七、注意事项

1. **数据一致性：** 确保两个Tab页的数据同步更新
2. **权限控制：** 按钮权限使用 `v-hasPermi` 指令
3. **数据验证：** 前端表单验证 + 后端数据验证
4. **异常处理：** 统一使用 `AjaxResult` 返回错误信息
5. **命名规范：** 遵循项目现有命名规范（驼峰命名、下划线命名转换）
6. **代码风格：** 遵循项目现有代码风格，不添加注释
