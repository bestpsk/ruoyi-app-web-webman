# 缓存监控数据字段名格式不匹配修复计划

## 一、当前问题现象

### 最新日志显示的数据结构
```json
{
  "code": 200,
  "msg": "",
  "info": {
    "redisVersion": "3.0.504",        // ← 驼峰命名！
    "redisGitSha1": "00000000",
    "redisGitDirty": "0",
    "redisBuildId": "a4f7a6e86f2d60b3",
    "redisMode": "standalone",        // ← 驼峰命名！
    "os": "Windows  ",
    "archBits": "64"
    // ... 更多驼峰命名字段
  }
}
```

### 前端期望的字段名（[index.vue:12-38](file:///d:\fuchenpro\front\src\views\monitor\cache\index.vue#L12-L38)）
```javascript
cache.info.redis_version              // 第12行 - 期望下划线格式
cache.info.redis_mode                // 第14行 - 期望下划线格式
cache.info.tcp_port                  // 第16行 - 期望下划线格式
cache.info.used_memory_human         // 第24行 - 期望下划线格式
cache.info.used_cpu_user_children    // 第26行 - 期望下划线格式
cache.info.maxmemory_human           // 第28行 - 期望下划线格式
cache.info.aof_enabled               // 第32行 - 期望下划线格式
cache.info.rdb_last_bgsave_status    // 第34行 - 期望下划线格式
cache.info.instantaneous_input_kbps  // 第38行 - 期望下划线格式
```

## 二、根本原因分析

### 问题1：后端返回的是驼峰命名

从日志可以确认，当前API返回的info对象中所有字段都是**驼峰命名**（camelCase）：
- ✅ `redisVersion`
- ✅ `redisMode`
- ❌ ~~`redis_version`~~ （不存在）
- ❌ ~~`redis_mode`~~ （不存在）

### 问题2：前端期望下划线命名

RuoYi原版的前端代码是基于Java Spring Boot后端的，Java后端通常会自动将驼峰转为下划线。但PHP Redis扩展直接返回原始的驼峰格式。

### 问题3：为什么之前的修改没生效？

可能的原因：
1. **Webman服务未重启**：代码修改后需要重启服务才能生效
2. **PHP Redis扩展版本差异**：不同版本返回的数据结构可能不同
3. **数据已经是扁平格式**：某些版本的phpredis返回的就是扁平的驼峰命名数组，不需要flattenInfo处理

## 三、解决方案

### 推荐方案：双重保障 - 后端转换 + 前端兼容

为了确保万无一失，采用以下策略：

#### 方案A：确保后端正确转换字段名（主要方案）

检查并强化 [CacheController.php](file:///d:\fuchenpro\webman\app\controller\monitor\CacheController.php) 的字段名转换逻辑，确保所有字段都从驼峰转为下划线。

#### 方案B：前端添加字段映射（备用方案）

如果后端转换仍有问题，在前端添加一个字段映射函数，将驼峰字段映射为下划线字段，作为兼容层。

## 四、实施步骤

### 步骤1：验证后端代码是否已生效

首先确认Webman服务是否已经重启加载了最新代码。如果没有重启，需要先重启服务。

### 步骤2：调试flattenInfo方法

在CacheController.php中添加临时日志，输出原始数据和转换后的数据，验证camelToSnake方法是否正常工作。

### 步骤3：强制转换所有字段名

修改getInfo()方法，确保对info数组的所有键执行snake_case转换：

```php
public function getInfo(Request $request)
{
    $redis = Redis::connection();
    $rawInfo = $redis->info();
    
    // 强制扁平化 + 字段名转换
    $flatInfo = [];
    foreach ($rawInfo as $key => $value) {
        if (is_array($value)) {
            foreach ($value as $subKey => $subValue) {
                $flatInfo[$this->camelToSnake($subKey)] = $subValue;
            }
        } else {
            $flatInfo[$this->camelToSnake($key)] = $value;
        }
    }
    
    $info = $flatInfo;  // 使用转换后的数据
    $dbSize = $redis->dbsize();
    // ... 其余逻辑不变
    
    return AjaxResult::success('', [
        'info' => $info,
        'dbSize' => $dbSize,
        'commandStats' => $commandStats,
    ]);
}
```

### 步骤4：测试验证

重启Webman服务后，刷新缓存监控页面，检查浏览器控制台日志，确认返回的数据字段名为下划线格式。
