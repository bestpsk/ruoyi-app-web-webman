# 行程安排功能优化与扩展计划

## 一、需求概述

对现有行程安排功能进行优化和扩展，主要包括：
1. **日历视图优化** - 连续行程合并单元格、拖动选择颜色提示、日期动态占满整列
2. **状态管理优化** - 状态改为：已预约、服务中、已完成、已取消，已取消区域可再次选中
3. **表单优化** - 员工支持多选、企业支持查找和拼音首字母查询
4. **企业拼音首字母** - 数据库增加拼音首字母字段，自动转换保存
5. **排班列表显示** - 日历视图下方显示排班列表
6. **新增员工配置Tab页** - 包含员工姓名、职位、部门、是否可排班、休息日期配置

#### 4.2.1 日历表格优化

**1. 连续行程合并单元格：**

实现思路：
- 后端返回的数据中包含每个员工的行程信息
- 前端遍历每个员工的行程数据，识别连续的相同目的行程
- 使用 el-table-column 的 :span 属性实现单元格合并
- 隐藏被合并的单元格内容

关键代码逻辑：
```javascript
function mergeSchedules(schedules) {
  const merged = []
  let currentMerge = null
  
  for (let day = 1; day <= daysInMonth.value; day++) {
    const schedule = schedules[day]
    
    if (!currentMerge) {
      currentMerge = {
        startDay: day,
        endDay: day,
        schedule: schedule,
        span: 1
      }
    } else if (schedule && currentMerge.schedule && 
               schedule.purpose === currentMerge.schedule.purpose &&
               schedule.enterpriseId === currentMerge.schedule.enterpriseId &&
               schedule.status !== '4') {
      currentMerge.endDay = day
      currentMerge.span++
    } else {
      merged.push(currentMerge)
      currentMerge = {
        startDay: day,
        endDay: day,
        schedule: schedule,
        span: 1
      }
    }
  }
  
  if (currentMerge) {
    merged.push(currentMerge)
  }
  
  return merged
}
```

**2. 拖动选择颜色提示：**

实现思路：
- 监听 mousedown、mousemove、mouseup 事件
- 记录起始单元格和当前鼠标位置
- 使用 CSS 高亮显示选中的区域
- 选中的单元格添加 .selected-cell 样式类

关键代码：
```javascript
const selectedCells = ref([])
const isSelecting = ref(false)

function handleCellMouseDown(row, day) {
  isSelecting.value = true
  startCell.value = { row, day }
  selectedCells.value = [{ row, day }]
}

function handleCellMouseMove(event, row, day) {
  if (!isSelecting.value || !startCell.value) return
  
  const startDay = Math.min(startCell.value.day, day)
  const endDay = Math.max(startCell.value.day, day)
  
  selectedCells.value = []
  for (let d = startDay; d <= endDay; d++) {
    selectedCells.value.push({ row, day: d })
  }
}

function handleCellMouseUp() {
  isSelecting.value = false
  // 弹出添加对话框
}
```

#### 4.2.2 状态管理优化

**1. 已取消区域可再次选中：**

实现思路：
- 判断单元格的状态是否为 '4'（已取消）
- 如果是已取消状态，允许拖动选择
- 其他状态只能点击查看/编辑，不能拖动选择

关键代码：
```javascript
function canSelectCell(schedule) {
  return !schedule || schedule.status === '4'
}

function handleCellMouseDown(row, day) {
  const schedule = row.schedules[day]
  if (!canSelectCell(schedule)) return
  
  isSelecting.value = true
  startCell.value = { row, day }
}
```

## 六、技术要点

### 6.1 单元格合并算法

**算法描述：**
1. 遍历每一天的行程数据
2. 判断当前行程是否与上一个行程连续且属性相同（同一员工、同一企业、同一目的、非取消状态）
3. 如果连续则合并，否则开始新的合并组
4. 返回合并后的数据结构

**数据结构：**
```javascript
{
  startDay: 6,       // 开始日期
  endDay: 10,         // 结束日期
  span: 5,            // 合并跨度
  schedule: {...}      // 行程数据
}
```

### 6.2 拖动选择实现

**事件流程：**
1. mousedown → 记录起始点，设置选择状态
2. mousemove → 计算选择范围，高亮显示
3. mouseup → 结束选择，弹出对话框

**性能优化：**
- 使用防抖处理 mousemove 事件
- 只在表格区域内监听事件
- 选择结束后及时清理事件监听

### 6.3 拼音转换方案

**后端方案：**
- 使用 PHP 的 pinyin 库（如 overtrue/pinyin）
- 在保存企业数据时自动生成拼音首字母
- 支持多音字处理

**前端方案：**
- 企业下拉框显示格式："企业名称(拼音)"
- 支持按拼音首字母过滤搜索

### 6.4 排班列表数据同步

**同步机制：**
- 切换 Tab 页时刷新列表数据
- 添加/编辑/删除行程后同时刷新日历和列表
- 列表数据从日历数据中提取，避免重复请求
